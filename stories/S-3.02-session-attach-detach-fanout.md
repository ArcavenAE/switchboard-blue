# S-3.02: Console Attach/Detach and Multi-Console Fan-Out

> **Execute:** `/vsdd-factory:deliver-story S-3.02`

## Narrative

- **As a** console operator
- **I want to** attach to a named session by name, receive the live downstream output stream, send keystrokes upstream, and detach without closing the session
- **So that** I can monitor and control remote terminal sessions collaboratively with other operators

## Acceptance Criteria

### AC-001 (traces to BC-2.04.003 postcondition 1)
`Session.Attach(console_key, session_name)` returns a bidirectional channel: downstream frames delivered to console, upstream keystrokes forwarded to access node.
- **Test:** `TestSession_Attach_BidirectionalChannel`

### AC-002 (traces to BC-2.04.004 postcondition 1)
`Session.Detach(console_key)` releases the console's attachment from the session without closing the session on the access node.
- **Test:** `TestSession_Detach_SessionContinues`

### AC-003 (traces to BC-2.04.004 postcondition 2)
After detach, any remaining attached consoles continue to receive downstream frames unaffected.
- **Test:** `TestSession_Detach_OtherObserversUnaffected`

### AC-004 (traces to BC-2.04.006 postcondition 1)
Two or more consoles may attach to the same session simultaneously; each console receives all downstream frames independently (fan-out).
- **Test:** `TestSession_MultiConsoleFanOut`

### AC-005 (traces to BC-2.04.006 postcondition 2)
Fan-out is complete: when the access node emits one downstream frame, all attached consoles receive it (no console is skipped).
- **Test:** `TestSession_FanOut_AllConsoleReceiveFrame`

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|

| EC-004 | Console crashes without explicit detach | Session detects closed channel and evicts console from fan-out set |
