# S-3.02: Console Attach/Detach and Multi-Console Fan-Out

> **Execute:** `/vsdd-factory:deliver-story S-3.02`

## Narrative

- **As a** console operator
- **I want to** attach to a named session, receive live downstream output, send upstream keystrokes, and detach without closing the session; and for multiple consoles to subscribe to the same session simultaneously
- **So that** I can monitor and control remote terminal sessions collaboratively with other operators without disrupting each other's connections

## Acceptance Criteria

### AC-001 (traces to BC-2.04.003 PC-1)
`Session.Attach(consoleKey, sessionName)` establishes a bidirectional channel: the console receives the downstream half-channel (terminal output) from the access node.
- **Test:** `TestSession_Attach_EstablishesBidirectionalChannel`

### AC-002 (traces to BC-2.04.003 PC-3)
The console's upstream half-channel (keystrokes) is accepted by the access node and forwarded to the tmux session.
- **Test:** `TestSession_Attach_UpstreamKeystrokesForwarded`

### AC-003 (traces to BC-2.04.003 EC-002)
`Session.Attach` returns `E-SES-001` ("session not found: <session_name>") when the named session does not exist.
- **Test:** `TestSession_Attach_NonexistentSession_ErrSesOne`

### AC-004 (traces to BC-2.04.004 PC-1 + PC-2)
`Session.Detach(consoleKey)` closes the console's channel cleanly. The tmux session on the access node continues running unchanged. No keystrokes are forwarded from the detached console after detach.
- **Test:** `TestSession_Detach_SessionContinues`

### AC-005 (traces to BC-2.04.004 PC-5)
After a full-access console detaches, any read-only observers already subscribed continue receiving the downstream stream unaffected.
- **Test:** `TestSession_Detach_ReadOnlyObserversUnaffected`

### AC-006 (traces to BC-2.04.006 PC-1)
Two or more consoles may attach to the same session simultaneously; each console receives all downstream frames independently (fan-out). No console is skipped.
- **Test:** `TestSession_MultiConsoleFanOut_AllReceiveFrames`

### AC-007 (traces to BC-2.04.006 Invariant 3)
Keystrokes from multiple full-access consoles attached simultaneously are serialized by the access node before forwarding to tmux — no keystroke interleaving or data corruption under concurrent sends.
- **Test:** `TestSession_ConcurrentKeystrokes_Serialized`

### AC-008 (traces to BC-2.04.004 EC-002 / BC-2.04.006)
When a console's channel closes unexpectedly (process crash without explicit detach), the access node detects the closed channel and evicts the console from the fan-out set. Remaining consoles continue unaffected.
- **Test:** `TestSession_CrashDetach_EvictsFromFanOut`

## Edge Cases

| ID | Source | Description | Expected Behavior |
|----|--------|-------------|-------------------|

| EC-002 | BC-2.04.003 EC-002 | Named session does not exist | E-SES-001 "session not found: <session_name>" |
| EC-003 | BC-2.04.003 EC-003 | Session exists but access node is unreachable | E-NET-005 "access node unreachable" |
| EC-004 | BC-2.04.004 EC-001 | Full-access console detaches; read-only observers remain | Read-only observers continue; session continues; presence: attached=false |
| EC-005 | BC-2.04.004 EC-002 | Console crashes without detach | Access node detects closed channel on next delivery attempt; evicts console; session released |
| EC-006 | BC-2.04.006 EC-003 | Two full-access consoles send keystrokes simultaneously | Both keystrokes reach tmux serialized; no crash or data corruption |
| EC-007 | BC-2.04.006 EC-004 | All consoles detach | Session continues on access node; no session teardown |
