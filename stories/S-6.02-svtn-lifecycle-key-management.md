# S-6.02: SVTN Lifecycle and Key Management via sbctl admin

> **Execute:** `/vsdd-factory:deliver-story S-6.02`

## Narrative

- **As a** control node operator
- **I want to** create SVTNs and manage admission/session-authorization key lifecycle via `sbctl admin`
- **So that** the network topology is under operator control and compromised keys can be revoked immediately

> **Scope note (CR-009):** This story covers BC-2.07.001 **PC-1 (Create) and PC-2 (Bootstrap) only**. PC-3 (Destroy) was deferred to S-6.05 per product-owner ruling 2026-06-29. The narrative, ACs, and tasks below do not include SVTN destroy operations.

## Acceptance Criteria

### AC-001 (traces to BC-2.07.001 postcondition 1)
`SVTNManager.Create(svtn_name)` creates a new SVTN with a generated SVTN-ID and bootstraps the first control key locally. Returns the SVTN-ID.
- **Test:** `TestSVTNManager_Create_BootstrapsControlKey`

### AC-002 (traces to BC-2.05.004 postcondition 1)
`sbctl admin key register --key <pubkey> --svtn <id>` registers an admission key for the SVTN. The key appears in subsequent admission checks. (CLI dispatch only; end-to-end behavior — daemon-side handler registration — is verified in S-6.06.)
- **Test:** `TestSbctlAdmin_KeyRegister_CLI`

### AC-003 (traces to BC-2.05.004 postcondition 2)
`sbctl admin key revoke --key <pubkey> --svtn <id>` removes the key from the admission set. Subsequent admission attempts with that key return `E-ADM-002`. (CLI dispatch only; end-to-end behavior — daemon-side handler registration — is verified in S-6.06.)
- **Test:** `TestSbctlAdmin_KeyRevoke_CLI`

### AC-004 (traces to BC-2.05.004 postcondition 3)
`sbctl admin key expire --key <pubkey> --after <duration>` sets a TTL; after expiry, the key behaves as revoked. (CLI dispatch only; end-to-end behavior — daemon-side handler registration — is verified in S-6.06.)
- **Test:** `TestSbctlAdmin_KeyExpire_CLI`

### AC-005 (traces to BC-2.05.004 precondition 1)
Control-to-control revocation requires `sbctl admin` human authorization (a `--confirm` flag or equivalent); revocation without confirmation is rejected. (CLI dispatch only; end-to-end behavior — daemon-side handler registration — is verified in S-6.06.)
- **Test:** `TestSbctlAdmin_ControlRevocation_RequiresConfirm`
