# S-6.02: SVTN Lifecycle and Key Management via sbctl admin

> **Execute:** `/vsdd-factory:deliver-story S-6.02`

## Narrative

- **As a** control node operator
- **I want to** create and destroy SVTNs and manage admission/session-authorization key lifecycle via `sbctl admin`
- **So that** the network topology is under operator control and compromised keys can be revoked immediately

## Acceptance Criteria

### AC-001 (traces to BC-2.07.001 postcondition 1)
`SVTNManager.Create(svtn_name)` creates a new SVTN with a generated SVTN-ID and bootstraps the first control key locally. Returns the SVTN-ID.
- **Test:** `TestSVTNManager_Create_BootstrapsControlKey`

### AC-002 (traces to BC-2.05.004 postcondition 1)
`sbctl admin key register --key <pubkey> --svtn <id>` registers an admission key for the SVTN. The key appears in subsequent admission checks.
- **Test:** `TestSbctlAdmin_KeyRegister`

### AC-003 (traces to BC-2.05.004 postcondition 2)
`sbctl admin key revoke --key <pubkey> --svtn <id>` removes the key from the admission set. Subsequent admission attempts with that key return `E-ADM-002`.
- **Test:** `TestSbctlAdmin_KeyRevoke`

### AC-004 (traces to BC-2.05.004 postcondition 3)
`sbctl admin key expire --key <pubkey> --after <duration>` sets a TTL; after expiry, the key behaves as revoked.
- **Test:** `TestSbctlAdmin_KeyExpire`

### AC-005 (traces to BC-2.05.004 invariant 1)
Control-to-control revocation requires `sbctl admin` human authorization (a `--confirm` flag or equivalent); revocation without confirmation is rejected.
- **Test:** `TestSbctlAdmin_ControlRevocation_RequiresConfirm`

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | Register key that is already registered | Last-write-wins per ADR-003 |
| EC-002 | Revoke key that is not registered | E-ADM-013 (key not found) |
| EC-003 | Expire duration of zero | E-CFG-001: invalid duration |
