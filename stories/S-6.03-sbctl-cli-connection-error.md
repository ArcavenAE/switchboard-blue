# S-6.03: sbctl Unified CLI with OpenSSH Auth and Connection Error Reporting

> **Execute:** `/vsdd-factory:deliver-story S-6.03`

## Narrative

- **As an** operator
- **I want to** use a single `sbctl` binary to manage any daemon type (router, access, console, control) with OpenSSH key authentication, and get a clear error when the daemon is unreachable
- **So that** I can operate the entire Switchboard network from one CLI tool

## Acceptance Criteria

### AC-001 (traces to BC-2.07.002 postcondition 1)
`sbctl --target <addr>` can connect to all four daemon types (router, access, console, control) using the same binary and OpenSSH key challenge-response authentication.
- **Test:** `TestSbctl_UnifiedTargeting_AllDaemonTypes`

### AC-002 (traces to BC-2.07.002 postcondition 2)
All sbctl commands accept `--json` flag and output the common JSON envelope format from interface-definitions.md.
- **Test:** `TestSbctl_JSONOutputEnvelope`

### AC-003 (traces to BC-2.07.003 postcondition 1)
When `sbctl` cannot connect to the daemon (connection refused, socket absent, timeout), it prints `E-NET-001` with the attempted address to stderr and exits with code 1.
- **Test:** `TestSbctl_ConnectionRefused_ExitsWithENET001`

### AC-004 (traces to BC-2.07.003 postcondition 2)
sbctl never produces successful-looking output when the daemon is unreachable. No partial results appear before the error.
- **Test:** `TestSbctl_NoPartialOutputOnConnectionFailure`
