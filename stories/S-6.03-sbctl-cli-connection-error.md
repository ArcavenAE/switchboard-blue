# S-6.03: sbctl Client Auth, Flag Parsing, JSON Envelope, and Connection Error Reporting

> **Execute:** `/vsdd-factory:deliver-story S-6.03`

## Scope Note (v2.0 re-scope)

This story implements the **client side** of the ADR-012 management plane in
`cmd/sbctl` only. The daemon-side management server (`internal/mgmt`) is
implemented by S-W5.01. The E2E integration harness is S-W5.02. This story
does NOT need the server to exist ã `Authenticate()` is unit-tested with a
`net.Pipe` mock server.

**Scope in:** `cmd/sbctl/main.go` (flag parsing, subcommand routing, JSON
envelope output), `cmd/sbctl/client.go` (connection, `Authenticate()`, RPC
dispatch).

**Scope out:** `internal/mgmt` (S-W5.01); e2e across all four daemon types
(S-W5.02).

## Narrative

- **As an** operator
- **I want to** use a single `sbctl` binary with OpenSSH key authentication that
  gracefully handles connection failures and authentication failures
- **So that** I can operate the Switchboard management plane securely and get
  clear, actionable errors when something goes wrong

## Acceptance Criteria

### AC-001 (traces to BC-2.07.002 postcondition 2 ã OpenSSH key load)
`sbctl --key <path>` loads an Ed25519 private key from an OpenSSH PEM file using
`golang.org/x/crypto/ssh.ParsePrivateKey`. The key file is read with
`io.LimitReader(f, 1<<16)` (64 KiB limit). The private key is extracted as
`ed25519.PrivateKey` and passed to `Authenticate()`. The key is never logged
or serialized.
- **Test:** `TestSbctl_KeyLoading_Ed25519` ã verify that a well-formed Ed25519
  OpenSSH private key file is successfully loaded and the resulting
  `ed25519.PrivateKey` has the correct length (64 bytes). Verify that a file
  larger than 64 KiB is rejected with an error.

### AC-002 (traces to BC-2.07.002 postcondition 2 ã Authenticate() fail-closed, VP-067)
`Authenticate(conn net.Conn, privKey ed25519.PrivateKey) error` in
`cmd/sbctl/client.go` is fail-closed per ADR-012 ôÏAuthenticate() FAIL-CLOSED
Contract:
1. Read CHALLENGE message via `json.NewDecoder(io.LimitReader(conn, 1<<16))`.
2. Decode `nonce_bytes` from the base64url `nonce` field; return error if absent
   or not valid base64url-encoded 32 bytes.
3. Send `{"type":"challenge_response","nonce_sig":"<base64url sig>","pubkey":"<base64url pubkey>"}`.
4. Read AUTH_OK or AUTH_FAIL response via `json.NewDecoder(io.LimitReader(conn, 1<<16))`.
5. Return `nil` **only** if `{"type":"auth_ok"}` was received and decoded.
6. Return non-nil error for: connection error, malformed CHALLENGE, AUTH_FAIL,
   any other response type, truncated stream, oversized response.
- **Test:** `TestAuthenticate_FailClosed_VP067` ã table-driven test covering
  all 8 failure sub-cases from VP-067.md plus the happy-path AUTH_OK. Each
  sub-case uses `net.Pipe` as the transport.

### AC-003 (traces to BC-2.07.002 postcondition 4 ã auth failure exit)
When `Authenticate()` returns a non-nil error (any cause), sbctl prints
`E-ADM-010 "authentication failed"` to stderr and exits with code 1. No stdout
output is produced.
- **Test:** `TestSbctl_AuthFailure_ExitsOneWithEADM010` ã mock server that
  sends AUTH_FAIL; verify stderr contains "E-ADM-010" and exit code is 1.

### AC-004 (traces to BC-2.07.003 postcondition 1 ã E-NET-001 connection error, VP-030)
When `sbctl` cannot connect to the daemon (connection refused, socket absent,
timeout), it prints `E-NET-001 "daemon unreachable: <address>: <reason>"` to
stderr and exits with code 1. No stdout output is produced.
- **Test:** `TestSbctl_ConnectionRefused_ExitsOneWithENET001_VP030` ã target
  a non-listening address; verify stderr contains "E-NET-001" and exit code 1.

### AC-005 (traces to BC-2.07.003 postcondition 3 ã no partial output)
When the daemon is unreachable, sbctl produces zero stdout output. The error
message appears only on stderr.
- **Test:** `TestSbctl_NoStdoutOnConnectionFailure` ã verify that stdout is
  empty when connection is refused.

### AC-006 (traces to BC-2.07.002 postcondition 3 ã JSON envelope output)
All sbctl subcommands accept `--json` flag and produce output conforming to the
JSON envelope from `interface-definitions.md ôÏJSON Output Schema`:
`{"ok":true,"error":null,"data":{...}}` on success,
`{"ok":false,"error":{"code":"E-xxx","message":"..."},"data":null}` on failure.
- **Test:** `TestSbctl_JSONEnvelopeFormat` ã verify that `--json` produces a
  valid JSON object with `ok`, `error`, and `data` fields for both success and
  error outcomes.

### AC-007 (traces to BC-2.07.003 invariant 2 ã connection timeout)
`sbctl` does not hang indefinitely when the daemon is unreachable. The connection
timeout defaults to 5 seconds and is configurable via `--timeout`. After timeout,
sbctl exits with E-NET-001.
- **Test:** `TestSbctl_ConnectionTimeout` ã use a listener that accepts but
  never reads; set `--timeout=100ms`; verify sbctl exits with E-NET-001 within
  500ms.

## Architecture Compliance Rules (MANDATORY)

| Rule | Source | Enforcement |
