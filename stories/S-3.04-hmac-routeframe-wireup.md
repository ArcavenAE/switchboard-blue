# S-3.04: HMAC Wire-Up into RouteFrame

> **Execute:** `/vsdd-factory:deliver-story S-3.04`

## Narrative

- **As a** router
- **I want** `RouteFrame` to call `verifyFrameHMAC` before the admitted-set check and before `SVTNRoute`
- **So that** forged or tampered frames are rejected at the wire boundary before touching any forwarding state, satisfying DI-006 and BC-2.05.008

## Context

`verifyFrameHMAC` is fully implemented in `internal/routing/routing.go` and tested in `routing_internal_test.go` (adversary pass-4 H-1 fix). It carries `//nolint:unused` at line 153 because it is not yet called from `RouteFrame`. This story wires it in: `RouteFrame` must call `verifyFrameHMAC` as the first operation after outer header parsing (ADR-009), returning `ErrHMACVerificationFailed` on any tag mismatch or missing forwarding-table entry. The `//nolint:unused` annotation is removed as part of this story.

This story modifies `internal/routing/routing.go` only. No new packages are created.

## Acceptance Criteria

### AC-001 (traces to BC-2.05.008 PC-1)
`RouteFrame` with a valid HMAC tag (tag matches `verifyFrameHMAC` over the zeroed-header + payload with `entry.FrameAuthKey`) proceeds to the admitted-set check and then to `SVTNRoute`. The frame is processed normally.
- **Test:** `TestRouteFrame_ValidHMAC_ProceedsToAdmission`

### AC-002 (traces to BC-2.05.008 PC-2)
`RouteFrame` with an invalid HMAC tag (any tag mismatch) returns `routing.ErrHMACVerificationFailed` immediately. Neither `IsAdmitted` nor `SVTNRoute` is reached. E-ADM-016 log entry is written.
- **Test:** `TestRouteFrame_InvalidHMAC_ReturnsErrHMACVerificationFailed`

### AC-003 (traces to BC-2.05.008 PC-3 / VP-058 property 1+2)
HMAC verification occurs BEFORE the admitted-set check. Verified via `errors.Is(err, routing.ErrHMACVerificationFailed)` — if the admitted-set check fired first, the error would be `admission.ErrNotAdmitted`. Test constructs a frame from an admitted node with an invalid HMAC tag to distinguish the two paths.
- **Test:** `TestRouteFrame_HMACEnforcedBeforeAdmission`

### AC-004 (traces to BC-2.05.008 PC-4 / VP-058 property 4)
When no forwarding-table entry exists for `(hdr.SVTNID, hdr.SrcAddr)`, `RouteFrame` returns `routing.ErrHMACVerificationFailed` (auth key unavailable → frame treated as unverifiable → dropped). The admitted-set check is not reached.
- **Test:** `TestRouteFrame_NoForwardingEntry_RejectsAsUnverifiable`

### AC-005 (traces to BC-2.05.008 EC-005 / invariant 2)
`routing.ErrHMACVerificationFailed` and `admission.ErrNotAdmitted` are distinct sentinels. A frame with valid HMAC from a node NOT in the admitted set returns `admission.ErrNotAdmitted` (not `ErrHMACVerificationFailed`). Callers use `errors.Is` to distinguish the two.
- **Test:** `TestRouteFrame_ValidHMAC_Unadmitted_ReturnsErrNotAdmitted`

## Edge Cases

| ID | Source | Description | Expected Behavior |
|----|--------|-------------|-------------------|
| EC-001 | BC-2.05.008 EC-001 | `HMACTag` field all-zeros | `verifyFrameHMAC` returns false; `ErrHMACVerificationFailed`; E-ADM-016 logged |
| EC-002 | BC-2.05.008 EC-002 | HMAC computed under a different node's key (cross-node forgery) | `verifyFrameHMAC` returns false; same rejection path (E-ADM-016 logged) |
| EC-003 | BC-2.05.008 EC-003 | Node is in admitted set but forwarding-table entry purged | No auth key → `ErrHMACVerificationFailed`; admitted-set never reached |
| EC-004 | BC-2.05.008 EC-004 | Empty-tick frame (zero-length payload) with correct HMAC | `verifyFrameHMAC` returns true; frame proceeds normally |
| EC-005 | BC-2.05.008 EC-005 | Valid HMAC, node not in admitted set | HMAC passes; admitted-set check then returns `admission.ErrNotAdmitted` (E-ADM-003) — two distinct errors |

## Previous Story Intelligence

| Decision | Rationale | Applies To |
