# S-1.01: Implement 44-Byte Outer Header Codec in internal/frame

> **Execute:** `/vsdd-factory:deliver-story S-1.01`

## Narrative

- **As a** router or access node
- **I want to** encode and decode Switchboard frames using the 44-byte outer header layout
- **So that** all nodes on the SVTN share a single wire format that routers can forward without parsing session content

## Acceptance Criteria

### AC-001 (traces to BC-2.01.004 postcondition 1)
`EncodeOuterHeader` produces exactly 44 bytes for any valid frame header. Layout: version(1), frame_type(1), payload_len(2), svtn_id(16), src_addr(8), dst_addr(8), hmac_tag(8).
- **Test:** `TestEncodeOuterHeader_ExactlyFortyFourBytes`

### AC-002 (traces to BC-2.01.004 postcondition 2)
`ParseOuterHeader(EncodeOuterHeader(h)) == h` for all valid headers (round-trip identity).
- **Test:** `TestParseEncodeRoundTrip`

### AC-003 (traces to BC-2.01.004 precondition 1)
`ParseOuterHeader` returns `E-FRM-001` when the input slice is shorter than 44 bytes.
- **Test:** `TestParseOuterHeader_TooShort`

### AC-004 (traces to BC-2.01.004 precondition 2)
`ParseOuterHeader` returns `E-FRM-002` when `version` field does not equal the current protocol version.
- **Test:** `TestParseOuterHeader_VersionMismatch`

### AC-005 (traces to BC-2.01.005 invariant 1)
The `channel_header` bytes (everything after the 44-byte outer header) pass through routers without parsing. `EncodeOuterHeader` does not include channel header fields; `ParseOuterHeader` does not decode them.
- **Test:** `TestChannelHeaderOpaque_NotInOuterHeader`

### AC-006 (traces to BC-2.01.006 postcondition 1)
`DeriveNodeAddress(svtn_id, public_key_bytes)` is deterministic: same inputs always produce same 8-byte node address using SHA-256(svtn_id || public_key).
- **Test:** `TestDeriveNodeAddress_Deterministic`

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | payload_len field exceeds actual payload length | ParseOuterHeader succeeds (outer header valid); caller responsible for payload bounds |
| EC-002 | svtn_id is all zeros | Valid per wire format; admission layer rejects if not registered |
| EC-003 | dst_addr equals src_addr | Valid encoding; routing layer handles loopback detection |
| EC-004 | hmac_tag is all zeros | Valid encoding; HMAC verification at router boundary rejects if key is active |
