# S-1.03: Session Continuity via Cryptographic Re-Authentication

> **Execute:** `/vsdd-factory:deliver-story S-1.03`

## Narrative

- **As a** console operator
- **I want to** maintain my active session when my IP address changes (e.g., laptop roaming between Wi-Fi and cellular)
- **So that** my terminal session is not interrupted by network topology changes

## Acceptance Criteria

### AC-001 (traces to BC-2.01.007 postcondition 1)
When a node's source IP changes, the node re-authenticates using the same keypair and the session resumes without data loss on the new path.
- **Test:** `TestSessionContinuity_ReauthOnIPChange`

### AC-002 (traces to BC-2.01.007 precondition 1)
Re-authentication is rejected if the keypair presented does not match the originally admitted keypair for the session.
- **Test:** `TestSessionContinuity_WrongKeyRejected`

### AC-003 (traces to BC-2.01.007 postcondition 2)
After successful re-authentication, the node address (derived from SVTN-ID and public key) remains stable — IP change does not alter the logical node address.
- **Test:** `TestSessionContinuity_NodeAddressStableAfterReauth`

| ID | Scenario | Expected Behavior |
