# S-6.07: Register `admin.svtn.create` Handler and `sbctl admin svtn create` CLI Subcommand

## Summary

Register the `admin.svtn.create` RPC handler in `BuildAdminHandlers` (control-mode daemon only) and wire the `sbctl admin svtn create --name=<svtn-name>` CLI subcommand. Adds caller-key-role enforcement (BC-2.07.001 Inv-3) so only RoleControl callers may invoke the handler. Completes the RPC-reachability gap identified in S-6.02 lens3 F-003 / F-010.

## Context

S-6.02 implemented `SVTNManager.Create()` (the Go method + local side-effects). S-6.06 wired `admin.key.*` handlers into `BuildAdminHandlers`. Neither story registered an `admin.svtn.create` handler or CLI subcommand, leaving BC-2.07.001 PC-1 without end-to-end RPC reachability (SCOPE DECISION: S-6.02 lens3 F-003 + F-010, both HIGH). This story closes that gap.

Approved: HUMAN-APPROVED PATH B (2026-06-29 adversarial reconverge pass).

## Architecture Compliance Rules

- No new external dependencies; stdlib only (encoding/json, net, crypto/sha256 via base64).
- `internal/mgmt` MUST NOT import `cmd/sbctl` (forbidden direction per module boundary).
