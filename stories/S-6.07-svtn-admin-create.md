# S-6.07: Register `admin.svtn.create` Handler and `sbctl admin svtn create` CLI Subcommand

## Summary

Register the `admin.svtn.create` RPC handler in `BuildAdminHandlers` (control-mode daemon only) and wire the `sbctl admin svtn create --name=<svtn-name>` CLI subcommand. Adds bootstrap-only authority enforcement (BC-2.07.001 Inv-3 tightened per Ruling-2 Option A): only the daemon bootstrap key may invoke the handler; cross-SVTN control-role keys are not authorized. Completes the RPC-reachability gap identified in S-6.02 lens3 F-003 / F-010.

## Context

S-6.02 implemented `SVTNManager.Create()` (the Go method + local side-effects). S-6.06 wired `admin.key.*` handlers into `BuildAdminHandlers`. Neither story registered an `admin.svtn.create` handler or CLI subcommand, leaving BC-2.07.001 PC-1 without end-to-end RPC reachability (SCOPE DECISION: S-6.02 lens3 F-003 + F-010, both HIGH). This story closes that gap.

Approved: HUMAN-APPROVED PATH B (2026-06-29 adversarial reconverge pass).

## Wire Envelope Contract (Ruling-11)

Per the project wire-envelope convention formalized in Ruling-11 (`decisions/wave-6-tranche-a-scope-rulings.md` v1.6), the mgmt-layer response envelope uses a two-level error code scheme:

- **`resp.Error.Code`** is always the **envelope code** (mgmt-layer authoritative). For handler-originated errors this is `E-RPC-011` (unknown/unmapped dispatch or handler-level rejection). For transport-level errors this is the transport code (e.g., `E-RPC-002` for decode failure).
- **`resp.Error.Message`** carries the **handler-specific code as prefix** in the form `"E-XXX-YYY: <human message>"`. Clients that need to discriminate handler-specific failure modes MUST parse the message prefix.
- **Transport-level errors are not prefixed**: when the error originates at the mgmt layer (e.g., `E-RPC-002` decode failure), the `Code` IS the transport code and the `Message` does not carry a handler-code prefix — the error never entered the handler.

This means:
- AC-003 authority failures: `{error: {code: "E-RPC-011", message: "E-ADM-009: ..."}}` — envelope is `E-RPC-011`, handler code `E-ADM-009` in message prefix.
- AC-005 duplicate-name failures: `{error: {code: "E-RPC-011", message: "E-SVTN-001: ..."}}` — same pattern.
- AC-004 transport decode failures (E-RPC-002): `{error: {code: "E-RPC-002", message: "..."}}` — no prefix, transport code is authoritative.

### Universality (Ruling-12)

**ALL handler-code errors follow the same envelope pattern — `{error: {code: "E-RPC-011", message: "E-XXX-YYY: <detail>"}}`.** This applies to E-ADM-*, E-SVTN-*, E-CFG-*, E-INT-*, and any future handler-code family. Only transport-layer codes (E-RPC-002 args-decode, E-RPC-004 handler-not-found, etc.) surface as the envelope `code` directly without a prefix in `message`.

Enumerated for S-6.07:

| Error Code | Wire Envelope |
|------------|--------------|
| E-ADM-009 | `{code: "E-RPC-011", message: "E-ADM-009: ..."}` |
| E-SVTN-001 | `{code: "E-RPC-011", message: "E-SVTN-001: SVTN already exists: <name>"}` |
| E-CFG-001 (args validation) | `{code: "E-RPC-011", message: "E-CFG-001: <detail>"}` |
| E-INT-001 (crypto/rand failure) | `{code: "E-RPC-011", message: "E-INT-001: admin.svtn.create: <detail>"}` |
| E-RPC-002 (args decode) | `{code: "E-RPC-002", message: "..."}` (transport — no prefix) |

The `role == RoleControl` check applies only when `HasAnySVTN() == true` (non-genesis path). On genesis create (`HasAnySVTN() == false`), `IsBootstrapKey` alone gates — the bootstrap role is guaranteed by the manager constructor and no keySet entry yet exists. Cross-SVTN control-role keys are NOT authorized for `admin.svtn.create`.
- **Live implementation path:** `cmd/switchboard/admin_handlers.go` `makeAdminSVTNCreateHandler`. Dead symbols in `internal/mgmt/handlers_admin.go` (`MakeAdminSVTNCreateHandler`, `SVTNCreator`, `SVTNCreateResult`, `adminSVTNCreateArgs`, `adminSVTNCreateResponse`) MUST be deleted (F-P1L3-002).
- `BootstrapFingerprint()` (exported) output format is `"SHA256:<base64>"` (BC-2.05.004 PC-4 canonical SSH format). The response field `bootstrap_fingerprint` MUST carry this value verbatim. Do NOT call the unexported `keyFingerprint()` from handler code (F-P1L3-004).

- `internal/mgmt` MUST NOT import `cmd/sbctl` (forbidden direction per module boundary).
