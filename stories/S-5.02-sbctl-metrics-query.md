# S-5.02: Expose Per-Path Metrics via sbctl router status

> **Execute:** `/vsdd-factory:deliver-story S-5.02`

## Narrative

- **As an** operator
- **I want to** query per-path RTT and loss metrics via `sbctl router status`
- **So that** I can diagnose connection quality without having to interpret raw logs

## Acceptance Criteria

### AC-001 (traces to BC-2.06.003 postcondition 1)
`sbctl router status --target <router>` outputs a table with per-path RTT (ms), loss (%), and quality indicator (green/yellow/red) in JSON when `--json` is passed.
- **Test:** `TestSbctlRouterStatus_OutputsPaths`

### AC-002 (traces to BC-2.06.003 postcondition 2)
When `--json` is passed, output conforms to the common JSON envelope format from interface-definitions.md; error responses include E-code.
- **Test:** `TestSbctlRouterStatus_JSONEnvelope`

## Architecture Mapping

| Component | Module | Pure/Effectful |
|-----------|--------|---------------|
| sbctl router status | cmd/sbctl | effectful |
| metrics RPC handler | internal/metrics | pure-core (query) |

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | Router has no paths yet | Empty paths array in output; no error |
| EC-002 | Router daemon unreachable | E-NET-001 exit code 1 (per BC-2.07.003) |
