# S-7.01: XOR Parity FEC for Single-Loss Recovery

> **Execute:** `/vsdd-factory:deliver-story S-7.01`

## Narrative

- **As a** router
- **I want to** add XOR parity frames that cover groups of data frames
- **So that** a single lost frame in a group can be recovered without a retransmit request

## Acceptance Criteria

### AC-001 (traces to BC-2.02.007 postcondition 1)
`FEC.Encode(frame_group)` produces one parity frame for every N data frames (N=4 default per ADR-002). The parity frame has `frame_type=fec=0x05`.
- **Test:** `TestFEC_Encode_ProducesParityFrame`

### AC-002 (traces to BC-2.02.007 postcondition 3)
`FEC.Recover(frame_group_with_gap, parity_frame)` recovers the missing frame using XOR when exactly one frame is missing from the group.
- **Test:** `TestFEC_Recover_SingleLoss`

### AC-003 (traces to BC-2.02.007 postcondition 4)
`FEC.Recover` returns `ErrTooManyLosses` when more than one frame is missing from the group (XOR only recovers single loss).
- **Sentinel:** `ErrTooManyLosses` is a package-local sentinel (`errors.New("fec: too many losses in group")`) in `internal/arq`. It has no operator-visible surface (no error-taxonomy code, no E-FEC-* slot) — it is consumed internally by the ARQ layer which triggers retransmit (AC-004). Do not expose via RPC or operator messaging.
- **Test:** `TestFEC_Recover_TwoLossesFail`

### AC-004 (traces to BC-2.02.007 postcondition 4 + VP-043 composition)
When `FEC.Recover` returns `ErrTooManyLosses` (2+ losses in the group), the ARQ layer triggers retransmit for the missing frames (falls back to BC-2.02.005). The caller that receives `ErrTooManyLosses` MUST NOT drop the frame group silently — it MUST invoke the ARQ SACK/retransmit path.
- **Test:** `TestFEC_FallbackToARQ_OnMultiLoss`
- **Oracle:** Test uses a `handle` closure that branches on `errors.Is(err, ErrTooManyLosses)` before invoking `GapsToRetransmit`. Both the positive path (ErrTooManyLosses → ARQ engaged, gaps non-empty, lost seqs present) and the negative path (nil err → handle returns nil, ARQ NOT invoked) must be asserted. See RULING-W6TB-E-s701-oracles.md §H-1.

### AC-005 (traces to BC-2.02.007 edge case EC-001)
`FEC.Encode` does not emit a parity frame when the session ends mid-group (the last group is incomplete — fewer than N data frames collected before session close). The incomplete group is passed to ARQ for normal handling.
- **Test:** `TestFEC_Encode_IncompleteLastGroup_NoParity`
