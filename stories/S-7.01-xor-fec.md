# S-7.01: XOR Parity FEC for Single-Loss Recovery

> **Execute:** `/vsdd-factory:deliver-story S-7.01`

## Narrative

- **As a** router
- **I want to** add XOR parity frames that cover groups of data frames
- **So that** a single lost frame in a group can be recovered without a retransmit request

## Acceptance Criteria

### AC-001 (traces to BC-2.02.007 postcondition 1)
`FEC.Encode(frame_group)` produces one parity frame for every N data frames (N=4 default per ADR-002). The parity frame has `frame_type=fec=0x05`.
- **Test:** `TestFEC_Encode_ProducesParityFrame`

### AC-002 (traces to BC-2.02.007 postcondition 2)
`FEC.Recover(frame_group_with_gap, parity_frame)` recovers the missing frame using XOR when exactly one frame is missing from the group.
- **Test:** `TestFEC_Recover_SingleLoss`

### AC-003 (traces to BC-2.02.007 precondition 1)
`FEC.Recover` returns `ErrTooManyLosses` when more than one frame is missing from the group (XOR only recovers single loss).
- **Test:** `TestFEC_Recover_TwoLossesFail`
