# S-7.02: SVTN-Scoped Multicast Session Discovery

> **Execute:** `/vsdd-factory:deliver-story S-7.02`

## Narrative

- **As a** console operator
- **I want to** enumerate all available sessions on my SVTN without specifying hostnames or IP addresses
- **So that** I can attach to any session by name, even if I don't know which access node is hosting it

## Acceptance Criteria

### AC-001 (traces to BC-2.03.001 postcondition 1)
`Discovery.Advertise(session_presence)` sends a presence advertisement on SVTN-scoped multicast on session state change and repeats every 30s (heartbeat interval).
- **Test:** `TestDiscovery_Advertise_OnStateChange`

### AC-002 (traces to BC-2.03.002 postcondition 1)
`Discovery.Enumerate()` returns all sessions visible on the local SVTN without requiring hostnames or IP addresses from the caller.
- **Test:** `TestDiscovery_Enumerate_NoHostnameRequired`

### AC-003 (traces to BC-2.03.003 postcondition 1)
Presence advertisement payload includes: session_name, attachment_status (attached/detached), and quality_indicator (green/yellow/red).
- **Test:** `TestDiscovery_Advertisement_RequiredFields`

### AC-004 (traces to BC-2.03.003 invariant 1)
Presence advertisement fields are stable across round-trips: `Encode(Decode(payload)) == payload` for any valid advertisement.
- **Test:** `TestDiscovery_AdvertisementRoundTrip`
