# S-3.01b: PTY Proxy Fallback

> **Execute:** `/vsdd-factory:deliver-story S-3.01b`

## Narrative

- **As an** access node
- **I want to** fall back to PTY proxy mode when tmux control mode is unavailable (at initial connect or after mid-session loss)
- **So that** the access node remains functional even on hosts without tmux or after control mode failures, and operators are never left with a silent failure

## Acceptance Criteria

### AC-001 (traces to BC-2.04.002 PC-1)
When `TmuxControlMode.Connect()` fails (tmux not found in PATH, socket absent, or `-C` flag not supported), the access node enters PTY proxy mode automatically. The fallback is triggered on initial connection failure.
- **Test:** `TestPTYProxy_FallbackOnInitialConnectFailure`

### AC-002 (traces to BC-2.04.002 PC-2 + PC-3)
In PTY proxy mode, the access node publishes the PTY session under a synthetic name (e.g., `"pty-<pid>"`), and writes a log entry of the form `"tmux control mode unavailable; using PTY proxy mode. Functionality limited: no structured session metadata, no content-type detection."` No silent failure.
- **Test:** `TestPTYProxy_PublishesSessionAndLogs`

### AC-003 (traces to BC-2.04.002 EC-004)
When both tmux control mode and PTY device are unavailable, `Connect()` returns `E-SYS-001` ("PTY device unavailable; cannot start access node") and the access node exits with a non-zero status. The failure is never silent.
- **Test:** `TestPTYProxy_NoPTY_ReturnsErrSysOne`

## Edge Cases

| ID | Source | Description | Expected Behavior |
|----|--------|-------------|-------------------|
| EC-001 | BC-2.04.002 EC-001 | tmux exists but old version (no `-CC` support) | PTY fallback; log: "tmux version does not support -CC flag" |
| EC-002 | BC-2.04.002 EC-002 | tmux not found in PATH | PTY fallback immediately; log: "tmux binary not found; using PTY proxy" |
| EC-003 | BC-2.04.002 EC-003 | tmux control mode drops after successful start (mid-operation) | Access node attempts control mode reconnect; if reconnect fails after 3 attempts, switches to PTY proxy mode. Log: "tmux control mode lost; falling back to PTY proxy" |
| EC-004 | BC-2.04.002 EC-004 | tmux unavailable AND PTY device unavailable | E-SYS-001 returned; non-zero exit; never silent |
