# S-7.04-FU-PE-CONNECTOR: Outbound TCP Dial Loop on PE Graduation

## Narrative

- **As an** operator who has graduated a router from E to PE mode (via SIGHUP reload)
- **I want** the router to actually dial and maintain TCP connections to the configured
  upstream routers
- **So that** node sessions on this PE router have access to alternate path options
  (multi-hop routing) per BC-2.09.001 PC-2/PC-3

## Context

S-7.04 delivered the *emission seam* for PE graduation: `upstreamRoutersFor(cfg)` and
`mode=PE upstream_routers=[...]` startup emission. The `keepaliveIntervalFor` helper is
in place (normative-fence test `TestKeepaliveIntervalNotSweepDeadline` guards it against
conflation with `sweepDeadline`). What is explicitly deferred is the TCP dial loop that
consumes `upstreamRoutersFor(cfg)` and establishes actual upstream router connections.

The outer-header wire-frame primitive required for session bootstrap was delivered by
S-BL.OA (merged PR #96, `internal/outerassembler`). The ARQ retransmit path was
delivered by S-BL.ARQ-TX (merged PR #98, `internal/arqsend`). S-7.04-FU-SIGHUP-RELOAD
delivered the SIGHUP reload path (merged; HEAD `bacc59b`), including the transitional
testenv seam (`SetSighupCh`/`SendReloadSignal`) whose retirement is an explicit obligation
of this story (FO-2 from that DELIVERY).

This story also carries the **live-egress anchor** obligation. E-FWD-001 (rate-limit,
receive-side split-horizon log) and the S-4.04 LOW findings (3 LOW + SEC-001) have been
re-anchored here since the arqsend integration tests could not trigger them (send-side
tests, no path exhaustion). The PE dial loop under sustained retransmit is the first
vehicle that exercises the full send+forward path with actual upstream connections.

## Design Constraints

### Dial-Loop Architecture and Package Placement (Q2, Q4)

**Binding (per placement note Q2 and Q4).**

The dial loop lives in a new `internal/upstreamdial` package as a `Connector` type.
`runRouter` constructs `Connector` at startup and passes the initial address list.
Reload passes a new address list via a channel. This follows the `internal/drain`
encapsulation precedent: `drainCoord` extracted drain coordination out of `runRouter`
for the same testability reason; `Connector` does the same for the dial loop.

**Package placement:** `internal/upstreamdial` at DAG position 19 (effectful, imports
`{frame, outerassembler}`). Current positions 19–22 renumber to 20–23 in the same commit.

```
| 19 | internal/upstreamdial | {frame, outerassembler} | effectful (network I/O) | Wave 7 (S-7.04-FU-PE-CONNECTOR) |
```

**ARCH-08 §6.4 registration is required before the first commit.** The story implementer
must update ARCH-08 §6.5 with the new row AND renumber positions 19–22 to 20–23.

**Forbidden edges:**

- `internal/upstreamdial` → `internal/drain` (drain is effectful-goroutine lifecycle; connector does not need it)
- `internal/upstreamdial` → `internal/routing` (routing is a boundary; connector does not participate in frame forwarding)
- `internal/upstreamdial` → `internal/testenv` (test-helper composition root; never imported by production code)

**Handle interface (proposed):**

```go
// In internal/upstreamdial:
type Handle interface {
    // ReloadAddrs reconciles the running set to the new address list.
    // Set-equal semantics (see Concurrency Contract section). Called from runRouter's select loop.
    ReloadAddrs(addrs []string)
    // Mode returns ModeE when no upstreams are connected, ModePE when >= 1.
    Mode() ConnMode
    // Stop cancels all dial goroutines and waits for them to exit.
    Stop()
}
```

Note on mode enum: `internal/upstreamdial` MUST define its own `ConnMode` type
(`ModeE`/`ModePE`) since `internal/testenv` is at DAG position 22/23 (test-helper
composition root — not importable by production code). `testenv` maps from
`upstreamdial.ConnMode` to `testenv.RouterMode` in the wiring glue.

### Concurrency Contract: Channel-Passed Snapshot (Q3, FO-4)

**Binding (per placement note Q3).**

`upstreamRouters` in the `runRouter` goroutine remains a local variable (consistent with
`mgmt_wire.go:483`). The `Connector` receives address updates via a **buffered
`chan []string`** (capacity 1). `runRouter`'s SIGHUP case, after computing the new
address list, sends a `[]string` snapshot on the channel using a non-blocking send
(pattern: `select { case ch <- v; default: _ = <-ch; ch <- v }`). The `Connector`'s
reconcile goroutine reads from this channel.

The `Connector` owns all reconnect state internally: the per-address dial state, the
per-address reconnect backoff timer, and the connected-count `atomic.Int32` that drives
`Mode()`. All state is accessed only from the `Connector`'s reconcile/reconnect
goroutine(s). The only cross-goroutine boundaries are:

- address update channel (`runRouter` → `Connector`, writes only)
- `Mode()` read (any goroutine — safe as `atomic.Load`, no mutex needed per go.md rule 12)

`runRouter` never reads the address list from the `Connector`; it maintains its own
local copy for emission purposes (BC-2.09.001 PC-1 mode line).

This design is consistent with the `sighupCh <-chan os.Signal` precedent from
S-7.04-FU-SIGHUP-RELOAD: signals flow in one direction through channels, state is not
shared.

### Session Bootstrap: "Connection Established" Definition (Q6)

**Binding (per placement note Q6 verbatim).**

BC-2.09.001 PC-2 ("router establishes connections to the configured upstream routers")
is satisfied when ALL THREE of the following succeed:

1. `net.Dial("tcp", addr)` returns `(conn, nil)`
2. `outerassembler.Assemble(env, cf)` returns `(wire, nil)` where:
   - `env.SrcAddr` — this router's node address (derived from its ephemeral Ed25519
     public key and SVTN ID per `frame.DeriveNodeAddress`)
   - `env.DstAddr` — the upstream router's expected node address (from config or
     derived at dial time)
   - `env.SVTNID` — the SVTN this PE router is serving
   - `env.FrameAuthKey` — derived via `hmac.DeriveKey` using the shared session key
   - `cf` — minimal `halfchannel.ChannelFrame` with `FrameTypePEConnect` payload
     (new constant to be defined, e.g., `frame.FrameTypePEConnect`)
3. `conn.Write(wire)` returns `(n, nil)` with `n == len(wire)`

A TCP connection that is not yet bootstrapped is NOT a "connection established" event for
BC purposes. The `connected-count atomic.Int32` increments only on step 3 success.

**Admission protocol:** The full `admission.GenerateChallenge`/`admission.AdmitNode`
challenge-response handshake is DEFERRED to a follow-on story if not feasible within
the 8-point scope. The story-writer assesses feasibility at implementation time. If
deferred, the dial bootstrap sends an unauthenticated PE-CONNECT frame and a
`Non-Goals` note records the deferral; if in-scope, the full `internal/admission` flow
is wired at dial time. Either way, the three-step definition above applies.

## Acceptance Criteria

### AC-001 — PE graduation: outbound dial, session bootstrap, and set-equal reconciliation

**BC Anchors:** BC-2.09.001 PC-2/PC-3 (upstream connections established; router becomes
PE); BC-2.09.001 EC-002 (reorder MUST NOT trigger redial); per placement note Q1 and Q6.

**Precondition:** `runRouter` is executing with a PE config (non-empty
`upstreamRoutersFor(cfg)`). A `Connector` has been constructed with the initial address
list. A cooperative upstream router fixture (testenv loopback) is accepting TCP
connections at each configured address.

**Postconditions:**

1. For each address in `upstreamRoutersFor(cfg)`, the Connector dials a TCP connection:
   `net.Dial("tcp", addr)` returns `(conn, nil)`.
2. The Connector calls `outerassembler.Assemble(env, cf)` with the bootstrap envelope
   (Q6 definition: `env.SrcAddr` = this router's node address, `env.DstAddr` = upstream
   address, `env.SVTNID` = this router's SVTN, `cf` carries `FrameTypePEConnect`).
3. `conn.Write(wire)` returns `(n, nil)` with `n == len(wire)`. All three of (1), (2),
   (3) succeed before the connected-count increments by 1. A TCP connection that has not
   completed the bootstrap write is NOT a "connection established" event for BC purposes
   (per placement note Q6 verbatim definition).
4. `connector.Mode()` returns `ModePE` when connected count is ≥ 1.
5. On SIGHUP reload that changes the upstream address set to a different ordering of the
   same addresses (set-equal content): no existing connections are torn down and no new
   dials are initiated. The reconciler uses `upstreamRoutersAsSet(addrs []string) map[string]struct{}`
   in `router_config.go` to compute `added = loaded \ running` and `removed = running \
   loaded` as set differences (per placement note Q1 ruling: "a reorder MUST NOT
   trigger redial or teardown").
6. On SIGHUP reload that adds new addresses: Connector dials the added addresses.
   On reload that removes addresses: Connector tears down the removed connections.
7. `runRouter` has not returned; the daemon continues executing the run loop throughout.

**Set-equal vs. order-sensitive note (per placement note Q1):** The emission diff in
`runRouter`'s SIGHUP case (`equalStringSlices` at `router_config.go:81`) keeps
order-sensitive semantics unchanged — a spurious re-emission on reorder is acceptable
noise. The connection reconciliation diff MUST use set-equal semantics. The two
comparators may lawfully differ; the spec does not require emission and connection
triggers to share semantics.

**Test names:**

- `TestConnector_DialSuccess_ModePE` (unit, `internal/upstreamdial/connector_test.go`)
- `TestRunRouter_PE_DialAndConnect_UpstreamReachable` (integration, `cmd/switchboard/router_pe_connector_test.go`)
- `TestConnector_ReorderReuse_NoTeardown` (unit, `internal/upstreamdial/connector_test.go`)
- `TestRunRouter_PE_SetEqualReconciliation_NoTeardownOnReorder` (integration, `cmd/switchboard/router_pe_connector_test.go`)

**Test level:** unit (Connector) + integration (runRouter end-to-end)
**Test files:** `internal/upstreamdial/connector_test.go`, `cmd/switchboard/router_pe_connector_test.go`

Outline (integration):

### AC-002 — Unreachable upstream: EC-001 graceful handling with exponential backoff

**BC Anchors:** BC-2.09.001 EC-001 (partial PE mode; log verbatim; retry in background);
BC-2.09.001 EC-004 (all upstreams lost → ModeE fallback); per placement note Q5 (exact
backoff parameters).

**Precondition:** The Connector is started with at least one upstream address that is
not accepting TCP connections (port closed or unreachable). Optionally: one address is
reachable (to exercise partial-PE).

**Postconditions (per placement note Q5 — exact parameters are normative):**

1. The dial attempt fails with a net error. The Connector emits exactly:
   ```
   upstream router <addr> unreachable
   ```
   to `w` (the `io.Writer` passed to `runRouter`) on every failed dial attempt. The
   `addr` is the string from `upstreamRoutersFor`. The implementation MAY append
   `: <net-error>` for operator diagnostics; that suffix is NOT asserted by ACs.
   (Log contract verbatim from BC-2.09.001 EC-001.)
2. The connected-count atomic remains at 0 for the unreachable address. `runRouter`
   continues its run loop without returning; existing node sessions are unaffected.
3. The Connector retries in background with exponential backoff:
   - **Base:** 500ms
   - **Cap:** 30s
   - **Jitter:** ±25% uniform (breaks synchronization across multiple PE routers)
   - **Reset on success:** backoff resets to base on first successful dial
4. If at least one other upstream IS connected (`connected count ≥ 1`), `connector.Mode()`
   returns `ModePE` (partial-PE semantics: router is in PE mode once ≥1 upstream is
   connected). No additional `mode=partial-PE` mode line is emitted; the BC does not
   define that observable.
5. If ALL upstreams are unreachable and the connected count drops from ≥1 to zero (the
   final upstream connection is lost), the Connector emits to `w`:
   ```
   mode=E (no upstream_routers configured)
   ```
   (EC-004 fallback observable from BC-2.09.001 EC-004.) `connector.Mode()` returns
   `ModeE`.

**Test names:**

- `TestConnector_DialFailure_EC001Log` (unit, `internal/upstreamdial/connector_test.go`)
- `TestConnector_BackoffParameters` (unit, `internal/upstreamdial/connector_test.go` — asserts base=500ms; cap reached after sufficient failures)
- `TestConnector_AllUpstreamsUnreachable_ModeE` (unit, `internal/upstreamdial/connector_test.go`)
- `TestRunRouter_PE_UnreachableUpstream_PartialPE` (integration, `cmd/switchboard/router_pe_connector_test.go` — one reachable + one unreachable upstream; Mode()=ModePE)

**Test level:** unit (Connector backoff + log) + integration (partial-PE end-to-end)
**Test files:** `internal/upstreamdial/connector_test.go`, `cmd/switchboard/router_pe_connector_test.go`

Outline (unit — EC-001 log):

## Non-Goals

- Does not implement DRAIN message broadcast to connected nodes. That is `S-7.04-FU-DRAIN-WIRE`.
- Does not implement SIGHUP signal handling. That is `S-7.04-FU-SIGHUP-RELOAD` (MERGED).
- Does not implement RESYNC control-frame protocol for session re-establishment after
  migration. That is `S-BL.RESYNC-FRAME`. The Connector delivers the raw TCP `net.Conn`
  as an upstream channel; RESYNC is a higher-level session protocol that uses that channel.
- Does not register `drainCoord` observers on upstream connections. The `drainCoordHook`
  test seam in `router_config.go:76` remains unchanged. Drain observer registration
  belongs to `S-7.04-FU-DRAIN-WIRE`.
- Does not implement the full `internal/admission` challenge-response handshake at dial
  time unless the implementer assesses it as feasible within the 8-point scope (see
  Session Bootstrap design constraint above).
