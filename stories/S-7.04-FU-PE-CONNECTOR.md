# S-7.04-FU-PE-CONNECTOR: Outbound TCP Dial Loop on PE Graduation

> **STATUS: BACKLOG STUB.** This story is the S-7.04 PE outbound connector residual
> and the live-egress anchor story for four accumulated drift observations. Acceptance
> criteria, file structure, and task list will be fleshed out when the story is scheduled.

## Narrative

- **As an** operator who has graduated a router from E to PE mode (via SIGHUP reload)
- **I want** the router to actually dial and maintain TCP connections to the configured
  upstream routers
- **So that** node sessions on this PE router have access to alternate path options
  (multi-hop routing) per BC-2.09.001 PC-2/PC-3

## Context

S-7.04 delivered the *emission seam* for PE graduation: `upstreamRoutersFor(cfg)` and
`mode=PE upstream_routers=[...]` startup emission. The `keepaliveIntervalFor` helper is
in place (normative-fence test `TestKeepaliveIntervalNotSweepDeadline` guards it against
conflation with `sweepDeadline`). What is explicitly deferred is the TCP dial loop that
consumes `upstreamRoutersFor(cfg)` and establishes actual upstream router connections.

The outer-header wire-frame primitive required for session bootstrap was delivered by
S-BL.OA (merged PR #96, `internal/outerassembler`). The ARQ retransmit path was
delivered by S-BL.ARQ-TX (merged PR #98, `internal/arqsend`).

This story also carries the **live-egress anchor** obligation. E-FWD-001 (rate-limit,
receive-side split-horizon log) and the S-4.04 LOW findings (3 LOW + SEC-001) have been
re-anchored here since the arqsend integration tests could not trigger them (send-side
tests, no path exhaustion). The PE dial loop under sustained retransmit is the first
vehicle that exercises the full send+forward path with actual upstream connections.

## Sketched Acceptance Criteria

> ACs are illustrative. Exact scope, test names, and BC postcondition references will
> be confirmed at scheduling time.

**AC-001 (BC-2.09.001 PC-2/PC-3):** On PE graduation (SIGHUP with non-empty
`upstream_routers` config), `runRouter` initiates an outbound TCP dial to each address
in `upstreamRoutersFor(cfg)`. Successful connection transitions the router to operational
PE mode with that upstream active.

**AC-002 (BC-2.09.001 EC-001):** Unreachable upstream router address is handled
gracefully: router logs `upstream router <addr> unreachable`, runs in partial PE mode,
and retries in background with exponential backoff. Existing node sessions are unaffected.

**AC-003 (BC-2.09.003 PC-8 keepalive application):** The `keepaliveIntervalFor(cfg)` value
drives the per-upstream-connection keepalive/reconnect ticker. The ticker must not be
wired to `sweepDeadline` (normative fence already in place via `TestKeepaliveIntervalNotSweepDeadline`).

**AC-004 (E-FWD-001 re-confirmation):** Integration test exercises the PE send path
under sustained ARQ retransmit load, confirming that E-FWD-001 (rate-limit log) fires
when the forward table is exhausted, and that it does not fire spuriously under normal
multi-path load.

**AC-005 (VP-037 / VP-038 activation):** With live upstream connections, the VP-037
drain-within-window test and VP-038 E→PE-via-config-only test can be fully discharged
(not just at seam level). Activation of both VPs' verification_lock is part of this
story's Definition of Done.

## Non-Goals

- Does not implement DRAIN message broadcast to nodes. That is `S-7.04-FU-DRAIN-WIRE`.
- Does not implement SIGHUP signal handling. That is `S-7.04-FU-SIGHUP-RELOAD`.
- Does not implement RESYNC control-frame protocol. That is `S-BL.RESYNC-FRAME`.
