# S-3.03: Tier-2 Per-Session Authorization and Read-Only Enforcement

> **Execute:** `/vsdd-factory:deliver-story S-3.03`

## Narrative

- **As an** access node
- **I want to** enforce per-session Tier-2 authorization at attach time and reject upstream keystrokes from read-only consoles
- **So that** session access is controlled at the granularity of individual sessions, and the router is never consulted for per-session auth decisions

## Acceptance Criteria

### AC-001 (traces to BC-2.05.003 PC-1)
`SessionAuth.Authorize(consoleKey, sessionName)` succeeds when the console's public key is in the session's authorization list (with any valid role). Returns nil error and the console's role (full-access or read-only).
- **Test:** `TestSessionAuth_Authorize_RegisteredKey_Succeeds`

### AC-002 (traces to BC-2.05.003 PC-2)
`SessionAuth.Authorize(consoleKey, sessionName)` returns `E-ADM-006` ("session authorization denied") when the console's key is NOT in the session's authorization list.
- **Test:** `TestSessionAuth_Authorize_UnregisteredKey_ErrAdmSix`

### AC-003 (traces to BC-2.05.003 PC-3 + invariant 1)
The router (`internal/routing`) has no per-session authorization data structure. Tier-2 enforcement is exclusively in `internal/session`. VP-012 code-audit verifies this: no import of per-session auth state in `routing.go`.
- **Test:** `TestSessionAuth_RouterHasNoTier2State` (code-audit assertion via `go vet` / `grep`)

### AC-004 (traces to BC-2.05.003 PC-4)
Authorization is per-session: a console authorized for `session-A` is NOT automatically authorized for `session-B`. `SessionAuth.Authorize` checks the named session's list specifically.
- **Test:** `TestSessionAuth_Authorize_PerSession_NoSpillover`

### AC-005 (traces to BC-2.04.005 PC-1 + PC-3)
A read-only console can attach and receive all downstream frames. Any upstream keystroke frame (payload-bearing) from a read-only console is rejected by the access node with `E-ADM-007` ("upstream rejected: read-only access"). The rejection does NOT terminate the console's downstream subscription.
- **Test:** `TestReadOnlyConsole_UpstreamRejected_DownstreamContinues`

### AC-006 (traces to BC-2.04.005 PC-3 + EC-004)
Empty-tick frames (liveness probes, zero-length payload) from a read-only console are accepted by the access node; only payload-bearing upstream frames are rejected.
- **Test:** `TestReadOnlyConsole_EmptyTickAccepted`

| EC-006 | BC-2.04.005 EC-004 | Read-only console sends empty-tick frame | Accepted (liveness probe); not treated as a keystroke |
