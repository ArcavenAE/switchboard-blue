# S-7.04-FU-DRAIN-WIRE: DRAIN-over-SVTN Wire Propagation

## Narrative

- **As an** operator gracefully shutting down a PE router
- **I want** connected nodes to receive a DRAIN signal over their SVTN channels before
  the router disconnects
- **So that** nodes can migrate active sessions to alternate routers within the drain
  window, rather than losing them abruptly

## Context

S-7.04 (merged PR #101/#103) delivered `internal/drain.Drain` with `RegisterObserver`,
`Signal`, `Wait`, and the `drainCoordHook` test-injection mechanism. The coordinator's
liveness model (fan-out-then-race under a bounded child context, EC-003 ErrTimeout) is
complete and tested. However, `runRouter` currently has zero registered observers: no
per-node connection identity model exists at the DRAIN observer call site.

S-BL.PE-RECEIVE-LOOP (merged PR #118 @ `e940fc2`, 2026-07-11) delivered the per-connection
receive goroutine, `upstreamdial.Connector.SetFrameCallback`, and the
`FrameArrivalHandler.OnFrameArrival` wiring. It also emitted FO-RECV-FWD-001: the nil
`ForwardFunc` at `mgmt_wire.go`:549 must be replaced when the interface set widens.

**v1.1 clarification (Q2-AMENDED):** FO-RECV-FWD-001 is NOT consumed by this story. The
nil `ForwardFunc` stays; the `interfaceSet` stays `[]{peIfaceID}`. DRAIN is a
router-originated broadcast dispatched directly from the drain observer closure to per-node
send channels ā€” it does NOT route through `OnFrameArrival`. The E-FWD-001 invariant is
preserved. FO-RECV-FWD-001 remains deferred to the future data-frame-widening story.

This story lands three deliverables:

1. **DRAIN frame on the wire** ā€” A single startup drain observer (registered once at
   `runRouter` setup) iterates the live per-node send map at `Signal` time. For each
   connected node, it assembles a `FrameTypeCtl` (0x03) frame with a 4-byte payload
   (`control_type = 0x01`, DRAIN opcode) and non-blocking sends it to the node's send
   channel. The per-node send map is a `sync.Map` owned by `runRouter`; entries are
   added via the `netingress.ServeConfig.OnAccept` hook and removed on conn close.

2. **netingress OnAccept seam** ā€” `netingress.Serve` gains a `ServeConfig` parameter
   carrying an `OnAccept func(conn net.Conn, h NodeHandle) func()` hook. `runRouter`
   supplies the hook to allocate `routing.InterfaceID`, create the per-node send channel,
   start the writer goroutine, and register the channel in the send map. Cleanup
   (channel close + map remove) is deferred on conn close. This is the binding Q-SEAM
   design.

3. **Per-observer panic recovery and EC-003 production-path evidence** ā€” `drain.Signal`'s
   fan-out goroutine currently has no `defer recover()`. This story adds the recovery
   wrapper. EC-003 is evidenced via `drainCoordHook` + blocked observer +
   `cfg.DrainTimeout=100ms` (the real drain-timeout injection seam per Q-AC005).

## Design Constraints

### DRAIN Frame Wire Schema (Q1 ā€” binding)

**Binding (per placement note Q1).**

The DRAIN signal is carried as a `FrameTypeCtl = 0x03` frame (existing constant, verified
`internal/frame/frame.go` at `e940fc2`). No new `FrameType` constant is introduced. The
DRAIN payload schema is a fixed 4-byte control message (BC-2.01.008 v1.0, schema home):

```
byte[0]   control_type   = 0x01   // DRAIN opcode
byte[1]   version        = 0x01   // protocol version; allows future extension
byte[2:4] reserved       = 0x0000 // zero-filled; receiver MUST ignore
```

Rationale: `FrameTypeCtl` is the existing discriminator for authenticated control frames.
Adding `FrameTypeDrain = 0x07` would require widening `Valid()` past `FrameTypePEConnect =
0x06` ā€” the guard S-BL.PE-RECEIVE-LOOP erected. Using `FrameTypeCtl` with a
`control_type` payload byte is the correct design: outer header identifies the wire
family (`ctl`); the control-message opcode lives in the payload. BC-2.01.008 is the
authoritative schema home; BC-2.01.004 Inv-2 carries the terminal-consumer carve-out
cross-reference.

**Assembly path:** The observer closure constructs the 4-byte payload and calls
`frame.EncodeOuterHeader` + `append(ehdr[:], payload...)`. The `Envelope` fields
(SVTN, SrcAddr, DstAddr, FrameAuthKey) are sourced from the per-node connection context
stored in the `runRouter` connection registry using the zero-envelope pattern identical to
S-7.04-FU-PE-CONNECTOR (FO-DRAIN-WIRE-002 notes that full Ed25519 bootstrap is a
session-bootstrap follow-on). `frame.EncodeOuterHeader` is an EXISTING function at
`e940fc2` (not new ā€” defined by this story).

**Wire-format spec-pair obligation (same-commit ā€” binding):** This story introduces the
`control_type` discriminator byte in the `FrameTypeCtl` payload schema. The following
same-commit amendments are required:

1. ARCH-02 Ā§3.1 (frame_type table): add a note that `ctl (0x03)` payloads carry a
   `control_type` byte discriminator; enumerate opcode `0x01 = DRAIN`.
2. BC-2.01.004 Postcondition 2 (outer-header layout): add the same `control_type` note
   under the `frame_type = ctl` row (with cross-ref to BC-2.01.008 as schema home).

These are a NARROWER obligation than the FrameTypePEConnect case (F-SP14-001) ā€” the
frame_type byte itself is unchanged (0x03 already listed); only the payload schema
documentation is new. The same-commit rule (F-P8-008) still applies.

**Forward-compatibility silent-ignore rule (FO-DRAIN-WIRE-001, binding; anchors to
BC-2.01.008 Postcondition 4):** Any `FrameTypeCtl` frame arriving at the PE receive
loop's `FrameFn` closure with an unrecognized `control_type` value MUST be silently
ignored (passed to `OnFrameArrival` with no ctl-specific dispatch). The DRAIN receive
path MUST NOT fail on unknown opcodes. `control_type = 0x02` (RESYNC) is reserved and
dispatched by `S-BL.RESYNC-FRAME` ā€” NOT this story. EC-003 in BC-2.01.008 specifies
that RESYNC received before S-BL.RESYNC-FRAME ships is silently ignored.

### Send-Path Seam and Per-Node Send Map (Q-SEAM + Q2-AMENDED ā€” binding)

**Binding (per placement note Q-SEAM and Q2-AMENDED, F-DW-SP1-001 and F-DW-SP1-002).**

**FO-RECV-FWD-001 disposition:** FO-RECV-FWD-001 is NOT consumed by this story. The
`SetFrameCallback` closure in `mgmt_wire.go` MUST NOT change the `interfaceSet` or
`ForwardFunc`. The nil `ForwardFunc` stays nil (correct ā€” E-FWD-001 invariant: split-horizon
exhausts before `fn` is reached with the single-element set `[]{peIfaceID}`). FO-RECV-FWD-001
remains deferred to the future data-frame-widening story.

**DRAIN dispatch path:** DRAIN is a router-originated broadcast, NOT a frame arriving
via `OnFrameArrival`. The drain observer closure in `runRouter` dispatches directly to
each node's send channel. The routing layer is not in the DRAIN path.

**netingress seam (Q-SEAM, binding):**

`netingress.Serve` gains an `OnAccept` hook via a `ServeConfig` struct parameter:

```go
// In internal/netingress:

type NodeHandle struct {
    IfaceID routing.InterfaceID
    Send    chan<- []byte  // caller writes assembled wire frames; closed by caller
}

type ServeConfig struct {
    // OnAccept is called for each accepted connection before ServeConn starts.
    // The returned func() (cleanup) is called in a defer inside the per-conn
    // goroutine when ServeConn returns. If nil, no hook fires.
    OnAccept func(conn net.Conn, h NodeHandle) func()
}

// Serve extended to: func Serve(ctx, ln, route, logger, cfg ServeConfig) error
// Backward compat: callers passing a zero ServeConfig are unaffected.
```

**ARCH-08 impact:** `internal/netingress` imports `internal/routing` (for
`routing.InterfaceID`) ā€” pos 18 ā†’ pos 17, a forward edge (18 > 17); LAWFUL under
ARCH-08 Ā§6.5. ARCH-08 Ā§6.5 row for `internal/netingress` MUST be amended in the same
commit: current `{frame}` ā†’ updated `{frame, routing}`. This is a same-story Ā§6.5
import-set update, not an ARCH-08 version bump.

**Per-node send map ownership:**

- `ifaceID` allocated by `runRouter` via a monotonic counter (`atomic.AddUint64`),
  starting at 2 (`peIfaceID = 1` is reserved for the PE upstream interface).
- `runRouter` `OnAccept` closure:
  1. Creates `send := make(chan []byte, 32)` (buffered 32 to decouple drain observer
     from backpressure).
  2. Starts a writer goroutine (sole writer to `conn`):
     ```go
     go func() {
         for frame := range send {
             if _, err := conn.Write(frame); err != nil {
                 return
             }
         }
     }()
     ```
  3. Inserts `(ifaceID, send)` into the per-node `sync.Map`.
  4. Returns cleanup `func()` that closes `send` + removes entry from the map.
- `ServeConn` is the sole reader of `conn`. No mutex is required (net.TCPConn read/write
  are independent paths).

### Single Startup Observer (Q-SINGLE-OBS ā€” binding)

**Binding (per placement note Q-SINGLE-OBS, F-DW-SP1-007).**

`runRouter` registers a SINGLE `drain.Observer` with `drainCoord`, ONCE at setup before
any connections arrive. The observer captures a reference to the per-node send map.

At `Signal` time, the single observer iterates the live map:
```go
sendMap.Range(func(key, value any) bool {
    ch := value.(chan []byte)
    frame := assembleDrainFrame(...)
    select {
    case ch <- frame:
    default: // best-effort per BC-2.09.002 v1.2
    }
    return true
})
```

The observer then returns (context deadline is the bound ā€” Q3.P1 BINDING option (2)).

**Consequence for OnAccept:** The `OnAccept` closure inserts the per-node send channel
into the live map and removes it on cleanup, but does NOT call
`drainCoord.RegisterObserver`. Only the single startup observer is registered.

**Consequence for `drain.Drain` API:** No deregister API is needed. `RegisterObserver`
retains its current shape.
