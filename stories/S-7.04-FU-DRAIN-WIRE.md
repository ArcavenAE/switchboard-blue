# S-7.04-FU-DRAIN-WIRE: DRAIN-over-SVTN Wire Propagation

> **STATUS: BACKLOG STUB.** This story is the S-7.04 AC-003 + AC-004 residual. Acceptance
> criteria, file structure, and task list will be fleshed out when the story is scheduled.

## Narrative

- **As a** node in a PE network
- **I want to** receive a DRAIN signal from the router before it disconnects
- **So that** I can migrate my active sessions to an alternate router within the drain
  window, rather than losing them abruptly

## Context

S-7.04 introduced `internal/drain.Drain` with `RegisterObserver`, `Signal`, and `Wait`.
The coordinator's liveness model (fan-out-then-race under a bounded child context) is
complete and tested. However, `runRouter` has no per-node connection model — `netingress`
accepts frames but does not track per-node connection state as a first-class entity.
The drain coordinator currently has no registered observers and shuts down via the
empty-observer fast path.

This story lands the production observer registration path. It requires:
1. Per-node connection identity in runRouter (as a prerequisite from S-BL.NI or its successor)
2. Observer functions that send a DRAIN message over the node's SVTN channel and ACK when
   the node migration completes or the per-node timeout elapses
3. Per-observer `defer recover()` for fault isolation (named silent-failure risk in S-7.04 DELIVERY)
4. Integration test for forced-exit with an unresponsive registered observer via the
   production registration path (not the drainCoordHook indirection)

## Sketched Acceptance Criteria

> ACs are illustrative. Exact scope, test names, and BC postcondition references will
> be confirmed at scheduling time.

**AC-001 (BC-2.09.002 PC-1):** When `drainCoord.Signal` fires, each per-node observer
sends a DRAIN message over the node's SVTN channel. The observer ACKs (returns nil from
the observer function) when the node acknowledges migration or the per-node ACK timeout
elapses (BC-2.09.002 layered: the per-node ACK budget is the inner bound; the drain
coordinator's total window is the outer bound).

**AC-002 (BC-2.09.002 EC-003 residual):** An integration test drives an unresponsive
registered observer — one registered via the production `drainCoord.RegisterObserver` path
(not the drainCoordHook injection) — that does not ACK within the drain window. `runRouter`
returns nil after the drain_timeout window, emitting the EC-003 marker log line. This is
the forced-exit evidence for the production registration path.

**AC-003 (per-observer panic recovery):** Each observer function is wrapped in
`defer recover()`. A panicking observer logs the panic and marks the ACK as failed
(returning a non-nil error) rather than killing the process. `runRouter` proceeds
with disconnect via the EC-003 path.

**AC-004 (VP-037):** Integration test asserts that when nodes have alternate paths
available and drain is signalled, sessions are preserved on the alternate paths after
drain completes (drain within window — the coordinator Wait returns nil, not ErrTimeout).

## Dependency Chain

- **Requires from S-BL.NI or successor:** Per-node connection identity in runRouter
  (a connection object per admitted node, not just netingress frame routing).
- **RegisterObserver seam exists:** `drainCoord.RegisterObserver` is in place from S-7.04.
  This story plugs production observers in; no coordinator refactor needed.
