# S-7.04-FU-DRAIN-WIRE: DRAIN-over-SVTN Wire Propagation

## Narrative

- **As an** operator gracefully shutting down a PE router
- **I want** connected nodes to receive a DRAIN signal over their SVTN channels before
  the router disconnects
- **So that** nodes can migrate active sessions to alternate routers within the drain
  window, rather than losing them abruptly

## Context

S-7.04 (merged PR #101/#103) delivered `internal/drain.Drain` with `RegisterObserver`,
`Signal`, `Wait`, and the `drainCoordHook` test-injection mechanism. The coordinator's
liveness model (fan-out-then-race under a bounded child context, EC-003 ErrTimeout) is
complete and tested. However, `runRouter` currently has zero registered observers: no
per-node connection identity model exists at the DRAIN observer call site, and the nil
`ForwardFunc` in `mgmt_wire.go` (FO-RECV-FWD-001) has never been wired.

S-BL.PE-RECEIVE-LOOP (merged PR #118 @ `e940fc2`, 2026-07-11) delivered the per-connection
receive goroutine, `upstreamdial.Connector.SetFrameCallback`, and the
`FrameArrivalHandler.OnFrameArrival` wiring. It also emitted FO-RECV-FWD-001: the nil
`ForwardFunc` at `mgmt_wire.go`:549 must be replaced when the interface set widens.
This story IS that widening event.

This story lands three deferred deliverables:

1. **DRAIN frame on the wire** — When `drainCoord.Signal` fires, each connected node
   receives a `FrameTypeCtl` (0x03) frame with a 4-byte payload (`control_type = 0x01`,
   DRAIN opcode) over its SVTN channel. The frame is assembled via `frame.EncodeOuterHeader`
   + append in the drain observer closure running inside `cmd/switchboard/runRouter`. The
   per-node send channels live in a `sync.Map[routing.InterfaceID, chan []byte]` owned
   by `runRouter`.

2. **FO-RECV-FWD-001 discharge** — The nil `ForwardFunc` at `mgmt_wire.go`:549 is replaced
   with a live per-node dispatch closure that reads from the per-node send map.

3. **Per-observer panic recovery and EC-003 production-path evidence** — `drain.Signal`'s
   fan-out goroutine currently has no `defer recover()`. This is a named code gap from the
   S-7.04 DELIVERY doc. This story adds the recovery wrapper. The EC-003 forced-exit path
   (unresponsive observer via the PRODUCTION `RegisterObserver` path, not `drainCoordHook`)
   is also evidenced here.

## Design Constraints

### DRAIN Frame Wire Schema (Q1 — binding)

**Binding (per placement note Q1).**

The DRAIN signal is carried as a `FrameTypeCtl = 0x03` frame (existing constant, verified
`internal/frame/frame.go` at `e940fc2`). No new `FrameType` constant is introduced. The
DRAIN payload schema is a fixed 4-byte control message:

```
byte[0]   control_type   = 0x01   // DRAIN opcode
byte[1]   version        = 0x01   // protocol version; allows future extension
byte[2:4] reserved       = 0x0000 // zero-filled; receiver MUST ignore
```

Rationale: `FrameTypeCtl` is the existing discriminator for authenticated control frames.
Adding `FrameTypeDrain = 0x07` would require widening `Valid()` past `FrameTypePEConnect =
0x06` — the guard S-BL.PE-RECEIVE-LOOP erected. Using `FrameTypeCtl` with a
`control_type` payload byte is the correct design: outer header identifies the wire
family (`ctl`); the control-message opcode lives in the payload.

**Assembly path:** The observer closure constructs the 4-byte payload and calls
`frame.EncodeOuterHeader` + `append(ehdr[:], payload...)`. The `Envelope` fields
(SVTN, SrcAddr, DstAddr, FrameAuthKey) are sourced from the per-node connection context
stored in the `runRouter` connection registry using the zero-envelope pattern identical to
S-7.04-FU-PE-CONNECTOR (FO-DRAIN-WIRE-002 notes that full Ed25519 bootstrap is a
session-bootstrap follow-on). `frame.EncodeOuterHeader` is an EXISTING function at
`e940fc2` (not new — defined by this story).

**Wire-format spec-pair obligation (same-commit — binding):** This story introduces the
`control_type` discriminator byte in the `FrameTypeCtl` payload schema. The following
same-commit amendments are required:

1. ARCH-02 §3.1 (frame_type table): add a note that `ctl (0x03)` payloads carry a
   `control_type` byte discriminator; enumerate opcode `0x01 = DRAIN`.
2. BC-2.01.004 Postcondition 2 (outer-header layout): add the same `control_type` note
   under the `frame_type = ctl` row.

These are a NARROWER obligation than the FrameTypePEConnect case (F-SP14-001) — the
frame_type byte itself is unchanged (0x03 already listed); only the payload schema
documentation is new. The same-commit rule (F-P8-008) still applies.

**Forward-compatibility silent-ignore rule (FO-DRAIN-WIRE-001, binding):** Any
`FrameTypeCtl` frame arriving at the PE receive loop's `FrameFn` closure with an
unrecognized `control_type` value MUST be silently ignored (passed to `OnFrameArrival`
with no ctl-specific dispatch). The DRAIN receive path MUST NOT fail on unknown opcodes.
`control_type = 0x02` (RESYNC) is reserved and dispatched by `S-BL.RESYNC-FRAME` — NOT
this story. This is a binding rule regardless of whether an explicit wire ACK is
implemented (Q3.P1 PROVISIONAL ruling below does not affect this rule).

### Per-Node Send Map and FO-RECV-FWD-001 Discharge (Q2 — binding)

**Binding (per placement note Q2).**

The per-node send map is a `sync.Map` keyed by `routing.InterfaceID`, storing
`chan []byte` per connected node. The map is owned by `runRouter` in
`cmd/switchboard/mgmt_wire.go` as a local variable. No new internal package is required.

**FO-RECV-FWD-001 discharge contract (binding):**

The `SetFrameCallback` closure at `mgmt_wire.go`:549 currently reads (verified `e940fc2`):

```go
connector.SetFrameCallback(func(hdr frame.OuterHeader, raw []byte) error {
    return arrivalHandler.OnFrameArrival(raw, peIfaceID, []routing.InterfaceID{peIfaceID}, nil)
})
```

After this story, the nil `ForwardFunc` is replaced with a live dispatch closure that
reads from the per-node send map. The `interfaceSet` parameter widens to include all
currently-registered node `InterfaceID`s. The `ForwardFunc` closure iterates the send
map and writes `frameBytes` to the per-node channels for interfaces other than the
arrival interface.

**Send path shape (binding — option (a) from placement note Q2):**

Each admitted node connection registers a `chan []byte` send channel in the per-node
send map under a newly allocated `routing.InterfaceID` (monotonic counter). The drain
observer closure captures a reference to the send map and the node's `InterfaceID`, not
a direct channel snapshot — ensuring a disconnected node's channel is not retained after
the node leaves.

The `sync.Map` is safe for concurrent access from the drain observer goroutine (reading
the map for drain broadcast) and the routing dispatch goroutine (reading the map for
`ForwardFunc` calls) simultaneously.

### Observer Registration and Inbound Acknowledgment Path (Q3)

**Binding on registration path (per placement note Q3).**

When a node connection is accepted by `netingress.ServeConn` (pos 18), `runRouter`
allocates a new `routing.InterfaceID` (atomic monotonic counter), creates the per-node
send channel `chan []byte`, inserts it into the per-node send map, and calls
`drainCoord.RegisterObserver(fn)` where `fn` is a `drain.Observer` closure that:

1. Assembles the 4-byte DRAIN payload and outer header via `frame.EncodeOuterHeader` +
   `append` (per Q1 assembly path above).
2. Looks up the node's send channel from the per-node send map by `InterfaceID`.
3. Sends the assembled wire frame on the channel (non-blocking with per-node context
   derived from `drainCtx` — the context deadline is the drain coordinator's total
   window bound).
4. Returns when done (see PROVISIONAL ruling below for ACK details).

**ARCH-08 lawfulness of new import edges (binding):**

| New edge | From pos | To pos | Verdict |
|----------|----------|--------|---------|
| `cmd/switchboard` calls `drain.RegisterObserver` | pos 18 | pos 16 | LAWFUL — 18 > 16; already imported |
| `cmd/switchboard` calls `frame.EncodeOuterHeader` in observer closure | pos 18 | pos 2 | LAWFUL — 18 > 2; already imported |
| `cmd/switchboard` reads per-node send map in `ForwardFunc` closure | internal | same package | LAWFUL — same package, no import edge |
| `internal/drain` (pos 16) → no new imports | — | — | drain remains stdlib-only; ARCH-08 §6.5 pos-16 row preserved |

No new import edges between internal packages. All path crossings remain within
`cmd/switchboard` (pos 18, the top leaf).

**PROVISIONAL ruling — ACK mechanism (Q3.P1):**

> **PROVISIONAL — pending spec-adversarial cycle convergence.** The exact acknowledgment
> mechanism for the drain observer is not yet fully specified. Three shapes are viable:
> (1) Node sends `FrameTypeCtl, control_type = 0x02 (DRAIN_ACK)` back on the PE
> connection; (2) No explicit wire ACK — observer returns immediately after send, context
> deadline is the only bound; (3) Node closes the PE connection as implicit ACK.
>
> **RECOMMENDED: option (2)** — observer sends the DRAIN frame then blocks on context
> deadline or channel-close signal. The drain coordinator's timeout (EC-003 ErrTimeout)
> is the safety net. This is the simplest shape and avoids a new opcode dispatch path.
>
> This story implements option (2) as the default. The spec-adversarial cycle may
> mandate a different shape. Do NOT harden option (2) to binding in downstream artifacts
> until this PROVISIONAL is resolved.

**Per-observer panic recovery — production CODE change (binding):**

The `drain.Signal` method fan-out goroutine at `e940fc2` lines 126–129 reads:

```go
go func() {
    defer obsWG.Done()
    fn(drainCtx)
}()
```

This story adds `defer recover()` around `fn(drainCtx)`:

```go
go func() {
    defer obsWG.Done()
    defer func() {
        if r := recover(); r != nil {
            // log the panic; drain coordinator continues
        }
    }()
    fn(drainCtx)
}()
```

This is a **production change to `internal/drain/drain.go`** (pos 16, stdlib-only). No
new import edge is introduced. The `defer recover()` uses only `recover()` and stdlib
logging primitives.

A unit test in `internal/drain/drain_test.go` pins the panic recovery behavior:
the panicking observer MUST NOT kill the process; the drain coordinator continues to
call remaining observers and eventually calls `Wait`.

- **ARCH-08 §6.5 pos-16 drain:** `internal/drain` MUST NOT gain any new import edges to
  internal packages. The `defer recover()` wrapper uses only stdlib primitives.
  `drain.go`'s package doc notes a potential `internal/routing` import (verified at
  `e940fc2`:11-13); this story does NOT instantiate that import. The ARCH-08
  `drain --> routing` Mermaid edge remains planned-but-not-realized.

  all new import edges from `cmd/switchboard` are unconditionally lawful. No ARCH-08
  §6.4 registration required.
- **`sync.Map` ownership:** The per-node send map is LOCAL to `runRouter` in
  `cmd/switchboard/mgmt_wire.go`. It MUST NOT be exported or passed to `internal/drain`
  — the drain observer closure captures a reference at registration time, and `drain.go`
  remains stdlib-only.

  MUST be amended in the same commit that adds the `control_type` discriminator to
  `mgmt_wire.go`. Failing to update the spec pair leaves the wire-format specification
  inconsistent with the implementation.
- **FO-DRAIN-WIRE-001 forward-compat:** Any new `FrameTypeCtl` dispatch branch in the
  `FrameFn` closure MUST have a silent-ignore default for unrecognized `control_type`
  values. A `switch control_type` with no `default: /* ignore */` clause is a spec
  violation.
- **go.md rule compliance:** New test files MUST pass `go test -race -count=1 ./...`
  and `golangci-lint run` without suppressions.
