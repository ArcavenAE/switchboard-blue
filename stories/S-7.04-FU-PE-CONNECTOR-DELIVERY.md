# S-7.04-FU-PE-CONNECTOR — DELIVERY

## What Landed

This story closes the outbound TCP dial loop deferred by S-7.04. It delivers the
`internal/upstreamdial` package and its `Connector` type, wires that connector into
`runRouter` at construction time, retires the transitional testenv seams left by
S-7.04-FU-SIGHUP-RELOAD, and discharges VP-038.

Four composed changes:

1. **`internal/upstreamdial/connector.go` (NEW)** — `Connector` type with `Handle`
   interface (`ReloadAddrs([]string)`, `Mode() ConnMode`, `Stop()`); `ConnMode` enum
   (`ModeE` / `ModePE`); `dialLoop` goroutine per address performing TCP dial +
   `outerassembler.Assemble` bootstrap write + connected-count increment on step-3
   success; reconnect backoff with schedule base = `keepaliveInterval` floored at
   `BackoffBase`=500ms (floor constant only), cap 30s, jitter ±25%; `BackoffBase` is
   a floor constant, NOT the operative schedule base (F-P2-002 ruling); `atomic.Int32`
   connected-count driving `Mode()`; set-equal reconciliation inline in
   `Connector.reconcile` (no separate helper — F-P1-007); `operativeBase` pure function
   (F-P2-002); `stopOnce sync.Once` for idempotent `Stop()` (F-P2-001); fast-path
   send + non-blocking drain + non-blocking resend for `ReloadAddrs` (F-P5-001 —
   blocking inner-receive deadlock fix); EC-004 drop-to-zero emission guarded with
   `ctx.Err() == nil` (F-P4-001 graceful-Stop polarity); single-atomic transition
   ownership via `newCount := Add(-1)` for exactly-one EC-004 per ≥1→0 event
   (F-P29-001 concurrent-drop race).

2. **`cmd/switchboard/mgmt_wire.go`** — `runRouter` constructs `upstreamdial.New` at
   startup with the initial address list, `keepaliveInterval`, and buffered
   `chan []string`; the SIGHUP case calls `connector.ReloadAddrs(newUpstreams)` instead
   of direct local-variable management for connection state; `connector.Stop()` in
   cleanup path; `peConnectorHook` stub removed (OBS-1 / F-P1-007); `#DEFERRED`
   comment block split — PE-CONNECTOR line moved to `#SHIPPED`, DRAIN-WIRE deferral
   preserved (F-P14-001). Zero-Envelope deferral documented in a comment (see
   Declared Divergences).

3. **`internal/testenv/testenv.go`** — Retires `SetSighupCh` and 1-arg
   `SendReloadSignal` (both methods removed, `sighupCh` field gone, `os`/`syscall`
   imports removed); adds `SetConnector(upstreamdial.Handle)` construction-time seam;
   `RouterHandle.Mode()` now delegates to the live `connector.Mode()` when connector
   non-nil (maps `upstreamdial.ConnMode` → `testenv.RouterMode`); `Restart()` always
   tears down any existing connector via `Stop()` and recreates via `upstreamdial.New`
   for non-empty `UpstreamRouters` (never calls `ReloadAddrs` — that belongs to the
   production SIGHUP seam in `runRouter`); `newEnv` allocates a dynamic PE fixture
   listener instead of stub `"127.0.0.1:9999"`.

## Declared Divergences

The following four intentional divergences from the story outline are declared per the
declared-divergence protocol. Each carries a class and rationale.

**Divergence 1 — AC-004 partial-discharge (class: unmet-deps — PO adjudication F-P1-002)**

The E-FWD-001 re-confirmation anchor (S404-OBS-F) and S404-LOW-1 were both re-anchored
from this story to `S-BL.PE-RECEIVE-LOOP`. Postcondition 2 ("no spurious E-FWD-001
under normal load") is DISCHARGED here by `TestRunRouter_PE_EFWD001ReconfirmationUnderLoad`.
Postcondition 1 ("E-FWD-001 fires under path exhaustion") and S404-OBS-F / S404-LOW-1
remain unmet because `routing.FrameArrivalHandler.OnFrameArrival` is never called from
`runRouter`'s ingress path in this story — the `Connector` only dials, bootstraps, and
keepalives; it has no receive/forward loop. The story that ships the PE-connection
receive/forward loop (`S-BL.PE-RECEIVE-LOOP`) owns the exhaustion discharge.
S-BL.PE-RECEIVE-LOOP is the direct prerequisite for S-7.04-FU-DRAIN-WIRE.

**Divergence 2 — Zero-Envelope Q6 deferral (class: not-core)**

The `Connector` is constructed with zero-valued `outerassembler.Envelope` fields
(`SrcAddr`, `DstAddr`, `SVTNID`, `FrameAuthKey`). Full node-identity derivation (Ed25519
key material, `frame.DeriveNodeAddress`, HMAC key derivation) is deferred to the
session-bootstrap follow-on story. The three-step "connection established" definition
(TCP dial + `Assemble` + `Write` all nil-returning) is satisfied with the zero envelope.
The deferral is documented in a comment in `mgmt_wire.go`. The full `internal/admission`
challenge-response handshake is likewise deferred (not-core within 8-point scope).

**Divergence 3 — FrameTypeData bootstrap placeholder (class: not-core — FO-PE-LOOP-001)**

The bootstrap `ChannelFrame.FrameType` uses `halfchannel.FrameTypeData` as a placeholder.
The distinct PE-CONNECT frame type (`frame.FrameTypePEConnect`, as cited in Q6 of the
placement note) is a phantom symbol (0 grep hits in the repo) deferred to
`S-BL.PE-RECEIVE-LOOP`, the consumer story whose receive loop must distinguish bootstrap
frames from session data. Using `FrameTypeData` makes bootstrap frames
indistinguishable from session data at the receiver until the distinct constant is
defined and adopted. The code comment in `internal/upstreamdial/connector.go` (dialLoop
bootstrap construction) documents this symmetrically alongside the Envelope deferral.
Forward obligation FO-PE-LOOP-001 is registered in `S-BL.PE-RECEIVE-LOOP`.

**Divergence 4 — VP-037 partial-discharge (class: unmet-deps)**

VP-037 ("drain within window") `verification_lock` remains `false`. The
`TestE2E_RouterDrain_NodesMigrateWithin2s` harness is present in
`router_pe_connector_test.go` but skipped with a partial-discharge note. The full
two-router drain-and-migrate scenario requires the DRAIN broadcast wire protocol owned
by `S-7.04-FU-DRAIN-WIRE`. This story delivers the live-egress infrastructure (the PE
dial loop and live upstream connections) that makes VP-037 achievable — it is a
necessary precondition, not the complete proof. VP-037 `verification_lock` will flip
`true` when `S-7.04-FU-DRAIN-WIRE` ships. Note: VP-038 `verification_lock` flipped to
`true` at code HEAD `6b6f0cf`; the pending merge-SHA anchor true-up (VP-038.md merge
SHA field update + any aspirational-skeleton `t-first` signature drift) is a PR-time
obligation (see PR-Time Obligations below).
