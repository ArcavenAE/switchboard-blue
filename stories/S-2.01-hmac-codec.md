# S-2.01: Implement HMAC-SHA256 Frame Authentication in internal/hmac

> **Execute:** `/vsdd-factory:deliver-story S-2.01`

## Narrative

- **As a** router
- **I want to** authenticate every incoming frame using HMAC-SHA256 with a per-(node, SVTN) HKDF-derived key
- **So that** only frames from legitimately admitted nodes are forwarded — unauthenticated or forged frames are dropped at the first router boundary

## Acceptance Criteria

### AC-001 (traces to BC-2.05.005 postcondition 1)
`ComputeHMAC(key, frame_bytes)` produces an 8-byte (64-bit) truncated HMAC-SHA256 tag.
- **Test:** `TestComputeHMAC_EightByteTag`

### AC-002 (traces to BC-2.05.005 postcondition 2)
`VerifyHMAC(key, frame_bytes, tag)` returns true when tag matches the expected HMAC-SHA256 truncated to 8 bytes.
- **Test:** `TestVerifyHMAC_ValidTag`

### AC-003 (traces to BC-2.05.005 postcondition 3)
`VerifyHMAC` returns false for any single-bit flip in the frame payload.
- **Test:** `FuzzVerifyHMAC_SingleBitFlip`

### AC-004 (traces to BC-2.05.005 precondition 1)
`VerifyHMAC` returns false when a different key is used to verify a tag computed with the correct key.
- **Test:** `TestVerifyHMAC_WrongKey`

### AC-005 (traces to BC-2.05.005 postcondition 4)
`DeriveKey(node_admission_pubkey, svtn_id)` uses HKDF-SHA256 with `node_admission_pubkey` as IKM to produce a per-(node, SVTN) key. Same inputs always produce same output (deterministic).
- **Test:** `TestDeriveKey_Deterministic`
