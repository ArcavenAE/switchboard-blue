# S-3.01: Tmux Control Mode Integration with PTY Fallback

> **Execute:** `/vsdd-factory:deliver-story S-3.01`

## Narrative

- **As an** access node
- **I want to** connect to the local tmux server via control mode and publish sessions over the SVTN
- **So that** console operators can attach to named terminal sessions without knowing the access node's hostname

## Acceptance Criteria

### AC-001 (traces to BC-2.04.001 postcondition 1)
`TmuxControlMode.Attach(session_name)` connects to the local tmux server via `-C` (control mode), subscribes to output events, and returns a bidirectional stream over the SVTN half-channel.
- **Test:** `TestTmuxControlMode_Attach_Success`

### AC-002 (traces to BC-2.04.001 postcondition 2)
Output event completeness: at least 99% of tmux output events received within the tmux control mode session are delivered to the downstream stream (integration test with real tmux session).
- **Test:** `TestTmuxControlMode_OutputCompleteness_99pct`

### AC-003 (traces to BC-2.04.002 postcondition 1)
When tmux control mode connection fails (tmux not installed, socket absent, or permission denied), `TmuxControlMode.Attach` falls back to PTY proxy mode automatically.
- **Test:** `TestTmuxControlMode_FallbackToPTY_OnFailure`

### AC-004 (traces to BC-2.04.002 postcondition 2)
PTY proxy mode produces functionally equivalent output to tmux control mode for basic keystroke-to-echo behavior.
- **Test:** `TestPTYProxy_BasicKeystrokeEcho`

## Architecture Mapping

| Component | Module | Pure/Effectful |
|-----------|--------|---------------|
| TmuxControlMode | internal/tmux | effectful |
| PTYProxy | internal/tmux | effectful |

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | tmux session exits while control mode is active | Stream is closed cleanly; console receives EOF |
| EC-002 | tmux control mode socket becomes unresponsive mid-session | Fall-through does NOT automatically trigger PTY fallback (fallback is only at initial connect) |
| EC-003 | PTY window size not set before first keystroke | Default 80x24 used; resize events handled when they arrive |
