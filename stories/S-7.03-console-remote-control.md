# S-7.03: Console Remote Control via sbctl

> **Execute:** `/vsdd-factory:deliver-story S-7.03`

## Narrative

- **As an** operator
- **I want to** remotely control a console daemon via sbctl (attach, detach, switch session, navigate)
- **So that** I can manage console sessions from a management station without interactive console access

## Acceptance Criteria

### AC-001 (traces to BC-2.08.001 postcondition 1)
`sbctl console attach --target <console_addr> --session <name>` causes the console daemon to attach to the named session.
- **Test:** `TestSbctlConsole_Attach`

### AC-002 (traces to BC-2.08.001 postcondition 2)
`sbctl console detach --target <console_addr>` causes the console daemon to detach from the current session without closing it.
- **Test:** `TestSbctlConsole_Detach`

### AC-003 (traces to BC-2.08.001 postcondition 3)
`sbctl console switch --target <console_addr> --session <name>` detaches from the current session and attaches to the named session in one atomic operation.
- **Test:** `TestSbctlConsole_Switch`

### AC-004 (traces to BC-2.06.001 postcondition 5 — console session-list surfacing; DRIFT-001b)
The console daemon's session-list view includes a `quality` field for each active session showing the current green/yellow/red quality indicator value. The quality value is derived from `QualityIndicator` state (from `internal/metrics`) and is visible when the operator lists sessions via the console UI. This is the "console session-list view" clause of BC-2.06.001 PC-5 — the complementary sbctl half is owned by S-5.02 AC-007.
- **Test:** `TestConsoleDaemon_SessionListIncludesQuality`

### AC-005 (traces to BC-2.06.002 postcondition 3 — missCount observability; DRIFT-002)
The console daemon exposes the `missCount` gap-event counter to the operator via `sbctl sessions status` output and the console session-list view. The counter is sourced from `QualityIndicator.MissCount()` (or equivalent accessor on `internal/metrics`). This closes the DRIFT-002 deferred observability obligation.
- **Test:** `TestConsoleDaemon_SessionListIncludesMissCount`
