# S-7.03: Console Remote Control via sbctl

> **Execute:** `/vsdd-factory:deliver-story S-7.03`

## Scope Note (RULING-W6TB-C)

**Transport:** mgmt-plane Unix-socket (S-6.03 pattern, ADR-006/ADR-012).
BC-2.08.001 Inv-3 has been patched to v1.4 (see
`.factory/decisions/RULING-W6TB-C-console-transport.md`): the "same SVTN channel
as regular traffic" wording was architecturally incompatible with the established
JSON-over-Unix-socket management-plane transport used by all sbctl commands.
Inv-3 v1.4 now correctly states the management-plane transport per BC-2.07.004 EC-013
(Unix socket for access mode; TCP loopback for console mode).

**Reduced scope:** AC-004 (console daemon session-list view) and AC-005
(MissCount accessor) have been removed from this story per RULING-W6TB-C and
assigned to follow-on backlog story **S-BL.CONSOLE-OBS**.

## Narrative

- **As an** operator
- **I want to** remotely control a console daemon via sbctl (attach, detach, switch session)
- **So that** I can manage console sessions from a management station without interactive console access

## Acceptance Criteria

### AC-001 (traces to BC-2.08.001 postcondition 1)
`sbctl console attach --target <console_addr> --session <name>` sends a JSON
attach-request over the management-plane transport per BC-2.07.004 EC-013
(Unix socket for access mode; TCP loopback for console mode) — per Inv-3 as
revised in BC-2.08.001 v1.4 (ADR-006/ADR-012 pattern) to the
console daemon's `internal/session` RPC handler, causing it to attach to the
named session. On unknown-session the response carries `E-SES-001`; on
auth-denied the response carries `E-ADM-006` in E-RPC-011 envelope.
- **Test:** `TestSbctlConsole_Attach`

### AC-002 (traces to BC-2.08.001 postcondition 2)
`sbctl console detach --target <console_addr>` sends a JSON detach-request over
the management-plane transport per BC-2.07.004 EC-013 (Unix socket for access
mode; TCP loopback for console mode) — per Inv-3 as revised in BC-2.08.001 v1.4,
causing the console daemon to detach from the current session without closing it.
On not-attached the response carries `E-SES-004`.
- **Test:** `TestSbctlConsole_Detach`

### AC-003 (traces to BC-2.08.001 postcondition 3)
`sbctl console switch --target <console_addr> --session <name>` sends a JSON
switch-request over the management-plane transport per BC-2.07.004 EC-013
(Unix socket for access mode; TCP loopback for console mode) — per Inv-3 as
revised in BC-2.08.001 v1.4, detaching from the current session and attaching
to the named session in one atomic operation. On unknown-session the response
carries `E-SES-001`; on not-attached the response carries `E-SES-004`;
on auth-denied the response carries `E-ADM-006` in E-RPC-011 envelope.
- **Test:** `TestSbctlConsole_Switch`

## Error Envelope Reference (Implementer)

| Error | Scenario | Wire code |
|-------|----------|-----------|
| `E-SES-001` | Unknown session (attach/switch) | Returned by console daemon |
| `E-SES-004` | Not attached for command (detach/switch) | Returned by console daemon |
| `E-ADM-006` | Auth denied (any operation) | E-RPC-011 wrapping E-ADM-006 |
