# S-7.04: E-to-PE Router Graduation and Graceful Drain

> **Execute:** `/vsdd-factory:deliver-story S-7.04`

## Narrative

- **As a** network operator
- **I want to** graduate an E-mode router to PE mode by adding upstream router connections to its config, and drain it before shutdown
- **So that** PE topology can be incrementally adopted without service interruption and routers can be taken down without dropping sessions

## Acceptance Criteria

### AC-001 (traces to BC-2.09.001 postcondition 1)
Adding upstream router connection entries to the router config and reloading causes the E-mode router to establish upstream connections and enter PE mode. No binary change required.
- **Test:** `TestRouterGraduation_EtoPE_ConfigOnly`

### AC-002 (traces to BC-2.09.001 precondition 1)
PE graduation is validated by `Config.Validate()` — missing required upstream fields produce `E-CFG-001` with an actionable message.
- **Test:** `TestRouterGraduation_InvalidConfig_RejectsGraduation`

### AC-003 (traces to BC-2.09.002 postcondition 1)
`Drain.Signal()` broadcasts a drain message to all connected nodes, causing them to migrate to alternate routers within 2 seconds.
- **Test:** `TestDrain_NodesMigrateWithin2s`

### AC-004 (traces to BC-2.09.002 postcondition 2)
After drain completes (all nodes migrated or timeout), the router closes all connections cleanly and exits.
- **Test:** `TestDrain_RouterExitsCleanlyAfterDrain`

### AC-005 (traces to BC-2.09.003 PC-7 / DEFERRED-APPLICATION)
`cfg.DrainTimeout` is wired into the `internal/drain` graceful-drain coordinator as the deadline for graceful shutdown and PE graduation. When `cfg.DrainTimeout` is zero (absent / use-daemon-default), the drain coordinator uses the ARCH-06 documented default of 10s. When non-zero and positive, the drain coordinator uses the configured value. The hardcoded drain-timeout constant MUST NOT be used when a config file is supplied with a non-zero `drain_timeout`. Cross-reference: BC-2.09.003 PC-7 DEFERRED-APPLICATION (S-7.04 obligation L-5).
- **Test:** `TestDrain_DrainTimeoutFromConfig`

### AC-006 (traces to BC-2.09.003 upstream application DEFERRED-APPLICATION)
`cfg.UpstreamRouters` (slice of `{addr string}`) is wired into the PE-mode upstream connector / peer-list initialization on PE graduation. The upstream address list is sourced from `cfg.UpstreamRouters`, replacing any hardcoded upstream address list. Cross-reference: BC-2.09.003 PC-9 DEFERRED-APPLICATION (S-7.04 obligation L-5, upstream_routers field).
- **Test:** `TestRouterGraduation_UpstreamRoutersFromConfig`

### AC-007 (traces to BC-2.09.003 PC-8 / DEFERRED-APPLICATION)
`cfg.KeepaliveInterval` is wired into the node-reconnect keepalive ticker — the mechanism described by FM-009 ("after `keepalive_interval`, default 1s, nodes attempt reconnect"). When `cfg.KeepaliveInterval` is zero (absent / use-daemon-default), the keepalive ticker uses the ARCH-06 documented default of 1s. When non-zero and positive, the configured value is used. This MUST NOT wire to `sweepDeadline` (the 60s console eviction window — semantically unrelated to node reconnect keepalive). Cross-reference: BC-2.09.003 PC-8 DEFERRED-APPLICATION (S-7.04 obligation L-5, keepalive_interval field).
- **Test:** `TestKeepalive_IntervalFromConfig`
