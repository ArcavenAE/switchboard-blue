# S-1.02: Implement Timeslice Clock State Machine in internal/halfchannel

> **Execute:** `/vsdd-factory:deliver-story S-1.02`

## Narrative

- **As a** session endpoint (access node or console)
- **I want to** drive a timeslice clock that fires at regular intervals regardless of keystroke availability
- **So that** liveness is maintained and the downstream display never goes silent even during idle periods

## Acceptance Criteria

### AC-001 (traces to BC-2.01.001 postcondition 1)
`HalfChannel.Tick()` produces exactly one frame per call, regardless of whether payload is available.
- **Test:** `TestHalfChannelTick_OneFramePerCall`

### AC-002 (traces to BC-2.01.002 postcondition 1)
When no payload is queued, `Tick()` produces an empty-tick frame with `frame_type=data` and zero-length payload. The frame is structurally valid (passes `ParseOuterHeader`).
- **Test:** `TestHalfChannelTick_EmptyFrameIsValid`

### AC-003 (traces to BC-2.01.003 postcondition 1)
Upstream and downstream `HalfChannel` instances maintain independent sequence counters. Ticking channel A does not advance channel B's sequence number.
- **Test:** `TestHalfChannelIndependentSequences`

### AC-004 (traces to BC-2.01.003 postcondition 2)
Each successive call to `Tick()` on the same half-channel increments the sequence number by exactly 1.
- **Test:** `TestHalfChannelSequenceIncrement`

### AC-005 (traces to BC-2.01.001 invariant 1)
Tick timing: p99 jitter of the tick interval must be ≤ 2ms (measured over 1000 ticks in test). This is the benchmark gate for VP-041.
- **Test:** `BenchmarkHalfChannelTickJitter`

### AC-006 (traces to BC-2.01.002 invariant 1)
K consecutive ticks with no payload produce K frames with contiguous sequence numbers (no gaps, no duplicates).
- **Test:** `TestHalfChannelEmptyTickSequence`

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | Tick called before any payload enqueued | Empty-tick frame emitted; no panic |
| EC-002 | Sequence number wraps around uint32 max | Sequence wraps to 0; no overflow panic |
| EC-003 | Multiple payloads queued before one tick | Single tick emits one frame; remaining payloads queued for subsequent ticks |

## Purity Classification

| Module | Classification | Justification |
|--------|---------------|---------------|
| internal/halfchannel | pure-core | State machine only; no goroutines, no timers. Tick is called externally by the effectful layer. |
