# S-3.01a: Tmux Control Mode Integration

> **Execute:** `/vsdd-factory:deliver-story S-3.01a`

## Narrative

- **As an** access node
- **I want to** connect to the local tmux server via control mode and publish sessions over the SVTN
- **So that** console operators can attach to named terminal sessions without knowing the access node's hostname

## Acceptance Criteria

### AC-001 (traces to BC-2.04.001 PC-1)
When the access node starts and tmux is available, `TmuxControlMode.Connect()` establishes an active control mode connection to the local tmux server (`tmux -C`).
- **Test:** `TestTmuxControlMode_Connect_EstablishesConnection`

### AC-002 (traces to BC-2.04.001 PC-2)
After `TmuxControlMode.Connect()` succeeds, all current tmux sessions are enumerated and published; the returned session list matches what `tmux ls` reports.
- **Test:** `TestTmuxControlMode_Connect_EnumeratesSessions`

### AC-003 (traces to BC-2.04.001 PC-3 + PC-4)
New tmux sessions created after startup are automatically discovered and published; closed sessions are automatically unpublished. Both events are processed via control mode event subscription (`%session-created` / `%session-closed`).
- **Test:** `TestTmuxControlMode_SessionLifecycleEvents`

### AC-004 (traces to BC-2.04.001 PC-5)
Session output events from control mode (`%output`) feed the downstream half-channel; at least 99% of events emitted by the tmux control mode session are delivered to the downstream stream (integration test with a real tmux session).
- **Test:** `TestTmuxControlMode_OutputEventsFeedDownstream`

## Edge Cases

| ID | Source | Description | Expected Behavior |
|----|--------|-------------|-------------------|
| EC-001 | BC-2.04.001 EC-001 / EC-004 | tmux not found in PATH | PTY fallback triggered (handled by S-3.01b); log: "tmux not found; using PTY proxy" |
| EC-002 | BC-2.04.001 EC-002 | Control mode connection drops mid-session | Reconnect attempted; if reconnect fails, PTY proxy fallback triggered (handled by S-3.01b) |
| EC-003 | BC-2.04.001 EC-003 | tmux server has no sessions on startup | Connect succeeds; session list empty; no sessions published until sessions are created |
