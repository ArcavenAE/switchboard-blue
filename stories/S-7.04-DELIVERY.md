# S-7.04 — E-to-PE Router Graduation and Graceful Drain (DELIVERY)

## What Landed

The story as written pre-#92 (netingress) and pre-#94 (routing DAG
freeze) assumed a runRouter with a live node-connection concept and
an inbound signal-loop that already accepted SIGHUP. Neither is true
in the current daemon. This delivery takes the largest honest subset
that today's runRouter + netingress reality permits, closes the three
BC-2.09.003 DEFERRED-APPLICATION obligations at their application
points, and lands the drain coordinator infrastructure that
BC-2.09.002 requires — leaving the observable seam that a follow-on
story can plug the wire protocol into without daemon-level refactor.

Three composed constructs:

1. **`internal/drain`** (NEW package, ARCH-08 §6 DAG position 16) —
   Single-use fan-out-then-race coordinator implementing BC-2.09.002.
   `drain.New(timeout)` (zero or negative → `DefaultTimeout = 10s`);
   `RegisterObserver(fn)` under a mutex before Signal; `Signal(ctx)`
   fires observers concurrently under a bounded child context and
   closes a done channel — idempotent (second Signal is a no-op);
   `Wait(ctx)` races observer ACK against the drain window, returning
   `ErrTimeout` on window elapsed (BC-2.09.002 EC-003 — caller
   proceeds with disconnect anyway) or the caller's ctx error on
   caller-side cancel. Nil observers are silently ignored. Zero
   external dependencies beyond stdlib.

2. **`cmd/switchboard/router_config.go`** — Three application-point
   helpers, one per BC-2.09.003 DEFERRED-APPLICATION field, each
   mirroring the shape of `tickIntervalFor` in access.go:
   - `drainTimeoutFor(cfg)` — returns `cfg.DrainTimeout` when positive
     or `drain.DefaultTimeout` otherwise. Negative values are
     impossible at this seam (Config.Validate rejects them via
     E-CFG-006).
   - `keepaliveIntervalFor(cfg)` — returns `cfg.KeepaliveInterval` when
     positive or `defaultKeepaliveInterval` (1s) otherwise. The
     `defaultKeepaliveInterval` constant is declared with an inline
     fence docstring citing BC-2.09.003 PC-8: it is DISTINCT from
     `sweepDeadline` (60s console eviction). Negative values impossible
     via E-CFG-007.
   - `upstreamRoutersFor(cfg)` — returns a fresh `[]string` of
     `UpstreamRouter.Addr` values (empty slice / nil semantically
     equivalent — E mode). The **fresh-slice contract** is load-bearing:
     mutating the returned slice does not affect subsequent calls or
     cfg. This isolates the runRouter call site from a future
     runtime-reload path that mutates cfg.

3. **`cmd/switchboard/mgmt_wire.go`** (edit) — `runRouter` now:
   - Resolves `drainWindow`, `keepaliveInterval`, `upstreamRouters` via
     the three helpers before establishing listeners.
   - Constructs `drainCoord = drain.New(drainWindow)` alongside the
     mgmt listener + net-ingress goroutine.
   - Emits four startup lines to the writer: `drain_timeout=%s`,
     `keepalive_interval=%s`, and either
     `mode=E (no upstream_routers configured)` or
     `mode=PE upstream_routers=[...]` — order-preserving, per
     BC-2.09.001 PC-1 (mode signal) and BC-2.09.003 PC-9 (upstream
     application seam). These are the operator-visible confirmation
     that config flowed through to the application points.
   - On `<-ctx.Done()`: calls `drainCoord.Signal(drainCtx)` +
     `drainCoord.Wait(drainCtx)` bounded by `drainCoord.Timeout()`
     via a fresh `context.WithTimeout(context.Background(), ...)` (the
     parent ctx is already cancelled). On `ErrTimeout` emits
     `runRouter: drain: <err> (proceeding with disconnect per
     BC-2.09.002 EC-003)`; either way proceeds to `ingressCancel()`
     and `dataWG.Wait()`.
   - Feeds `drainCoord.Timeout()` into `mgmt.Shutdown` — replacing the
     hardcoded 5s. One operator lever now governs both the drain
     window and the mgmt-listener graceful-close budget.

## Liveness Design + Spec Citation

The drain coordinator's liveness model is fan-out-then-race under a
bounded child context (BC-2.09.002 EC-003):

- `Signal(ctx)` fans observers out concurrently as goroutines under
  a `context.WithTimeout(context.Background(), timeout)` child. The
  parent ctx (caller-provided) governs Signal's own cancellation for
  observer teardown; the child ctx bounds observer runtime to the
  configured drain window.
- `Wait(ctx)` selects between `<-obsAckCh` (all observers returned) and
  `<-drainCtx.Done()` (window elapsed) and `<-ctx.Done()` (caller
  cancelled). Returns nil, ErrTimeout, or ctx.Err respectively.
- The fresh child context is INTENTIONALLY decoupled from the parent
  ctx that will typically already be `Done()` at the time Signal is
  invoked in `runRouter` (Signal is called AFTER `<-ctx.Done()`
  fires). Without the fresh context, the drain window would collapse
  to zero on the first shutdown signal — nodes would get no chance
  to migrate.

## Blast Radius

**1. Operator-visible surfaces touched:**
- **Router startup log** — four new lines emitted to stderr (writer):
  `drain_timeout=%s`, `keepalive_interval=%s`, and either
  `mode=E (no upstream_routers configured)` or
  `mode=PE upstream_routers=%v`. Additive — existing lines
  (`data plane listening on`, `management socket at`) unchanged.
- **Router shutdown budget** — was hardcoded 5s for `mgmt.Shutdown`;
  now derived from `cfg.DrainTimeout` (default 10s). Operators
  running with default config get a longer graceful window;
  operators who want the old 5s set `drain_timeout = "5s"`.
  Documented in the DEFERRED-APPLICATION seam.
- **Log line on drain-window timeout** — new
  `runRouter: drain: <err> (proceeding with disconnect per BC-2.09.002
  EC-003)` message on the EC-003 path. Fires only when observers
  exist AND fail to ACK within window — currently unreachable (no
  observers registered) but load-bearing when the wire protocol
  lands.

**2. Silent-failure risk:**
- `drain.Drain` deliberately does NOT wrap observer bodies in
  `defer recover()` — a panicking observer would kill the process.
  Acceptable for this story because no observers are registered yet;
  when the DRAIN-over-SVTN wire protocol lands, the follow-on must
  add per-observer recovery if per-node fault isolation is required.
  Named explicitly here so the follow-on story can't ship without
  addressing it.
- `drainCoord.Signal` is called *after* `<-ctx.Done()` fires and uses
  a fresh `context.WithTimeout(context.Background(), ...)` — this is
  intentional (drain must survive the parent ctx cancel) but means a
  stuck drain now delays shutdown up to `drain_timeout` — previously
  bounded at 5s. The `EC-003` log line is the visible signal; there
  is no metric yet.
- `upstreamRoutersFor` returns a nil slice for zero upstreams (not
  empty non-nil). Callers must treat nil and empty identically; the
  emission branch uses `len(...) == 0` for exactly this reason.
- `keepaliveIntervalFor` emission does not yet drive a reconnect
  ticker (no ticker exists — that's the deferred half). A future
  contributor could conflate "value is emitted" with "value is
  applied" — mitigated by the fresh-slice contract test + the
  BC-2.09.003 PC-8 fence test failing loudly if
  `defaultKeepaliveInterval` ever equals `sweepDeadline`.

**3. Smoke gate touched:**
- No T2-4 / T3-4-taxonomy surface change: the only new `runRouter`
  error path (nil cfg) was already covered by
  `TestRunRouter_NilConfigCleanError` pre-existing. Startup output
  additions do not affect taxonomy classification.
- `go test -race ./...` passes; `just lint` passes with zero warnings.
  `just verify` (codesign) not run — not a release artifact for this
  PR.
- Existing integration tests (`TestRunRouter_StartsWithMgmt`,
  `TestRunRouter_DataListenerBinds`, `TestRunRouter_NoAdminHandlers`,
  `TestRunRouter_SIGTERMLifecycle`, `TestRunRouter_NilConfigCleanError`)
