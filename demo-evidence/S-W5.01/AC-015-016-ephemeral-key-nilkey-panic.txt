## AC-015: Ephemeral Keypair Generation in runAccess

Tests: TestRunAccess_GeneratesEphemeralKey_AC015, TestRunAccess_MgmtStartFailureAborts_RulingJ

Sub-case (real_key_serves_wellformed_challenge):
- runAccess generates ephemeral ed25519.PrivateKey via ed25519.GenerateKey(rand.Reader).
- startMgmtServer called with a non-nil, 64-byte daemonPrivKey
  (len(daemonPrivKey) == ed25519.PrivateKeySize).
- Server issues a well-formed CHALLENGE on the first client connection.
- Key is ephemeral: generated every process start, never written to disk.

Sub-case (startMgmtServer_with_nil_key_serves_no_challenge):
- NewServer construction-time guard (VP-068) fires when nil key is passed.
- startMgmtServer wraps the panic in a recover() and returns an error.
- runAccess receives the error and returns it immediately (Ruling J).
- Data plane NOT started.

Fatal-policy (Ruling J — TestRunAccess_MgmtStartFailureAborts_RulingJ):
- startMgmtServer returning any error (config-class or bind failure) → runAccess returns error.
- Neither buildAccessComponents nor runAccessWithConnector is called after the failure.
- Verified for both an explicit socket path and the default path (both unbindable in the test).

## AC-016: nil/Short-Key Construction Guard (VP-068 / Invariant 8)

Tests: TestNewServer_PanicsOnNilKey_VP068, TestNewServer_PanicsOnShortKey_VP068

Sub-cases:
- nil daemonKey (len==0): NewServer panics immediately at construction.
  Prevents nil-pointer dereference mid-connection (remote-panic DoS vector).
- len_32_public_key_size (32 bytes): common mistake of passing public key for private key → panic.
- len_63_one_short (63 bytes): one byte under ed25519.PrivateKeySize → panic.
- len_1_single_byte (1 byte): single byte → panic.

Guard code (mgmt.go):
    if len(daemonKey) != ed25519.PrivateKeySize {
        panic("mgmt.NewServer: daemonKey must be ed25519.PrivateKeySize bytes (ADR-012 §Invariant 8 / AC-016)")
    }
