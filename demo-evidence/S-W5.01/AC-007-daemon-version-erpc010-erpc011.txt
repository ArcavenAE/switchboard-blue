## AC-007: daemon_version Injection

Tests: TestMgmtServer_DaemonVersion_Injected_AC007, TestMgmtServer_AuthOK_DispatchesRPC_AC007

Sub-case (auth_ok_carries_injected_version):
- NewServer constructed with daemonVersion="0.1.0-test" as fifth parameter.
- Client authenticates → AUTH_OK received.
- AUTH_OK.daemon_version == "0.1.0-test" (not hardcoded "dev").
- Confirms daemonVersion comes from NewServer parameter (ldflags-injected in production).

Sub-case (empty_daemonVersion_panics):
- NewServer called with daemonVersion="" → panic recovered by test.
- Confirms: construction-time fail-fast guard prevents empty version sentinel at runtime.

## AC-007: RPC Happy Path

Test: TestMgmtServer_AuthOK_DispatchesRPC_AC007

Observable behavior:
- Client authenticates (authorized key + correct signature) → AUTH_OK received.
- AUTH_OK carries daemon_version field (non-empty).
- Client sends {"type":"request","id":"r1","command":"test.echo","args":{}}.
- Server responds with {"type":"response","id":"r1","ok":true,"data":{"echo":"ok"},...}.
- RPC handler invoked exactly once.
