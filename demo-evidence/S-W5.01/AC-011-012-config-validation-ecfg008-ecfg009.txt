## AC-011: management_socket Validation (E-CFG-008)

Test: TestConfig_Validate_ManagementSocket_E_CFG_008_AC011

Sub-cases:
(a) empty_string_accepted_as_absent: management_socket="" (Go zero-value) → no error.
    In Go a string YAML field cannot distinguish explicit "" from absent — both unmarshal
    to "". BC-2.09.003 PC-10 / EC-014 specify absent is accepted. PASS.
(b) whitespace_only_rejected: management_socket="   " → E-CFG-008 error reported.
(c) tab_only_rejected: management_socket="\t" → E-CFG-008 error reported.
(d) valid_unix_path_accepted: management_socket="/run/sb.sock" → no error.
(e) valid_tcp_address_accepted: management_socket="127.0.0.1:9091" → no error.
(f) exhaustive_reporting_with_other_errors: invalid tick_interval + whitespace
    management_socket → both errors reported together in one pass (collect-all-failures
    pattern per BC-2.09.003 Inv-4).

## AC-012: authorized_operator_keys PEM Validation (E-CFG-009)

Test: TestConfig_Validate_AuthorizedOperatorKeys_E_CFG_009_AC012

Sub-cases:
(a) invalid_pem_at_index_0: not-PEM data at entry[0] → E-CFG-009[0] reported.
(b) valid_at_0_invalid_at_1: valid Ed25519 PEM at entry[0], invalid at entry[1]
    → E-CFG-009[1] only (entry[0] passes).
(c) empty_list_accepted: [] → no error (bootstrap mode; daemon key is sole authority).
(d) nil_list_accepted: nil → no error (same as empty; bootstrap mode).
(e) valid_ed25519_pem_accepted: PEM PUBLIC KEY with 32-byte Ed25519 key → no error.
(f) multiple_valid_pem_accepted: two valid Ed25519 PEM entries → no error.
(g) rsa_pem_rejected_wrong_key_type: RSA PUBLIC KEY in PEM → E-CFG-009 (wrong key type).
(h) wrong_pem_block_type_rejected: PEM block with type "PRIVATE KEY" → E-CFG-009.
(i) exhaustive_reporting_both_bad_entries_collected: two invalid entries →
    both E-CFG-009[0] and E-CFG-009[1] collected and returned together (Inv-4).
