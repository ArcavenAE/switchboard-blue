### F-P4L3-MED-002 — BC-2.08.001 Inv-3 unconditionally states "Unix-socket" transport; BC-2.07.004 EC-013 authorizes TCP loopback for console mode

**Confidence: HIGH.**

BC-2.08.001 v1.3 Invariant 3:
> "Remote control commands use the management-plane Unix-socket transport (ADR-006 / ADR-012)
> — the same authenticated channel used by all sbctl commands."

BC-2.07.004 EC-013 / AC-014 Ruling D states that the management-plane transport for console
mode (i.e., the running switchboard console daemon) binds to `127.0.0.1:9091` (TCP loopback)
rather than a Unix socket. The Unix-socket transport applies to the access-mode daemon.
ADR-012 establishes the management-plane pattern; EC-013 specifies the per-mode transport type.

Implementation at `cmd/switchboard/mgmt_wire.go:143,153–158` unconditionally uses
`"tcp"` / `"127.0.0.1:9091"` for console mode. This is CORRECT per BC-2.07.004 EC-013.

The spec defect is in BC-2.08.001 v1.3 Inv-3, which states "Unix-socket" unconditionally.
This extrapolates from the access-mode (S-6.03) pattern without accounting for EC-013's
per-mode differentiation. A console-mode operator following BC-2.08.001 Inv-3 verbatim would
expect a Unix socket; the actual transport is TCP loopback.

The RULING-W6TB-C intent (mgmt-plane authenticated channel, no data-plane bypass) is
preserved by TCP loopback. The transport-type wording is the defect.

**Severity rationale — MED:** The implementation is correct; the spec wording is incorrect.
A downstream test author following the BC literally could target a Unix socket and miss the
TCP loopback path. Story AC-001/AC-002/AC-003 repeat the "mgmt-plane Unix socket" language
from the BC, propagating the defect into AC text.

**Fix locus:**
- BC-2.08.001: Reword Inv-3 to defer per-mode transport type to BC-2.07.004 EC-013:
  "management-plane transport per BC-2.07.004 EC-013 (Unix socket for access mode;
  TCP loopback for console mode)". Bump BC-2.08.001 v1.3 → v1.4.
- Story: AC-001/AC-002/AC-003 transport clause: replace "mgmt-plane Unix socket
  (ADR-006/ADR-012 pattern)" with "management-plane transport per BC-2.07.004 EC-013
  (Unix socket for access mode; TCP loopback for console mode)". Bump story v1.3 → v1.4
  (or v1.4 if F-P4L3-MED-001 already produces that bump).
- VP-050: Story Trace row transport clause updated to match.
- RULING-W6TB-C: append retrospective note documenting that the ruling's "Unix-socket"
  language was extrapolated from access-mode pattern and that EC-013 authorizes TCP loopback
  for console mode. The ruling itself is not revised (historical audit trail).
