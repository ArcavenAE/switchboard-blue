## Important Findings (HIGH)

### F-P1L3-HIGH-1 — [process-gap] Wave-gate (Perimeter-2) adversary dispatch lacks HEAD-SHA verification gate

**Severity:** HIGH
**Category:** process-gap — pipeline hardening
**Status:** DEFERRED to upstream drbothen/vsdd-factory issue (item drafted in .vsdd-factory-issues-pending.md)

**Analysis:**
The Worktree-Identity Preflight rule (as this adversary understands it) is
anchored to Perimeter 1 (per-story adversarial review). The per-story dispatch
embeds `(worktree-abs-path, story-tip-SHA, story-id, canonical-repo-root)` and
the adversary verifies the tuple. Perimeter 2 (wave-gate) dispatches embed a
`develop_tip` SHA in the prompt, but the adversary has no tool-visible mechanism
to verify `git rev-parse HEAD == develop_tip`. This pass succeeded in catching
the mismatch only because it inspected `.git/refs/heads/develop` opportunistically
before reading any code artifact.

A less-thorough pass — or a future adversary that processes the dispatch and
immediately begins reading specs (which are always current on factory-artifacts,
regardless of develop HEAD) — could produce a silent false-green.

**Proposed fix:** Extend the Worktree-Identity Preflight rule to Perimeter 2.
Require a verification tuple `(worktree-abs-path, wave-tip-SHA, wave-id,
canonical-repo-root)` in every wave-gate dispatch. The adversary should verify
`git rev-parse HEAD == wave-tip-SHA` before reading any code artifact. On
mismatch: HALT with dispatch-integrity finding rather than proceeding.

Additionally, the state-manager burst that dispatches wave-gate reviews should
verify `git merge-base --is-ancestor <closure-merge-SHA> HEAD` for every
constituent story merge SHA against the dispatched worktree HEAD. Refuse
dispatch on failure.

**Deferred:** Filed as an upstream issue against `drbothen/vsdd-factory` per
the switchboard-blue local tracker.

root: the pipeline lacks a fetch/fast-forward pre-dispatch verification gate.

**Addressed by:** HIGH-1 remediation (verification-tuple embedding closes this
class of failure).
