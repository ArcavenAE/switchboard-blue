**Result: structurally identical.**

No divergence.

## Q5: Auth chain integration (ADR-004 admission + ADR-012 challenge/response)

**Result: both stories use the same chain; no bypass.**

- Client leg: both admin destroy and all three console commands route through `connectAndRun` (`cmd/sbctl/main.go`, `admin.go:262`, `console.go:102/120/151`). `connectAndRun` calls `Authenticate` which performs ADR-012 challenge/nonce/signature (evidence: `cmd/sbctl/client_test.go:39-96`, `main_test.go:208-213` — mock daemon emits `{type:"challenge","nonce":…,"daemon_sig":…}`, client verifies + returns signed nonce).
- Server leg — admin plane: `admin_handlers.go:419` returns `E-ADM-011` from `resolveAndVerifyCallerRole` when caller.Role != Control; defense-in-depth check repeated inside `svtnmgmt.go:774-781` (ErrDestroyUnauthorized).
- Server leg — console plane: `cmd/switchboard/console_handlers.go:41-79` invokes `verifyConsoleCallerRole` against `admission.AdmittedKeySet` on every attach/detach/switch, requiring RoleControl or RoleConsole and emitting `E-ADM-006` on failure. Handlers dispatched only after auth succeeds.
- The admission set (`internal/admission`) is a shared source of truth; both stories consult it via the same `AdmittedKeySet` reference (BuildConsoleHandlers takes `*admission.AdmittedKeySet` at construction).

S-7.03 does **not** bypass the auth chain S-6.05 depends on — they share it.

Observation O-A-3: `internal/session/remote.go` intentionally does not import `admission` (see line 17: "Allowed imports: {frame, admission} per ARCH-08 §6.6" — permissive comment, but the file actually imports neither); the admission enforcement lives at the wire boundary in `cmd/switchboard/console_handlers.go`, and `remote.go` is a pure state-transition layer. Layering is correct; not a defect.
