### F-P7L1-MED-1 — `runAdminSvtnDestroy` corrupts `--json` envelope with trailing plain-text line (in-scope AC-003; interface-definitions.md §162)

**Confidence:** HIGH
**File:** `/Users/skippy/work/aae-orc/run/switchboard-blue/.worktrees/S-6.05/cmd/sbctl/admin.go:262-269`
**Lens:** L1 (Correctness / Coverage)
**Perimeter:** 1 (story-scope — the offending line is in the story's own diff)

**Evidence:**

```go
// admin.go:262-269
if err := connectAndRun(ctx, target, keyPath, useJSON, "admin.svtn.destroy", rpcArgs, sio); err != nil {
    return err
}

// Print SVTN name so the operator can confirm which SVTN was destroyed
// (test: outBuf must contain svtnName — client-side print, not from server response).
_, _ = fmt.Fprintf(sio.out, "destroyed SVTN: %s\n", *nameFlag)
return nil
```

`connectAndRun` (client.go:346) invokes `writeSuccess` (main.go:95-107) which, when `useJSON=true`, writes the canonical JSON envelope to `sio.out`:

```go
// main.go:95-107
func writeSuccess(useJSON bool, data json.RawMessage, sio sbctlIO) {
    if useJSON {
        env := newSuccessEnvelope(data)
        out, err := json.Marshal(env)
        ...
        _, _ = fmt.Fprintln(sio.out, string(out))
        return
    }
    _, _ = fmt.Fprintln(sio.out, string(data))
}
```

The subsequent unconditional `fmt.Fprintf(sio.out, "destroyed SVTN: %s\n", *nameFlag)` runs **regardless of `useJSON`**. In `--json` mode, stdout therefore emits:

```
{"$schema":"…","ok":true,"error":null,"data":{"status":"destroyed"}}
destroyed SVTN: my-svtn
```

This violates the universal envelope contract in `interface-definitions.md:164`: "JSON output is produced when --json flag is present on any sbctl command. All JSON responses share a common envelope." Machine consumers piping `sbctl --json admin svtn destroy` into `jq` (or any single-document JSON parser) will fail on the trailing plain-text line.

**Cross-check against peer admin commands (same story wave / same file):**

| Command | Post-`connectAndRun` behavior |
|---|---|
| `admin svtn create` (admin.go:188) | `return connectAndRun(...)` — no post-print |
| `admin key register` (admin.go:395) | `return connectAndRun(...)` — no post-print |
| `admin key revoke` (admin.go:439) | `return connectAndRun(...)` — no post-print |
| `admin key expire` (admin.go:485) | `return connectAndRun(...)` — no post-print |
| `admin svtn destroy` (admin.go:262-268) | **prints `"destroyed SVTN: %s\n"` after `writeSuccess` returns** |

`admin svtn destroy` is the sole outlier. The peer commands' pattern (return the connectAndRun result directly) already satisfies AC-003's "prints confirmation to stdout on success" via `writeSuccess`'s default text-mode branch (main.go:106 `Fprintln(sio.out, string(data))`), which emits the server-returned JSON payload (e.g., `{"status":"destroyed"}`). The additional line at admin.go:268 is a story-scope regression against the file's own established pattern AND breaks the universal JSON envelope contract.

**Test coverage gap:** `TestSbctlAdmin_SVTNDestroy_HappyPath` (admin_test.go:1699) invokes `runAdmin(..., false, ...)` — `useJSON=false`. The JSON path is uncovered. The only assertion (`outBuf` contains `svtnName`, line 1727) is satisfied by the trailing plain-text line but does not exercise envelope integrity.

**Why this is MEDIUM (not HIGH):**
- Text-mode output remains functional (verbose but AC-003-satisfying).
- The JSON envelope corruption is deterministic and would surface immediately on the first `--json` invocation by an integrator, so blast radius is bounded.
- AC-003 is spec-silent on `--json` explicitly, but `interface-definitions.md:164` is universal across sbctl commands.

**Recommended remediation:** either (a) remove line 268 and let `writeSuccess` provide the confirmation (matches peer commands), or (b) gate line 268 behind `if !useJSON { … }`. Add a `TestSbctlAdmin_SVTNDestroy_HappyPath_JSON` subtest that asserts `outBuf.String()` parses cleanly as exactly one JSON envelope.
