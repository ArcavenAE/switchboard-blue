# VP-048: Control Node Creates/Destroys SVTNs

## Property Statement

(1) svtn create → the new SVTN appears in svtn list
(2) svtn destroy → the SVTN no longer appears in svtn list
    ∧ nodes can no longer admit to the destroyed SVTN
(3) **Ruling-7 defense-in-depth (Pass-3 L3 handoff):** The `admin.svtn.create` handler
    MUST explicitly check `caller.role == RoleControl` after `IsBootstrapKey(caller)` returns
    true. A caller that passes the bootstrap-key check but does NOT carry `RoleControl` MUST
    be rejected with E-ADM-009, even when the requested SVTN name matches no existing SVTN
    (existence oracle closed). This check is mandatory even though the bootstrap key is
    provisioned with `RoleControl` by construction — the explicit role check provides
    defense-in-depth against future bootstrap-key rotation or provisioning refactors that
    might inadvertently relax the role assignment.

    **Genesis Carve-Out (Ruling-8, 2026-07-01):** Property (3) applies only when the
    SVTN keySet is non-empty (`HasAnySVTN() == true`). On genesis create, `IsBootstrapKey`
    alone gates admission — no keySet entry yet exists, so the `role == RoleControl` keySet
    lookup is correctly skipped.

    **Mutation-test invariant:** Removing the `role == RoleControl` check must cause
    `TestAdminSVTNCreate_NonBootstrapControlKey_RejectsWithEADM009` to fail. The mutation
    test operates on the non-genesis path (i.e., a bootstrap key with non-RoleControl role
    must be rejected when at least one SVTN already exists). A mutation test on the genesis
    path (`HasAnySVTN() == false`) would falsely pass because the keySet is empty and the
    role lookup returns false by construction regardless of the guard.

SVTN lifecycle management must be immediately consistent: a create must be
visible in the next list call; a destroy must prevent any further admission.

## Source Contract

- **BC:** BC-2.07.001 v1.12 — Control node creates and destroys SVTNs; first control key bootstrapped locally
- **Postcondition/Invariant:** Postcondition — "(1) created SVTN appears in list; (2) destroyed SVTN absent from list and admission rejected."
- **Invariant 3 (Ruling-7 clause):** "Implementations MUST check `caller.role == RoleControl` explicitly after `IsBootstrapKey(caller)` — defense-in-depth against bootstrap-key rotation refactors."

//go:build integration

package svtnmgmt_test

import (
	"context"
	"testing"
	"time"

	"github.com/arcavenae/switchboard/internal/testenv"
)

// TestIntegration_ControlNode_SVTNCreateDestroy verifies create and destroy
// lifecycle operations on the control node.
func TestIntegration_ControlNode_SVTNCreateDestroy(t *testing.T) {
	t.Parallel()

	ctx, cancel := context.WithTimeout(context.Background(), 20*time.Second)
	t.Cleanup(cancel)

	env := testenv.New(t, ctx)
	t.Cleanup(env.Close)

	// (1) Create a new SVTN; it must appear in svtn list.
	t.Run("create appears in list", func(t *testing.T) {
		svtnID := env.ControlNode(t).CreateSVTN(t, "test-svtn-a")
		svtns, err := env.ControlNode(t).ListSVTNs(t)
		if err != nil {
			t.Fatalf("ListSVTNs error: %v", err)
		}
		found := false
		for _, s := range svtns {
			if s.ID == svtnID {
				found = true
				break
			}
		}
		if !found {
			t.Errorf("newly created SVTN %v not found in list", svtnID)
		}
	})

	// (2) Destroy a SVTN; it must not appear in list and admission must fail.
	t.Run("destroy absent from list and blocks admission", func(t *testing.T) {
		svtnID := env.ControlNode(t).CreateSVTN(t, "test-svtn-b")
		env.ControlNode(t).DestroySVTN(t, svtnID)

		// Give control plane a moment to propagate.
		time.Sleep(100 * time.Millisecond)

		svtns, err := env.ControlNode(t).ListSVTNs(t)
		if err != nil {
			t.Fatalf("ListSVTNs error: %v", err)
		}
		for _, s := range svtns {
			if s.ID == svtnID {
				t.Errorf("destroyed SVTN %v still appears in list", svtnID)
			}
		}

		// Attempt admission to the destroyed SVTN.
		creds := env.GenerateCredentialsForSVTN(t, svtnID)
		err = env.ConnectWithKey(t, creds)
		if err == nil {
			t.Error("expected admission to destroyed SVTN to fail, but it succeeded")
		}
	})

	// (3) Ruling-7 defense-in-depth: bootstrap key passing IsBootstrapKey with
	// role != RoleControl must be rejected E-ADM-009 (non-genesis path only).
	t.Run("bootstrap key non-RoleControl rejected E-ADM-009", func(t *testing.T) {
		t.Skip("Story Trace property (3) — implementer to fill")
	})

	// (4) admin.svtn.create handler + sbctl CLI makes svtn create end-to-end RPC-reachable.
	t.Run("admin.svtn.create handler + sbctl create-svtn", func(t *testing.T) {
		t.Skip("Story Trace property (4) — implementer to fill")
	})

	// (5) Non-bootstrap RoleControl caller rejected with E-ADM-009 even when name
	// matches (existence oracle closed) — Ruling-7 mutation-test invariant.
	t.Run("TestAdminSVTNCreate_NonBootstrapControlKey_RejectsWithEADM009", func(t *testing.T) {
		t.Skip("Story Trace property (5) — implementer to fill")

}
