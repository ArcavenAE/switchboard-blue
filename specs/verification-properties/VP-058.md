# VP-058: RouteFrame Calls verifyFrameHMAC Before IsAdmitted and SVTNRoute

## Property Statement

∀ call to `RouteFrame(hdr, payload, r)`:

1. `verifyFrameHMAC(hdr, payload, entry.FrameAuthKey)` is evaluated before `r.admittedKeySet.IsAdmitted(...)` is called.
2. `verifyFrameHMAC(hdr, payload, entry.FrameAuthKey)` is evaluated before `SVTNRoute(...)` is called.
3. If `verifyFrameHMAC` returns false, `RouteFrame` returns `ErrHMACVerificationFailed` and neither `IsAdmitted` nor `SVTNRoute` is reached.
4. If no forwarding-table entry exists for `(hdr.SVTNID, hdr.SrcAddr)`, `RouteFrame` returns `ErrHMACVerificationFailed` (auth key unavailable → unverifiable → dropped).

This property captures BC-2.05.008 invariant 1 (DI-006): no frame reaches `SVTNRoute` without a successful HMAC verification.

## Source Contract

- **BC:** BC-2.05.008 — RouteFrame wire-layer HMAC enforcement (Fail-Closed for Writes)
- **Postcondition 3:** "HMAC verification occurs BEFORE the admitted-set check."
- **Invariant 1 (DI-006):** "No frame reaches `SVTNRoute` without a successful `verifyFrameHMAC` call."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| code-audit | manual + grep | yes (static) | Verify call-graph ordering in `RouteFrame`: `verifyFrameHMAC` call site appears before `IsAdmitted` call site in the function body |
| proptest | gopter | no | For all (hdr, payload, key) triples: `RouteFrame` with a bad HMAC never reaches the forwarding-table lookup |

The primary verification for this property is code-audit (static ordering in `RouteFrame` body). The proptest adds dynamic evidence that the rejection is actually exercised at runtime.

## Proof Harness Skeleton

```go
package routing_test

import (
    "bytes"
    "crypto/ed25519"
    "errors"
    "testing"

    "github.com/arcavenae/switchboard/internal/admission"
    "github.com/arcavenae/switchboard/internal/frame"
    "github.com/arcavenae/switchboard/internal/hmac"
    "github.com/arcavenae/switchboard/internal/routing"
)

// seedKey generates a deterministic Ed25519 keypair from a fixed 32-byte seed.
// Using bytes.NewReader satisfies ed25519.GenerateKey's io.Reader contract
// without invoking crypto/rand, making the test fully reproducible.
func seedKey(t *testing.T, seed [32]byte) (ed25519.PublicKey, ed25519.PrivateKey) {
    t.Helper()
    pub, priv, err := ed25519.GenerateKey(bytes.NewReader(seed[:]))
    if err != nil {
        t.Fatalf("seedKey: %v", err)
    }
    return pub, priv
}

// TestRouteFrame_HMACEnforcedBeforeAdmission verifies VP-058 property 1–3:
// HMAC failure returns ErrHMACVerificationFailed without touching admitted-set.
//
// Setup:
//   1. RegisterKey (admitted=false initially; must complete challenge-response).
//   2. GenerateChallenge + sign nonce with node key → AdmitNode → admitted=true.
//   3. RegisterForwardingEntry with all-zero authKey (wrong key → HMAC will fail).
//   4. RouteFrame must return ErrHMACVerificationFailed, not ErrNotAdmitted.
func TestRouteFrame_HMACEnforcedBeforeAdmission(t *testing.T) {
    t.Parallel()

    ks := admission.NewAdmittedKeySet()
    r := routing.NewRouter(ks)

    var svtnID [16]byte
    copy(svtnID[:], "test-svtn-000001")

    // Step 1: deterministic keypairs for node and router.
    var nodeSeed [32]byte
    copy(nodeSeed[:], "vp058-node-seed-0000000000000001")
    nodePub, nodePriv := seedKey(t, nodeSeed)

    var routerSeed [32]byte
    copy(routerSeed[:], "vp058-rtr-seed-00000000000000001")
    _, routerPriv := seedKey(t, routerSeed)

    // Step 2: register the node key (admitted=false until challenge-response completes).
    // RegisterKey has no return value — void per admission.go line 155.
    ks.RegisterKey(svtnID, nodePub, admission.RoleAccess)

    // Step 3: derive the node address so we can build the forwarding entry.
    nodeAddr := frame.DeriveNodeAddress(svtnID, []byte(nodePub))

    // Step 4: complete challenge-response so IsAdmitted returns true.
    // AdmitNode is a package-level function: AdmitNode(challenge, resp, pubKey, svtnID, ks).
    challenge, err := admission.GenerateChallenge(routerPriv)
    if err != nil {
        t.Fatalf("GenerateChallenge: %v", err)
    }
    nonceSig := ed25519.Sign(nodePriv, challenge.Nonce[:])
    resp := admission.ChallengeResponse{NonceSig: nonceSig}
    if err := admission.AdmitNode(challenge, resp, nodePub, svtnID, ks); err != nil {
        t.Fatalf("AdmitNode: %v", err)
    }

    // Step 5: register a forwarding entry with an all-zero auth key.
    // The real frame auth key is derived from (pubkey, svtnID) via HKDF —
    // an all-zero override guarantees HMAC verification will fail.
    var wrongAuthKey [hmac.KeySize]byte // zero value
    r.RegisterForwardingEntry(svtnID, nodeAddr, wrongAuthKey)

    // Step 6: send a frame. HMAC tag is all-zeros (invalid) → must fail HMAC check.
    hdr := frame.OuterHeader{
        SVTNID:  svtnID,
        SrcAddr: nodeAddr,
        // HMACTag intentionally zero (invalid)
    }
    payload := []byte("test-payload")

    err = routing.RouteFrame(hdr, payload, r)
    if !errors.Is(err, routing.ErrHMACVerificationFailed) {
        t.Errorf("expected ErrHMACVerificationFailed, got: %v", err)
    }
    // Property 1: if ErrNotAdmitted surfaces, the admission check fired before
    // HMAC verification — that is the ordering violation VP-058 guards against.
    if errors.Is(err, admission.ErrNotAdmitted) {
        t.Error("ordering violation: admission check fired before HMAC verification")
    }
}

// TestRouteFrame_NoForwardingEntry_RejectsAsUnverifiable verifies VP-058 property 4:
// a frame with no forwarding-table entry (auth key unavailable) is rejected as
// unverifiable — ErrHMACVerificationFailed, not ErrNotAdmitted.
func TestRouteFrame_NoForwardingEntry_RejectsAsUnverifiable(t *testing.T) {
    t.Parallel()

    ks := admission.NewAdmittedKeySet()
    r := routing.NewRouter(ks)

    var svtnID [16]byte
    copy(svtnID[:], "test-svtn-000002")

    var nodeSeed [32]byte
    copy(nodeSeed[:], "vp058-node-seed-0000000000000002")
    nodePub, nodePriv := seedKey(t, nodeSeed)

    var routerSeed [32]byte
    copy(routerSeed[:], "vp058-rtr-seed-00000000000000002")
    _, routerPriv := seedKey(t, routerSeed)

    // Register and admit the node — so admission WOULD pass if reached.
    ks.RegisterKey(svtnID, nodePub, admission.RoleAccess)
    challenge, err := admission.GenerateChallenge(routerPriv)
    if err != nil {
        t.Fatalf("GenerateChallenge: %v", err)
    }
    nonceSig := ed25519.Sign(nodePriv, challenge.Nonce[:])
    resp := admission.ChallengeResponse{NonceSig: nonceSig}
    if err := admission.AdmitNode(challenge, resp, nodePub, svtnID, ks); err != nil {
        t.Fatalf("AdmitNode: %v", err)
    }

    // No forwarding entry registered — auth key is unavailable → unverifiable.
    nodeAddr := frame.DeriveNodeAddress(svtnID, []byte(nodePub))
    hdr := frame.OuterHeader{SVTNID: svtnID, SrcAddr: nodeAddr}
    err = routing.RouteFrame(hdr, nil, r)
    if !errors.Is(err, routing.ErrHMACVerificationFailed) {
        t.Errorf("expected ErrHMACVerificationFailed for missing forwarding entry, got: %v", err)
    }
}
```
