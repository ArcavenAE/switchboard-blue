# VP-043: XOR FEC — Single Loss in Group Recoverable

## Property Statement

∀ group of N=4 data frames {f0, f1, f2, f3} and parity frame p = f0 ⊕ f1 ⊕ f2 ⊕ f3:
  ∀ i ∈ {0,1,2,3}:
    Recover({f0,..,f3} \ {fi}, p) == fi

When exactly one data frame is missing from a FEC group, the XOR parity
frame must allow recovery of the missing frame. This is the invariant for
N=4 XOR FEC as defined in BC-2.02.007.

## Source Contract

- **BC:** BC-2.02.007 — XOR parity FEC covers frame groups; single loss in group recoverable without retransmit
- **Postcondition/Invariant:** Postcondition — "FEC Recover reconstructs missing frame from XOR parity."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| strong-oracle | independent xorOracle() reference + LCG-driven case loop | yes | ~35 000 pseudo-random groups of 4 byte-slice payloads (MMIX LCG, deterministic seed) |

### Implementing Test

See `internal/arq/fec_test.go::TestBC_2_02_007_VP043_SingleLossRecovery_Property` — deterministic MMIX LCG loop over ~35 000 pseudo-random group configurations, asserting `enc.Recover(group_with_gap, parity) == original[i]` for each single-loss position i in {0,1,2,3}. Oracle is `xorOracle()` at lines 50-67 of the same file (independent XOR reference, no SUT calls).
