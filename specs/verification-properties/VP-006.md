# VP-006: VerifyHMAC Rejects Wrong Key

## Property Statement

∀ key1 []byte, key2 []byte, data []byte where key1 != key2 ∧ len(key1) > 0 ∧ len(key2) > 0:
  VerifyHMAC(key2, data, ComputeHMAC(key1, data)) == false

A tag computed with key1 must never verify successfully with key2.
This enforces HMAC key binding: an attacker who does not hold the node's
admission key cannot forge a valid tag for any frame on that SVTN.

## Source Contract

- **BC:** BC-2.05.005 — HMAC frame authentication at first router boundary
- **Postcondition/Invariant:** Invariant DI-006 — "Every frame carrying SVTN-scoped traffic is verified against the admitted key set." The key set is admission-specific; cross-key acceptance would break SVTN isolation.

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| proptest | gopter v0.2.9+ | no | random (key1, key2, data) triples filtered to key1 != key2; key lengths 1–64 bytes; data 0–1024 bytes |

## Proof Harness Skeleton

Note: S-2.01 implementation (commit 5d6f928) merged the wrong-key rejection property into TestPropComputeVerifyConsistency — each of its 10 table cases asserts `!VerifyHMAC(otherKey, frame, tag)` after the same-key verification. This is functionally equivalent to a standalone TestPropVerifyHMAC_RejectsWrongKey; the merger reduces test-suite duplication for the (key, frame) variation axis. A standalone test is retained as an option below.

```go
package hmac_test

import (
	"bytes"
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/hmac"
)

func TestPropVerifyHMAC_RejectsWrongKey(t *testing.T) {
	properties := gopter.NewProperties(nil)

	properties.Property("tag computed with key1 does not verify with key2 when key1 != key2", prop.ForAll(
		func(key1, key2, data []byte) bool {
			if len(key1) == 0 || len(key2) == 0 {
				return true // skip degenerate keys
			}
			if bytes.Equal(key1, key2) {
				return true // keys accidentally equal; skip (not the case we're testing)
			}
			tag := hmac.ComputeHMAC(key1, data)
			return !hmac.VerifyHMAC(key2, data, tag)
		},
		gen.SliceOf(gen.UInt8()), // key1
		gen.SliceOf(gen.UInt8()), // key2
		gen.SliceOf(gen.UInt8()), // data
	))

	properties.TestingRun(t)
}
```
