# VP-023: Replay Delivers Keystrokes in Sequence Order

## Property Statement

∀ sequence of OnUpstream calls producing deliveries d[0], d[1], ..., d[k]:
  ∀ i < k: d[i].ChanSeq ≤ d[i+1].ChanSeq

Delivered upstream frames must have monotonically non-decreasing chan_seq
values. Frames arriving out of order must be buffered and reordered before
delivery, not passed through in arrival order.

## Source Contract

- **BC:** BC-2.02.004 — Replay Delivers Upstream Frames in chan_seq Order
- **Postcondition/Invariant:** Invariant — "Deliver order is monotonically non-decreasing by chan_seq."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|

## Proof Harness Skeleton

```go
package replay_test

import (
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/frame"
	"github.com/arcavenae/switchboard/internal/replay"
)

func TestProp_Replay_MonotoneDeliveryOrder(t *testing.T) {
	properties := gopter.NewProperties(nil)

	properties.Property("delivered chan_seq values are monotonically non-decreasing", prop.ForAll(
		func(seqPerm []uint8) bool {
			var delivered []uint64
			r := replay.New(replay.Config{WindowSize: 64})
			r.OnDeliver(func(f frame.Frame) {
				delivered = append(delivered, f.ChanSeq)
			})

			// Inject frames in the generated order.
			for _, s := range seqPerm {
				f := frame.Frame{}
				f.ChanSeq = uint64(s % 8)
				r.OnUpstream(f)
			}

			// Drain any buffered frames.
			r.Flush()

			for i := 1; i < len(delivered); i++ {
				if delivered[i] < delivered[i-1] {
					return false
				}
			}
			return true
		},
		gen.SliceOfN(16, gen.UInt8()),
	))

	properties.TestingRun(t)
}
```
