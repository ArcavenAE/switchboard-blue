# VP-056: Console Detach Releases Session Without Closing It — Session and Observers Unaffected

## Property Statement

Given an attached console C on session S, when C issues detach:

(a) C's downstream half-channel transitions to closed state and stops delivering
    frames to C.
(b) The access node retains session S in active state (session is not terminated).
(c) Other consoles attached to S continue receiving downstream frames uninterrupted.
(d) C may subsequently re-attach to S and resume receiving the current downstream
    output.

Formally:

State before detach:
  S.State        == active
  C.ChannelState == attached
  O.ChannelState == attached  (O = a read-only observer on S, if present)

After C.Detach():
  C.ChannelState == closed              (postcondition 1)
  S.State        == active              (postcondition 2 — session unchanged)
  O.ChannelState == attached            (postcondition 5 — observer unaffected)
  O can receive frame F from S          (postcondition 5 — delivery continues)

After C.Attach(S) [re-attach]:
  C.ChannelState == attached            (postcondition 6)
  C can receive frame F2 from S         (resumes downstream output)

## Rationale

VP-033 (`Console attach/detach lifecycle`) exercises the full attach/detach
lifecycle as an e2e test. However, VP-033 is scoped to the happy path and
does not verify the multi-observer scenario in postcondition 5, nor the
re-attach scenario in postcondition 6. BC-2.04.004 is the primary BC for
detach semantics, but VP-033's source_bc is BC-2.04.003 (attach) and
BC-2.04.004 (detach) — it covers the lifecycle but not the observer
continuity or re-attach properties.

VP-056 closes these specific gaps:
- Observer continuity: a read-only observer must not lose frames when a
  full-access console detaches (EC-001 in BC-2.04.004).
- Re-attach capability: postcondition 6 of BC-2.04.004 is unverified.

Integration (rather than e2e) is used because the test uses in-process
session management with a fake transport; no tmux process is required.
The session-management state machine (`internal/session`) is the unit
under test, wired with a fake downstream transport.

## Source Contract

- **BC:** BC-2.04.004 — Console detach releases session without closing it; session continues on access node
- **Postcondition 1:** "The console's channel to the access node is closed cleanly."
- **Postcondition 2:** "The tmux session on the access node continues running."
- **Postcondition 5:** "Read-only observers (if any) continue receiving."
- **Postcondition 6:** "The session becomes available for a new console to attach."
- **Invariant 1:** "Detach is non-destructive: it never terminates the tmux session."
- **EC-001:** "Full-access console detaches; read-only observers remain."

## Proof Harness Skeleton

```go
//go:build integration

package session_test

import (
	"testing"

	"github.com/arcavenae/switchboard/internal/session"
)

// TestIntegration_ConsoleDetach_SessionContinues verifies BC-2.04.004:
// console detach closes the console's channel but leaves the session active,
// observers unaffected, and re-attach possible.
func TestIntegration_ConsoleDetach_SessionContinues(t *testing.T) {
	t.Parallel()

	mgr := session.NewManager(session.ManagerConfig{
		Transport: session.NewFakeTransport(),
	})

	// Create session S.
	s, err := mgr.CreateSession("agent-01")
	if err != nil {
		t.Fatalf("create session: %v", err)
	}

	// Attach full-access console C.
	c, err := mgr.AttachConsole(s.ID, session.RoleFullAccess)
	if err != nil {
		t.Fatalf("attach console C: %v", err)
	}

	// Attach read-only observer O.
	o, err := mgr.AttachConsole(s.ID, session.RoleReadOnly)
	if err != nil {
		t.Fatalf("attach observer O: %v", err)
	}

	// Detach C.
	if err := mgr.DetachConsole(c.ID); err != nil {
		t.Fatalf("detach console C: %v", err)
	}

	// (a) C's channel is closed.
	if c.ChannelState() != session.ChannelStateClosed {
		t.Errorf("C channel state: want closed, got %v", c.ChannelState())
	}

	// (b) Session S remains active.
	if s2, err := mgr.GetSession(s.ID); err != nil || s2.State != session.StateActive {
		t.Errorf("session state: want active, got %v (err %v)", s2.State, err)
	}

	// (c) Observer O is still attached.
	if o.ChannelState() != session.ChannelStateAttached {
		t.Errorf("O channel state: want attached, got %v", o.ChannelState())
	}

	// Verify O still receives a downstream frame.
	testFrame := []byte("output-line")
	mgr.InjectDownstream(s.ID, testFrame) // fake: inject a frame from the session
	received := o.DrainReceived()
	if len(received) == 0 || string(received[0]) != string(testFrame) {
		t.Errorf("O delivery: want %q, got %v", testFrame, received)
	}

	// (d) C can re-attach.
	c2, err := mgr.AttachConsole(s.ID, session.RoleFullAccess)
	if err != nil {
		t.Fatalf("re-attach console: %v", err)
	}
	if c2.ChannelState() != session.ChannelStateAttached {
		t.Errorf("re-attached C2 state: want attached, got %v", c2.ChannelState())
	}

	// C2 receives downstream output after re-attach.
	mgr.InjectDownstream(s.ID, testFrame)
	received2 := c2.DrainReceived()
	if len(received2) == 0 {
		t.Error("C2 re-attach: expected downstream frame after re-attach, got none")
	}
}
```
