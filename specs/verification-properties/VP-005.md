# VP-005: VerifyHMAC Rejects Single-Bit Tag Flips

## Property Statement

∀ key []byte, frame []byte, tag [16]byte where tag == ComputeHMAC(key, frame):
  ∀ i ∈ [0, 127]:  // 128 bits in 16-byte tag
    VerifyHMAC(key, frame, flipBit(tag, i)) == false

Flipping any single bit in a valid HMAC tag must cause VerifyHMAC to return
false. This property is the minimum security bar for truncated HMAC-SHA256:
no single-bit corruption or targeted forgery attempt on the tag field passes
verification.

## Source Contract

- **BC:** BC-2.05.005 — HMAC Frame Authentication at First Router Boundary
- **Postcondition/Invariant:** Postcondition 2 — "HMAC verification fails: frame dropped." Invariant DI-006 — "Every frame carrying SVTN-scoped traffic is verified against the admitted key set."

## Proof Harness Skeleton

```go
package hmac_test

import (
	"testing"

	"github.com/arcavenae/switchboard/internal/hmac"
)

// FuzzVerifyHMAC_RejectsBitFlips verifies that a single-bit corruption of any
// tag byte causes VerifyHMAC to return false.
func FuzzVerifyHMAC_RejectsBitFlips(f *testing.F) {
	// Seed corpus: representative key and frame sizes.
	f.Add(
		[]byte("test-admission-key-32-bytes-long!"),
		[]byte("frame payload for fuzz testing"),
	)
	f.Add(
		[]byte{0xFF, 0x00, 0xAB, 0xCD},
		[]byte{},
	)

	f.Fuzz(func(t *testing.T, key, frame []byte) {
		if len(key) == 0 {
			t.Skip()
		}
		tag := hmac.ComputeHMAC(key, frame)

		// Exercise every single-bit flip across the 16-byte tag.
		for byteIdx := 0; byteIdx < len(tag); byteIdx++ {
			for bitIdx := 0; bitIdx < 8; bitIdx++ {
				flipped := tag
				flipped[byteIdx] ^= 1 << uint(bitIdx)
				if hmac.VerifyHMAC(key, frame, flipped) {
					t.Errorf(
						"VerifyHMAC accepted flipped tag: byte=%d bit=%d key=%x frame=%x",
						byteIdx, bitIdx, key, frame,
					)
				}
			}
		}
	})
}
```
