# VP-020: ARQ Delivers Frames In-Order

## Property Statement

∀ sequence of send/ack events on an ARQ instance producing deliveries d[0], d[1], ..., d[k]:
  ∀ i < k: d[i].Sequence < d[i+1].Sequence
  ∧ ∀ i: d[i].Sequence is not delivered before d[i-1].Sequence is delivered

Delivered frames must have strictly increasing sequence numbers. No frame N
may be passed to the deliver callback before frame N-1 has been delivered,
even if acks arrive out-of-order.

## Source Contract

- **BC:** BC-2.02.005 — Downstream ARQ with piggybacked ACK and SACK bitmap
- **Postcondition/Invariant:** Invariant — "Deliver order matches sequence order; no frame is skipped or reordered."

## Proof Harness Skeleton

```go
package arq_test

import (
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/arq"
	"github.com/arcavenae/switchboard/internal/frame"
)

func TestProp_ARQ_InOrderDelivery(t *testing.T) {
	properties := gopter.NewProperties(nil)

	properties.Property("delivered frames have strictly increasing sequence numbers", prop.ForAll(
		func(ackOrder []uint8) bool {
			// Send frames 0..N-1, then ack them in the given order.
			const N = 8
			var deliveryOrder []uint64
			a := arq.New(arq.Config{WindowSize: N})
			a.OnDeliver(func(f frame.Frame) {
				deliveryOrder = append(deliveryOrder, f.Sequence)
			})

			// Send all frames first.
			for i := 0; i < N; i++ {
				f := frame.Frame{}
				f.Sequence = uint64(i)
				_ = a.Send(f)
			}

			// Ack in the generated order (indices mod N, deduplicated).
			seen := make(map[uint8]bool)
			for _, idx := range ackOrder {
				idx = idx % N
				if !seen[idx] {
					seen[idx] = true
					a.OnAck(uint64(idx))
				}
			}

			// Verify delivery order is strictly increasing.
			for i := 1; i < len(deliveryOrder); i++ {
				if deliveryOrder[i] <= deliveryOrder[i-1] {
					return false
				}
			}
			return true
		},
		gen.SliceOfN(8, gen.UInt8()),
	))

	properties.TestingRun(t)
}
```
