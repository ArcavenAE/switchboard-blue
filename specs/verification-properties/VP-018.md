# VP-018: HalfChannel Emits Empty Frame When No Payload

## Property Statement

∀ state HalfChannelState:
  let f = Tick(state, nil)[0]
  len(f.Payload) == 0

When `Tick` is called with a nil or zero-length payload, the returned frame's
payload field must be zero-length. The frame must still be emitted (exactly
one frame per tick — see VP-016) but must carry no data bytes.

## Source Contract

- **BC:** BC-2.01.001 — Timeslice clock fires on every tick regardless of data availability
- **BC:** BC-2.01.002 — Empty-tick frame is a valid liveness signal
- **Postcondition/Invariant:** Postcondition 2 — "Frame.Payload is zero-length when input is nil."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| proptest | gopter v0.2.9+ | no | random state values; payload explicitly nil or []byte{}; asserts len(frame.Payload) == 0 |

## Proof Harness Skeleton

```go
package halfchannel_test

import (
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/halfchannel"
)

func TestProp_Tick_NilPayload_EmitsEmptyFrame(t *testing.T) {
	properties := gopter.NewProperties(nil)

	properties.Property("Tick(state, nil) produces a frame with zero-length payload", prop.ForAll(
		func(seq uint64, flags uint8) bool {
			ch := halfchannel.New(halfchannel.State{Sequence: seq, Flags: flags})

			// Test with nil
			frames := ch.Tick(nil)
			if len(frames) != 1 {
				return false
			}
			if len(frames[0].Payload) != 0 {
				return false
			}

			// Also verify with explicit empty slice
			ch2 := halfchannel.New(halfchannel.State{Sequence: seq, Flags: flags})
			frames2 := ch2.Tick([]byte{})
			if len(frames2) != 1 {
				return false
			}
			return len(frames2[0].Payload) == 0
		},
		gen.UInt64(),
		gen.UInt8(),
	))

	properties.TestingRun(t)
}
```
