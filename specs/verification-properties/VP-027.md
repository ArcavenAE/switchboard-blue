# VP-027: QualityIndicator Degradation Transitions Only Monotonically Downward

## Property Statement

∀ sustained degradation sequence (RTT rising, loss rising):
  quality transitions follow only: green → yellow → red
  ∧ no single-step red → green transition occurs without an intermediate yellow

Under a monotone degradation workload the quality indicator must not
improve without first passing through an intermediate state. The
red → green skip is explicitly forbidden; recovery must go red → yellow → green.

## Source Contract

- **BC:** BC-2.06.001 — Quality indicator (green/yellow/red) derived from measured path latency and loss
- **BC:** BC-2.06.002 — Missing expected frame is a degradation signal triggering indicator downgrade
- **Postcondition/Invariant:** Invariant 3 (BC-2.06.001) — "Indicator transitions are hysteretic over 3 consecutive measurements."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| proptest | gopter v0.2.9+ | no | random monotone-increasing RTT/loss sequences; records state transitions; asserts no skip-transition |

## Proof Harness Skeleton

The as-built implementation in `internal/metrics/metrics_prop_test.go` fully
covers this property using the real API. The skeleton below mirrors that
delivered harness for completeness.

```go
package metrics_test

import (
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/metrics"
)

// degradationStep is a single (rttMs, lossPct) update pair.
type degradationStep struct {
	rttMs   float64
	lossPct float64
}

// genMonotoneRisingSteps generates 16 steps with monotonically non-decreasing
// RTT and loss values by accumulating random increments.
func genMonotoneRisingSteps() gopter.Gen {
	return gen.SliceOfN(32, gen.UInt32Range(0, 50)).Map(func(incs []uint32) []degradationStep {
		steps := make([]degradationStep, 16)
		var rttAcc, lossAcc float64
		for i := 0; i < 16; i++ {
			rttAcc += float64(incs[i*2])
			lossAcc += float64(incs[i*2+1] % 6) // 0-5 loss increment
			if lossAcc > 100 {
				lossAcc = 100
			}
			steps[i] = degradationStep{rttMs: rttAcc, lossPct: lossAcc}
		}
		return steps
	})
}

// genMonotoneRecoverySteps generates 16 steps starting at Red-range and
// monotonically decreasing toward Green-range.
func genMonotoneRecoverySteps() gopter.Gen {
	return gen.SliceOfN(32, gen.UInt32Range(0, 50)).Map(func(decs []uint32) []degradationStep {
		steps := make([]degradationStep, 16)
		rttAcc := float64(800)
		lossAcc := float64(25)
		for i := 0; i < 16; i++ {
			rttAcc -= float64(decs[i*2])
			lossAcc -= float64(decs[i*2+1] % 6)
			if rttAcc < 0 {
				rttAcc = 0
			}
			if lossAcc < 0 {
				lossAcc = 0
			}
			steps[i] = degradationStep{rttMs: rttAcc, lossPct: lossAcc}
		}
		return steps
	})
}

// TestProp_QualityIndicator_NoGreenToRedSkip verifies that under a monotone
// degradation workload the indicator never transitions directly from Green to Red.
func TestProp_QualityIndicator_NoGreenToRedSkip(t *testing.T) {
	properties := gopter.NewProperties(gopter.DefaultTestParameters())

	properties.Property("no Green→Red skip during monotone degradation", prop.ForAll(
		func(steps []degradationStep) bool {
			qi := metrics.NewQualityIndicator()
			prev := qi.Current()

			for _, s := range steps {
				qi.Update(s.rttMs, s.lossPct)
				cur := qi.Current()
				if prev == metrics.Green && cur == metrics.Red {
					return false // illegal skip
				}
				prev = cur
			}
			return true
		},
		genMonotoneRisingSteps(),
	))

	properties.TestingRun(t)
}

// TestProp_QualityIndicator_NoRedToGreenSkipDuringRecovery verifies that during
// monotone recovery the indicator never jumps directly from Red to Green.
func TestProp_QualityIndicator_NoRedToGreenSkipDuringRecovery(t *testing.T) {
	properties := gopter.NewProperties(gopter.DefaultTestParameters())

	properties.Property("no Red→Green skip during monotone recovery", prop.ForAll(
		func(steps []degradationStep) bool {
			qi := metrics.NewQualityIndicator()
			// Pre-drive to Red.
			for i := 0; i < metrics.HysteresisCount; i++ {
				qi.Update(600, 25)
			}
			prev := qi.Current()

			for _, s := range steps {
				qi.Update(s.rttMs, s.lossPct)
				cur := qi.Current()
				if prev == metrics.Red && cur == metrics.Green {
					return false // illegal skip
				}
				prev = cur
			}
			return true
		},
		genMonotoneRecoverySteps(),
	))

	properties.TestingRun(t)
}
```
