# VP-047: Per-Path Metrics Queryable via sbctl

## Property Statement

`sbctl paths list --json` returns a JSON response where:
  len(response.paths) ≥ 1
  ∧ ∀ path p in response.paths:
      p.rtt_ms is present
      ∧ p.loss_pct is present
      ∧ p.status is present

Per-path metrics must be machine-readable via the CLI. Operators and
monitoring systems depend on these fields to assess path health.

## Source Contract

- **BC:** BC-2.06.003 — Per-path RTT and loss metrics queryable via sbctl
- **Postcondition/Invariant:** Postcondition — "sbctl paths list --json returns paths with rtt_ms, loss_pct, and status."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| integration | go test (testing.T) | yes | run sbctl paths list --json against a live stack; parse JSON; assert required fields present in each path entry |

## Proof Harness Skeleton

```go
//go:build integration

package metrics_test

import (
	"bytes"
	"context"
	"encoding/json"
	"os/exec"
	"testing"
	"time"

	"github.com/arcavenae/switchboard/internal/testenv"
)

// NOTE (F-T4): The JSON envelope is defined in interface-definitions.md v1.3+ as:
//   {"ok": true, "data": {"paths": [...]}}
// The BC-2.06.003 Canonical Test Vectors (§Canonical Test Vectors) show bare
// path arrays; those vectors document the per-path entry schema only.
// interface-definitions.md v1.3 is the authority for the outer envelope shape.
// The structs below reflect the full enveloped response.

type pathEntry struct {
	RTTMs   *float64 `json:"rtt_ms"`
	LossPct *float64 `json:"loss_pct"`
	Status  *string  `json:"status"`
}

type pathsData struct {
	Paths []pathEntry `json:"paths"`
}

type pathsEnvelope struct {
	OK   bool       `json:"ok"`
	Data *pathsData `json:"data"`
}

// TestIntegration_PerPathMetrics_QueryableViaSbctl verifies that
// `sbctl paths list --json` returns at least one path with required fields.
func TestIntegration_PerPathMetrics_QueryableViaSbctl(t *testing.T) {
	t.Parallel()

	ctx, cancel := context.WithTimeout(context.Background(), 15*time.Second)
	t.Cleanup(cancel)

	env := testenv.New(t, ctx)
	t.Cleanup(env.Close)

	env.CreateSession(t)

	// Allow metrics to accumulate.
	time.Sleep(500 * time.Millisecond)

	cmd := exec.CommandContext(ctx, "sbctl",
		"--target", env.RouterAddr(t),
		"paths", "list", "--json",
	)
	var stdout, stderr bytes.Buffer
	cmd.Stdout = &stdout
	cmd.Stderr = &stderr

	if err := cmd.Run(); err != nil {
		t.Fatalf("sbctl paths list --json failed: %v\nstderr: %s", err, stderr.String())
	}

	var env2 pathsEnvelope
	if err := json.Unmarshal(stdout.Bytes(), &env2); err != nil {
		t.Fatalf("failed to parse JSON envelope: %v\nbody: %s", err, stdout.String())
	}
	if !env2.OK {
		t.Fatalf("envelope ok=false; body: %s", stdout.String())
	}
	if env2.Data == nil {
		t.Fatal("envelope data is null")
	}

	if len(env2.Data.Paths) == 0 {
		t.Fatal("expected at least one path in response; got empty list")
	}

	for i, p := range env2.Data.Paths {
		if p.RTTMs == nil {
			t.Errorf("path[%d]: missing rtt_ms field", i)
		}
		if p.LossPct == nil {
			t.Errorf("path[%d]: missing loss_pct field", i)
		}
		if p.Status == nil || *p.Status == "" {
			t.Errorf("path[%d]: missing or empty status field", i)
		}
	}
}
```
