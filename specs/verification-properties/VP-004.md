# VP-004: ComputeHMAC / VerifyHMAC Consistency

## Property Statement

∀ key []byte, data []byte where len(key) > 0:
  VerifyHMAC(key, data, ComputeHMAC(key, data)) == true

A tag produced by ComputeHMAC must always be accepted by VerifyHMAC with
the same key and data. This is the fundamental self-consistency check: if
the implementation were to use different internal representations or
truncation schemes in compute vs. verify, this property catches it.

## Source Contract

- **BC:** BC-2.05.005 — HMAC frame authentication at first router boundary
- **Postcondition/Invariant:** Postcondition 1 — "HMAC verification succeeds: frame forwarded to destination." (The precondition is that the sending node computed the HMAC correctly; this VP proves that compute→verify is a no-op under identical inputs.)

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| table-driven | stdlib testing | yes | hand-curated cases covering empty data, small, MTU, and max payload lengths; 10 boundary cases |
| proptest | gopter v0.2.9+ | no | random (key, data) pairs; key length 1–64 bytes; data length 0–1024 bytes |

**proof_method_rationale:** HMAC-SHA256 is a deterministic primitive; correctness holds by construction for all input lengths. Either a hand-curated table covering boundary cases (empty, small, MTU, max payload) or a gopter property test exercising random inputs is acceptable. The table-driven approach is preferred for deterministic primitives where bug surface is bounded by length-class coverage rather than statistical-distribution exploration.

## Proof Harness Skeleton

Note: S-2.01 implementation (commit 5d6f928) uses table-driven (TestPropComputeVerifyConsistency) covering 10 boundary cases. A gopter harness is retained below for reference if random-input proof is desired in a future iteration.

```go
package hmac_test

import (
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/hmac"
)

func TestPropComputeVerifyConsistency(t *testing.T) {
	properties := gopter.NewProperties(nil)

	properties.Property("VerifyHMAC(key, data, ComputeHMAC(key, data)) is always true", prop.ForAll(
		func(key, data []byte) bool {
			if len(key) == 0 {
				return true // skip degenerate case; covered separately
			}
			tag := hmac.ComputeHMAC(key, data)
			return hmac.VerifyHMAC(key, data, tag)
		},
		gen.SliceOf(gen.UInt8()).SuchThat(func(b []byte) bool { return len(b) > 0 }), // key
		gen.SliceOf(gen.UInt8()), // data (may be empty — empty-tick frames)
	))

	properties.TestingRun(t)
}
```
