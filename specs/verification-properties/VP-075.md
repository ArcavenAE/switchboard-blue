# VP-075: admin.key.register/revoke/expire Handlers Reject Non-Control Callers with E-ADM-009 (Handler-Layer Caller-Role Enforcement)

## Property Statement

∀ authenticated connection to `mgmt.Server` where:
- the caller's key role is NOT `control` (i.e., role is `console`, `access`, or any
  other non-control role), AND
- the caller issues a **mutating** `admin.key.*` RPC command (`admin.key.register`,
  `admin.key.revoke`, or `admin.key.expire`),

the server MUST:
1. Perform a server-side role lookup of the authenticated caller's key against the
   SVTN's admitted key store.
2. Reject the request with error `E-ADM-009`:
   `"insufficient authority for operation admin.key.<op>: key <fingerprint> has role <role>"`
3. Return an in-band `ok: false` response — the connection MUST remain open (not closed).
4. NOT apply the requested key change to the admitted key store.

For callers with `control` role, the handlers MUST proceed to the key operation
(register/revoke/expire) per BC-2.05.004 Postconditions 1–3.

**Scope exclusion (F-L2-003):** `admin.key.list-keys` is a read-only operation open to
any admitted role and is NOT covered by this property. A console-role or access-role key
calling `admin.key.list-keys` MUST receive `ok: true` (subject to valid SVTN args),
NOT E-ADM-009. F-L2-003 excludes admin.key.list-keys from the CONTROL-ONLY AUTHORITY
gate (any admitted role suffices to call it). It does NOT exclude admin.key.list-keys
from the ADMISSION gate — callers must still be admitted to the target SVTN in some
active role, or be a member of the operator-set, or be the daemon bootstrap key. See
BC-2.05.004:155 for the list-keys authority sharpening and EC-008 for the trilogy of
reachable admission failure modes, and CWE-862 for the underlying cross-SVTN
enumeration defense. Errata: interface-definitions.md v1.25.

The property is fail-closed: the default outcome for any role lookup that does not
resolve to `control` is E-ADM-009 rejection without mutation (for mutating operations).

## Source Contract

- **BC:** BC-2.05.004 — Key Lifecycle — Register, Revoke, and Expire Admission and
  Session-Authorization Keys
- **Precondition 1 (management authority):** The requesting node must have management
  authority for the operation — control-role key, not revoked, not expired
  (admission-control authority per ADR-004 and DI-001). This is the authority gate
  that VP-075 verifies.
- **Postconditions 1–3:** For callers that pass the authority gate, the key operation
  (register/revoke/expire) proceeds per BC-2.05.004 Postconditions 1–3.
- **Domain Invariant DI-001:** Admission-control authority — only control-role keys
  may perform key management operations.
- **Capability Anchor:** CAP-019 ("Key lifecycle management (register, revoke, expire)")
  per capabilities.md §CAP-019.

## Proof Harness Skeleton

```go
//go:build integration

package main

import (
	"context"
	"crypto/ed25519"
	"crypto/rand"
	"net"
	"path/filepath"
	"testing"
	"time"

	"github.com/arcavenae/switchboard/internal/admission"
	"github.com/arcavenae/switchboard/internal/mgmt"
	"github.com/arcavenae/switchboard/internal/svtnmgmt"
)

// TestIntegration_AdminKeyHandlers_RejectsNonControlCaller_VP075 verifies VP-075:
// admin.key.* handlers (built via BuildAdminHandlers) reject non-control callers
// with E-ADM-009 (in-band, connection kept open).
//
// Helper functions sendAdminRPC / readAdminRPCResponse / authenticateAdminClient
// follow the pattern in internal/mgmt/mgmt_test.go and
// cmd/switchboard/admin_handlers_e2e_test.go — see those files for canonical
// implementations (F-P10L2-003).
func TestIntegration_AdminKeyHandlers_RejectsNonControlCaller_VP075(t *testing.T) {
	t.Parallel()

	// Generate a console-role keypair (non-control).
	consolePub, consolePriv, err := ed25519.GenerateKey(rand.Reader)
	if err != nil {
		t.Fatalf("generate console key: %v", err)
	}

	// Generate an access-role keypair.
	accessPub, accessPriv, err := ed25519.GenerateKey(rand.Reader)
	if err != nil {
		t.Fatalf("generate access key: %v", err)
	}

	// Build an SVTNManager with the console and access keys registered as non-control.
	svtnMgr := svtnmgmt.New()
	// Create the SVTN so RegisterKey has a valid target.
	if _, err := svtnMgr.Create("test-svtn"); err != nil {
		t.Fatalf("create test-svtn: %v", err)
	}
	// Register consolePub and accessPub as non-control roles so the role-resolution
	// path fires when they issue mutating admin.key.* RPCs.
	if _, err := svtnMgr.RegisterKey("test-svtn", consolePub, admission.RoleConsole); err != nil {
		t.Fatalf("register console key: %v", err)
	}
	if _, err := svtnMgr.RegisterKey("test-svtn", accessPub, admission.RoleAccess); err != nil {
		t.Fatalf("register access key: %v", err)
	}

	// Build operator key set including both non-control keys so auth passes.
	// The role check happens AFTER auth, at the handler layer (in BuildAdminHandlers).
	ops := mgmt.NewOperatorKeySet([]ed25519.PublicKey{
		consolePub,
		accessPub,
	})

	// Build admin handlers via the S-6.06 AC-001 contract entry point.
	handlers := BuildAdminHandlers(svtnMgr, ops)

	subCases := []struct {
		name    string
		keyPriv ed25519.PrivateKey
		role    string
	}{
		{"console_role_rejected", consolePriv, "console"},
		{"access_role_rejected", accessPriv, "access"},
	}

	for _, tc := range subCases {
		tc := tc
		t.Run(tc.name, func(t *testing.T) {
			t.Parallel()

			// Use canonical Unix-socket pattern (F-P10L2-001) matching
			// admin_handlers_e2e_test.go: net.Listener required by mgmt.NewServer.
			sockDir := t.TempDir()
			sockPath := filepath.Join(sockDir, "mgmt.sock")
			ln, err := net.Listen("unix", sockPath)
			if err != nil {
				t.Fatalf("listen: %v", err)
			}
			t.Cleanup(func() { ln.Close() })

			// Generate daemon keypair per sub-case (isolated from other cases).
			_, daemonPriv, err := ed25519.GenerateKey(rand.Reader)
			if err != nil {
				t.Fatalf("generate daemon key: %v", err)
			}

			ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
			t.Cleanup(cancel)
			go func() {
				_ = mgmt.NewServer(ln, daemonPriv, ops, handlers, "dev").Serve(ctx)
			}()

			// Dial the Unix socket and complete the handshake with the non-control key.
			conn, err := net.Dial("unix", sockPath)
			if err != nil {
				t.Fatalf("dial: %v", err)
			}
			t.Cleanup(func() { conn.Close() })

			// Complete the handshake successfully so we reach RPC dispatch.
			// authenticateAdminClient follows the pattern in admin_handlers_e2e_test.go.
			authenticateAdminClient(t, conn, tc.keyPriv, daemonPriv.Public().(ed25519.PublicKey))

			// Issue admin.key.register with a non-control key.
			// sendAdminRPC / readAdminRPCResponse follow the pattern in admin_handlers_e2e_test.go.
			sendAdminRPC(t, conn, "admin.key.register", map[string]interface{}{
				"svtn_id": "test-svtn",
				"key":     "ssh-ed25519 AAAA...",
				"role":    "access",
			})

			// Expect in-band ok:false with E-ADM-009; connection must stay open.
			resp := readAdminRPCResponse(t, conn)
			if resp["ok"] != false {
				t.Errorf("expected ok:false, got ok:%v", resp["ok"])
			}
			code, _ := resp["code"].(string)
			if code != "E-ADM-009" {
				t.Errorf("expected E-ADM-009, got code:%q", code)
			}

			// Verify connection is still open by sending a second mutating RPC.
			// (list-keys is open to all roles; use admin.key.expire to stay within the gated set.)
			sendAdminRPC(t, conn, "admin.key.expire", map[string]interface{}{
				"svtn_id": "test-svtn",
				"key":     "ssh-ed25519 AAAA...",
				"after":   "720h",
			})
			resp2 := readAdminRPCResponse(t, conn)
			if resp2["ok"] != false {
				t.Errorf("expected ok:false on second RPC for non-control key, got ok:%v", resp2["ok"])
			}
			// Connection must still be alive (not EOF).
			conn.SetReadDeadline(time.Now().Add(100 * time.Millisecond))
			// Receiving another response (or timeout, not EOF) means connection is open.
		})
	}
}
```
