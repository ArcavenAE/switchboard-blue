# VP-053: Empty-Tick Frame Sequence — K Consecutive Empty Ticks Emit K Frames with Contiguous Sequence Numbers

## Property Statement

Given a half-channel with no application data queued for K consecutive ticks (K ≥ 1),
exactly K empty-tick frames are emitted with sequence numbers s, s+1, …, s+K-1, each
carrying zero-length payload, `frame_type=EMPTY_TICK (0x02)`, and a valid HMAC tag.

Formally:

∀ HalfChannel H, ∀ K ∈ [1, 100], ∀ starting sequence s:
  let frames = H.TickN(K, payload=nil)
  assert len(frames) == K
  ∀ i ∈ [0, K-1]:
    frames[i].ChannelHeader.SequenceNumber == s + i
    frames[i].OuterHeader.FrameType        == EMPTY_TICK (0x02)
    frames[i].OuterHeader.PayloadLength    == 0
    frames[i].OuterHeader.HMAC             is valid (VerifyHMAC returns nil)

VP-018 verifies that a *single* tick emits an empty frame; this VP covers the
multi-tick case and explicitly verifies contiguous sequence numbering across a
run of empty ticks — the invariant that receivers rely on to detect gaps (DI-008).

## Rationale

BC-2.01.002 postconditions 1–3 specify that every empty tick must emit a frame with:
- zero-length payload
- `frame_type = EMPTY_TICK`
- an incremented sequence number

VP-018 exercises the single-tick case. Without VP-053, an implementation could
fail to increment the sequence number on the second consecutive empty tick (a
fence-post bug) and still pass VP-018. The quality-observability subsystem counts
*gaps* in the received sequence — if sequence numbers are not contiguous, the
receiver miscounts missed frames and produces spurious quality degradations
(BC-2.06.002 depends on this regularity).

## Source Contract

- **BC:** BC-2.01.002 — Empty-tick frame is a valid liveness signal
- **Postcondition 1:** "An empty-tick frame is emitted with zero-length payload."
- **Postcondition 2:** "The outer header is fully populated (frame type = EMPTY_TICK, HMAC)."
- **Postcondition 3 (via BC-2.01.001):** "The channel header carries the sequence number."
- **Invariant 1 (DI-008):** "Empty-tick frames are never skipped."
- **Invariant 3:** "An empty-tick frame increments the sequence number."

## Proof Harness Skeleton

```go
package halfchannel_test

import (
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/halfchannel"
	"github.com/arcavenae/switchboard/internal/hmac"
)

// TestPropEmptyTickSequence verifies that K consecutive empty ticks produce K frames
// with contiguous sequence numbers, zero-length payloads, and EMPTY_TICK frame type.
func TestPropEmptyTickSequence(t *testing.T) {
	properties := gopter.NewProperties(nil)

	genK := gen.IntRange(1, 100)

	properties.Property(
		"K empty ticks emit K frames with contiguous seq, EMPTY_TICK type, zero payload",
		prop.ForAll(
			func(k int) bool {
				clock := halfchannel.NewFakeClock()
				hc := halfchannel.New(halfchannel.Config{
					Direction: halfchannel.Upstream,
					Key:       testKey(), // test fixture; never nil
				}, clock)

				startSeq := hc.NextSequenceNumber()
				frames := make([]halfchannel.Frame, 0, k)

				for i := 0; i < k; i++ {
					clock.AdvanceTick()
					f, err := hc.Tick(nil) // nil payload = no application data
					if err != nil {
						return false
					}
					frames = append(frames, f)
				}

				if len(frames) != k {
					return false
				}
				for i, f := range frames {
					if f.ChannelHeader.SequenceNumber != startSeq+uint32(i) {
						return false
					}
					if f.OuterHeader.FrameType != halfchannel.FrameTypeEmptyTick {
						return false
					}
					if f.OuterHeader.PayloadLength != 0 {
						return false
					}
					if err := hmac.Verify(testKey(), f.Bytes()); err != nil {
						return false
					}
				}
				return true
			},
			genK,
		),
	)

	properties.TestingRun(t)
}
```
