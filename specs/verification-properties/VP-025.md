# VP-025: DropCache Never Exceeds Configured Capacity

## Property Statement

∀ DropCache c with capacity N:
  after adding any number of entries:
    len(c) ≤ N

Adding the (N+1)th entry must evict the oldest entry, keeping the cache
at exactly N entries. The cache must never silently grow beyond its
configured bound.

## Source Contract

- **BC:** BC-2.02.009 — Bounded drop cache suppresses looping duplicate frames by checksum
- **Postcondition/Invariant:** Invariant — "len(dropcache) ≤ configured_capacity at all times."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| proptest | gopter v0.2.9+ | no | random capacity N in [1,256]; random sequence of distinct checksums; asserts len ≤ N after each insert |

## Proof Harness Skeleton

```go
package multipath_test

import (
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/multipath"
)

func TestProp_DropCache_NeverExceedsCapacity(t *testing.T) {
	properties := gopter.NewProperties(nil)

	properties.Property("DropCache len never exceeds configured capacity", prop.ForAll(
		func(capacity uint8, checksums []uint32) bool {
			if capacity == 0 {
				capacity = 1
			}
			cap := int(capacity)
			c := multipath.NewDropCache(cap)

			for _, cksum := range checksums {
				c.Add(uint64(cksum))
				if c.Len() > cap {
					return false
				}
			}
			return true
		},
		gen.UInt8Range(1, 64),
		gen.SliceOfN(128, gen.UInt32()),
	))

	properties.TestingRun(t)
}
```
