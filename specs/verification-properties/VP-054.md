# VP-054: Receiver Deduplication — First-Arriving Copy Delivered, Subsequent Identical Frames Discarded

## Property Statement

Given two byte-identical frames f1 and f2 arriving at a receiver in order (f1
before f2, same channel, same sequence number), the application sees exactly one
delivery (f1's payload); f2 is silently discarded; the delivered sequence number
equals f1's sequence number; and no ACK side-effects are generated for f2.

Formally:

∀ frame F with seq=N, payload=P:
  let delivered = receiver.Receive(F)        // first arrival
  assert delivered.Payload == P
  assert delivered.SeqNum  == N

  let result2  = receiver.Receive(F)         // duplicate arrival
  assert result2 == nil (no delivery)
  assert receiver.AckPendingFor(N) == false  // no duplicate ACK side-effect

Additionally, two distinct frames with seq=N and seq=N+1 are both delivered,
confirming only true duplicates (same seq, same content) are suppressed:

  assert receiver.Receive(frameN)   → delivered (seq=N)
  assert receiver.Receive(frameN+1) → delivered (seq=N+1)

## Rationale

VP-024 (`Multipath delivers first copy, discards duplicates`) tests the
pure-core deduplication logic at the `internal/multipath` layer in isolation.
However, BC-2.02.002 postcondition 2 specifies the behavior "at the receiver"
spanning from network layer to application delivery — a property that crosses
the pure-core boundary into the effectful shell (network receive → dedup →
application deliver). An integration test is required to verify that:

1. The deduplication window is correctly wired between the network input path
   and the application delivery path.
2. No ACK side-effects are generated for discarded duplicates (postcondition 2),
   which requires the ARQ layer to also be in scope.

VP-054 closes this coverage gap: VP-024 proves the dedup algorithm is correct;
VP-054 proves the algorithm is correctly wired in the receive pipeline.

## Source Contract

- **BC:** BC-2.02.002 — Receiver Delivers First-Arriving Copy and Silently
  Discards Subsequent Duplicates
- **Postcondition 1:** "The first-arriving copy of a frame is delivered to
  the application layer."
- **Postcondition 2:** "Any subsequent frame with the same sequence number
  from the same channel is discarded without error, without ACK side-effects."
- **Invariant 1 (DI-009):** "First arrival wins."
- **Invariant 2:** "Deduplication window covers at least 1 second of history."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| integration | go test (testing.T) with in-process receiver stack | yes | inject f1 then f2 (identical); assert single delivery; inject f1 then f3 (different seq); assert two deliveries; verify no ACK side-effects for f2 |

Integration is chosen over proptest because deduplication correctness spans
multiple layers (network receive, dedup window, ARQ ACK path, application
delivery queue). The in-process receiver stack wires these layers together
with a fake transport; no real network I/O is required.

## Proof Harness Skeleton

```go
//go:build integration

package multipath_test

import (
	"testing"
	"time"

	"github.com/arcavenae/switchboard/internal/arq"
	"github.com/arcavenae/switchboard/internal/multipath"
)

// TestIntegration_ReceiverDedup_FirstArrivalDelivered verifies that only the
// first-arriving copy of a frame is delivered; duplicates are silently discarded
// with no ACK side-effects.
func TestIntegration_ReceiverDedup_FirstArrivalDelivered(t *testing.T) {
	t.Parallel()

	ackLog := &ackRecorder{}
	recv := multipath.NewReceiver(multipath.ReceiverConfig{
		DeduplicationWindow: time.Second,
		OnAck:               ackLog.Record,
	})

	// Build two byte-identical frames with the same sequence number.
	f := buildTestFrame(t, 42, []byte("hello"))
	fDup := buildTestFrame(t, 42, []byte("hello")) // identical

	// First arrival: must be delivered.
	delivered1, err := recv.Receive(f)
	if err != nil {
		t.Fatalf("first receive: %v", err)
	}
	if delivered1 == nil {
		t.Fatal("first receive: expected delivery, got nil")
	}
	if delivered1.SeqNum != 42 {
		t.Errorf("seq: want 42, got %d", delivered1.SeqNum)
	}

	// Second arrival (duplicate): must be discarded.
	delivered2, err := recv.Receive(fDup)
	if err != nil {
		t.Fatalf("duplicate receive: %v", err)
	}
	if delivered2 != nil {
		t.Errorf("duplicate: expected nil delivery, got seq=%d", delivered2.SeqNum)
	}

	// No ACK side-effect for the discarded duplicate (postcondition 2).
	if ackLog.CountFor(42) > 1 {
		t.Errorf("ACK side-effect: ack count for seq=42 is %d, want ≤1", ackLog.CountFor(42))
	}
}

// TestIntegration_ReceiverDedup_DistinctSeqBothDelivered verifies that two
// frames with different sequence numbers are both delivered (non-suppression).
func TestIntegration_ReceiverDedup_DistinctSeqBothDelivered(t *testing.T) {
	t.Parallel()

	recv := multipath.NewReceiver(multipath.ReceiverConfig{
		DeduplicationWindow: time.Second,
	})

	fA := buildTestFrame(t, 10, []byte("first"))
	fB := buildTestFrame(t, 11, []byte("second"))

	dA, err := recv.Receive(fA)
	if err != nil || dA == nil {
		t.Fatalf("frame seq=10: want delivery, got err=%v dA=%v", err, dA)
	}
	dB, err := recv.Receive(fB)
	if err != nil || dB == nil {
		t.Fatalf("frame seq=11: want delivery, got err=%v dB=%v", err, dB)
	}
}

// ackRecorder counts ACK callbacks per sequence number.
type ackRecorder struct {
	counts map[uint32]int
}

func (a *ackRecorder) Record(seq uint32) {
	if a.counts == nil {
		a.counts = make(map[uint32]int)
	}
	a.counts[seq]++
}

func (a *ackRecorder) CountFor(seq uint32) int {
	return a.counts[seq]
}

func buildTestFrame(t *testing.T, seq uint32, payload []byte) arq.Frame {
	t.Helper()
	// Build a minimal valid frame for the dedup test. Full HMAC verification
	// is out of scope here; the receiver uses a pre-validated frame path.
	return arq.Frame{SeqNum: seq, Payload: payload}
}
```
