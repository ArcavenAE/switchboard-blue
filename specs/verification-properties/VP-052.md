# VP-052: Missing Expected Tick Within Deadline Triggers Quality Indicator Downgrade

## Property Statement

When a frame source stops producing expected ticks and the deadline passes without
receiving a tick, the quality indicator downgrades:

∀ QualityMonitor M in state Green:
  given expected_tick_interval T and 3 consecutive missed ticks:
    M.Indicator() transitions from Green → Yellow

∀ QualityMonitor M in state Yellow:
  given expected_tick_interval T and 3 consecutive missed ticks:
    M.Indicator() transitions from Yellow → Red

Formally: missing an expected tick past the deadline (`now > last_tick_time + T + jitter_budget`)
is treated as a signal equivalent to RTT or loss exceeding the relevant threshold, per
the quality thresholds defined in ARCH-03.

## Rationale

BC-2.06.002 requires the quality monitor to detect missing frames and trigger an
indicator downgrade. VP-027 tests monotonicity of degradation transitions (quality
never upgrades spontaneously) but does not inject missing-frame events. BC-2.06.002's
core claim — that a missed tick causes a downgrade — is not exercised by VP-027.

Empty-tick frames are the liveness signal of the protocol (BC-2.01.002). Their
absence is the primary signal for path failure. Without VP-052, an implementation
that reacts only to explicit TLPKTDROP events and never to missed ticks would pass
VP-027 while violating BC-2.06.002.

The 3-measurement hysteresis (canonical value from BC-2.01.002 EC-001 and BC-2.06.001
invariant 3) applies: 3 consecutive missed ticks trigger the downgrade, not 1.

## Source Contract

- **BC:** BC-2.06.002 — Missing expected frame is a degradation signal triggering indicator downgrade
- **Core claim:** "The quality monitor detects missing frames and degrades the
  quality indicator."
- **BC-2.01.002 EC-001:** "After threshold (implementation: ≥3 consecutive missed
  ticks), quality indicator degrades."
- **ARCH-03 thresholds:** Green→Yellow at RTT >100ms OR loss >5%; Yellow→Red at
  RTT >500ms OR loss >20%.

## Proof Harness Skeleton

```go
//go:build integration

package metrics_test

import (
	"testing"
	"time"

	"github.com/arcavenae/switchboard/internal/metrics"
)

// FakeClock implements metrics.Clock for deterministic testing.
type FakeClock struct {
	now time.Time
}

func (f *FakeClock) Now() time.Time        { return f.now }
func (f *FakeClock) Advance(d time.Duration) { f.now = f.now.Add(d) }

// TestIntegration_MissingTick_GreenToYellow verifies that 3 consecutive missed
// ticks cause the quality indicator to downgrade from Green to Yellow.
func TestIntegration_MissingTick_GreenToYellow(t *testing.T) {
	t.Parallel()

	clock := &FakeClock{now: time.Now().UTC()}
	tickInterval := 10 * time.Millisecond
	jitterBudget := 2 * time.Millisecond // NFR-009: ≤ 2ms p99

	m := metrics.NewMonitor(metrics.Config{
		TickInterval:    tickInterval,
		JitterBudget:    jitterBudget,
		HysteresisCount: 3, // canonical: BC-2.01.002 EC-001, BC-2.06.001 invariant 3
		Clock:           clock,
	})

	// Establish stable Green state: deliver 10 frames on schedule.
	for i := 0; i < 10; i++ {
		clock.Advance(tickInterval)
		m.RecordTick(clock.Now())
	}
	if got := m.Indicator(); got != metrics.Green {
		t.Fatalf("expected Green after 10 on-time ticks, got %v", got)
	}

	// Miss 3 consecutive ticks by advancing the clock past their deadlines.
	for i := 0; i < 3; i++ {
		// Advance past deadline: tickInterval + jitterBudget + 1ms
		clock.Advance(tickInterval + jitterBudget + time.Millisecond)
		m.CheckDeadline(clock.Now()) // called by the quality monitor's timer loop
	}

	// After 3 consecutive misses, indicator must have downgraded to Yellow.
	if got := m.Indicator(); got != metrics.Yellow {
		t.Errorf("expected Yellow after 3 consecutive missed ticks, got %v", got)
	}
}

// TestIntegration_MissingTick_YellowToRed verifies that 3 consecutive missed
// ticks in Yellow state downgrade the indicator to Red.
func TestIntegration_MissingTick_YellowToRed(t *testing.T) {
	t.Parallel()

	clock := &FakeClock{now: time.Now().UTC()}
	tickInterval := 10 * time.Millisecond
	jitterBudget := 2 * time.Millisecond

	m := metrics.NewMonitor(metrics.Config{
		TickInterval:    tickInterval,
		JitterBudget:    jitterBudget,
		HysteresisCount: 3,
		Clock:           clock,
	})

	// Drive to Yellow via 3 missed ticks.
	for i := 0; i < 3; i++ {
		clock.Advance(tickInterval + jitterBudget + time.Millisecond)
		m.CheckDeadline(clock.Now())
	}
	if got := m.Indicator(); got != metrics.Yellow {
		t.Fatalf("prerequisite: expected Yellow, got %v", got)
	}

	// Miss 3 more consecutive ticks to reach Red.
	for i := 0; i < 3; i++ {
		clock.Advance(tickInterval + jitterBudget + time.Millisecond)
		m.CheckDeadline(clock.Now())
	}

	if got := m.Indicator(); got != metrics.Red {
		t.Errorf("expected Red after 3 consecutive missed ticks in Yellow, got %v", got)
	}
}

// TestIntegration_MissingTick_RecoveryResetsHysteresis verifies that a single
// on-time tick resets the consecutive-miss counter (hysteresis is per-streak,
// not cumulative).
func TestIntegration_MissingTick_RecoveryResetsHysteresis(t *testing.T) {
	t.Parallel()

	clock := &FakeClock{now: time.Now().UTC()}
	tickInterval := 10 * time.Millisecond
	jitterBudget := 2 * time.Millisecond

	m := metrics.NewMonitor(metrics.Config{
		TickInterval:    tickInterval,
		JitterBudget:    jitterBudget,
		HysteresisCount: 3,
		Clock:           clock,
	})

	// Deliver 10 ticks to establish Green.
	for i := 0; i < 10; i++ {
		clock.Advance(tickInterval)
		m.RecordTick(clock.Now())
	}

	// Miss 2 ticks (below hysteresis threshold of 3).
	for i := 0; i < 2; i++ {
		clock.Advance(tickInterval + jitterBudget + time.Millisecond)
		m.CheckDeadline(clock.Now())
	}

	// Deliver one on-time tick — resets the consecutive-miss counter.
	clock.Advance(tickInterval)
	m.RecordTick(clock.Now())

	// Indicator must still be Green (streak reset before threshold).
	if got := m.Indicator(); got != metrics.Green {
		t.Errorf("expected Green after streak reset, got %v", got)
	}
}
```
