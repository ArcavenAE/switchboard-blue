# VP-079: paths.ping RPC Handler Performs Zero PathTracker Interaction

## Property Statement

∀ invocation of the `paths.ping` RPC handler registered via
`internal/mgmt.RegisterPingHandler`:

1. The handler's request argument is the empty object `{}` — it reads no
   fields from the incoming `json.RawMessage`.
2. The handler's response is exactly `{"pong": true}` — no additional
   field, no omission, no falsified value.
3. The handler performs zero reads or writes against any `PathTracker`,
   `internal/paths`, or `internal/metrics` state — no per-path metrics
   computation, no quality/status derivation, no side effect of any kind
   beyond returning the constant response.

This property enforces BC-2.06.004 Postcondition 1 (bodyless RPC, no
per-path metrics computation), Invariant 1 (no additional Tier-2 authority
— implying no metrics-store dependency), and Invariant 2 (response carries
no session content or per-path metrics state; the wire payload is the
literal constant `{"pong": true}`).

## Source Contract

- **BC:** BC-2.06.004 — On-Demand Single-Target Reachability Probe via `sbctl paths ping`
- **PC-1:** "...issues `paths.ping` with empty args (`{}`). On success, the daemon returns `{"pong": true}`..."
- **PC-4:** "`paths.ping` performs no per-path metrics computation and returns no quality classification."
- **Invariant 2:** "The response carries no session content, keystroke data, or per-path metrics state (DI-001-equivalent content absence — the wire payload is the literal constant `{"pong": true}`)."
- **Architecture Module:** `internal/mgmt` (BC-2.06.004 Traceability — deliberately distinct from `internal/metrics`/`internal/paths` per the SS-06 registry, since `paths.ping` does no metrics-package work).

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| code-audit | manual + grep/AST analysis of `internal/mgmt` | yes (static) | Enumerate the handler's imports and call graph; verify no reference to `PathTracker`, `internal/paths`, or `internal/metrics` types |

### Audit Protocol

The auditor executes the following steps:

1. Locate the `paths.ping` handler registration (`RegisterPingHandler` /
   its handler closure) in `internal/mgmt`.
2. `grep -rn 'PathTracker\|internal/paths\|internal/metrics'` scoped to the
   file(s) implementing the `paths.ping` handler — assert zero matches
   (excluding comments that merely describe the *absence* of such
   interaction).
3. Walk the Go import graph for the handler's file(s)
   (`go list -f '{{join .Imports "\n"}}'`) to confirm no direct or
   transitive import of `internal/paths` or `internal/metrics`.
4. Verify the handler's request-decode step performs no field access on
   the incoming `json.RawMessage` (empty-args contract — Property 1).
5. Verify the handler's response literal is exactly `{"pong": true}` by
   inspection — no additional struct field, no computed value.
6. Record findings.
