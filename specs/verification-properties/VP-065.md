# VP-065: Management Server Rejects Replayed Challenge Nonce Within a Connection

## Property Statement

For any single `mgmt.Server` connection where the client successfully receives
a CHALLENGE with nonce N:

1. If the client sends a CHALLENGE_RESPONSE with `nonce_sig = Sign(key, N)` and
   the key is authorized, the first auth attempt succeeds (AUTH_OK returned) — this
   confirms the nonce was valid.

2. If the client then attempts a second auth exchange on the same connection
   presenting the same nonce bytes N again (as if renegotiating), the server
   MUST reject the second attempt with AUTH_FAIL (E-ADM-010) and close the
   connection.

The property covers the per-connection nonce set described in ADR-012 §Replay
Safety. Cross-connection replay is not a risk by construction (fresh nonce per
connection), but within-connection reuse MUST be rejected.

Note: the standard one-handshake-per-connection protocol makes this a
forward-compatibility safety check — it ensures that any future protocol extension
that adds renegotiation cannot silently accept a replayed nonce.

## Source Contract

- **BC:** BC-2.07.004 — Daemon Management Server Authenticates All Connections via
  Ed25519 Challenge-Response (Fail-Closed)
- **Postcondition PC-3:** nonce is recorded in per-connection nonce set; same nonce
  not accepted again within connection lifetime.
- **Invariant 4:** nonce is always fresh per connection.

## Proof Harness Skeleton

```go
package mgmt_test

import (
    "context"
    "crypto/ed25519"
    "crypto/rand"
    "encoding/json"
    "net"
    "path/filepath"
    "testing"
)

// TestMgmtServer_RejectsReplayedNonce_VP065 verifies VP-065:
// within a single connection, a nonce that was already used in a successful
// auth exchange cannot be replayed.
//
// This test requires the mgmt.Server to expose a test hook or the protocol
// to allow a second challenge-response cycle on the same connection. If the
// server closes after AUTH_OK (normal one-RPC protocol), verify instead that
// a new connection issuing a replayed nonce_sig against the server's new nonce
// fails (the new nonce is different, so the old sig won't verify — this tests
// the cross-connection safety which holds by construction).
//
// If the server supports renegotiation in a future extension, the within-connection
// replay test applies directly.
func TestMgmtServer_RejectsReplayedNonce_VP065(t *testing.T) {
    t.Parallel()

    _, daemonPriv, err := ed25519.GenerateKey(rand.Reader)
    if err != nil {
        t.Fatalf("generate daemon key: %v", err)
    }

    opPub, opPriv, err := ed25519.GenerateKey(rand.Reader)
    if err != nil {
        t.Fatalf("generate operator key: %v", err)
    }

    keySet := mgmt.NewOperatorKeySet([]ed25519.PublicKey{opPub})
    handlers := []mgmt.Handler{} // no handlers needed for auth-only test

    // Scenario: client records nonce from connection-1, then opens connection-2.
    // Connection-2 issues a DIFFERENT nonce. Client tries to send the old sig.
    // Because ed25519.Verify(newPubkey, newNonce, oldSig) = false, this is
    // rejected — demonstrating cross-connection replay safety by construction.

    t.Run("cross_connection_replay_fails_by_construction", func(t *testing.T) {
        t.Parallel()

        // Connection 1: record the nonce and produce a valid sig.
        sockDir1 := t.TempDir()
        sockPath1 := filepath.Join(sockDir1, "mgmt.sock")
        ln1, err := net.Listen("unix", sockPath1)
        if err != nil {
            t.Fatalf("listen srv1: %v", err)
        }
        defer ln1.Close()
        go mgmt.NewServer(ln1, daemonPriv, keySet, handlers, "dev").Serve(context.Background()) //nolint:errcheck
        client1, err := net.Dial("unix", sockPath1)
        if err != nil {
            t.Fatalf("dial srv1: %v", err)
        }

        challenge1 := readMessage(t, client1)
        nonce1 := mustDecodeBase64URL(t, challenge1["nonce"].(string))
        sig1 := ed25519.Sign(opPriv, nonce1)
        client1.Close() // abandon connection 1 without responding

        // Connection 2: server issues a fresh nonce. Try to replay sig1.
        sockDir2 := t.TempDir()
        sockPath2 := filepath.Join(sockDir2, "mgmt.sock")
        ln2, err := net.Listen("unix", sockPath2)
        if err != nil {
            t.Fatalf("listen srv2: %v", err)
        }
        defer ln2.Close()
        go mgmt.NewServer(ln2, daemonPriv, keySet, handlers, "dev").Serve(context.Background()) //nolint:errcheck
        client2, err := net.Dial("unix", sockPath2)
        if err != nil {
            t.Fatalf("dial srv2: %v", err)
        }
        defer client2.Close()

        challenge2 := readMessage(t, client2)
        nonce2 := mustDecodeBase64URL(t, challenge2["nonce"].(string))

        // Sanity: nonces must differ (crypto/rand virtually guarantees this).
        if string(nonce1) == string(nonce2) {
            t.Skip("nonces collided — astronomically unlikely, skipping")
        }

        // Send old sig1 (signed over nonce1) against connection-2's nonce2.
        sendMessage(t, client2, map[string]interface{}{
            "type":      "challenge_response",
            "nonce_sig": encodeBase64URL(sig1),
            "pubkey":    encodeBase64URL([]byte(opPub)),
        })

        // ed25519.Verify(opPub, nonce2, sig1) = false → AUTH_FAIL.
        resp := readMessage(t, client2)
        if resp["type"] != "auth_fail" {
            t.Errorf("expected auth_fail for replayed sig, got: %v", resp["type"])
        }
        if resp["code"] != "E-ADM-010" {
            t.Errorf("expected E-ADM-010, got: %v", resp["code"])
        }
    })
}
```
