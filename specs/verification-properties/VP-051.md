# VP-051: HalfChannel Independence — B's Tick Schedule Unaffected by A's Frame Production

## Property Statement

Given two HalfChannels A and B configured with different `tick_interval` values
and independent seeds, B's tick events and frame count are unaffected by A's
frame production.

Formally:

∀ HalfChannel A (tick_interval_A, seed_A), HalfChannel B (tick_interval_B, seed_B)
  where tick_interval_A ≠ tick_interval_B ∧ seed_A ≠ seed_B:
  ∀ N ∈ [1, 1000]:
    let b_before = B.tick_count()
    produce N frames from A
    let b_after = B.tick_count()
    assert b_before == b_after ∧ B.next_tick_time() is unchanged

## Rationale

BC-2.01.003 guarantees that upstream and downstream half-channels have completely
independent clocks and sequence spaces. VP-016 and VP-017 verify single-half-channel
properties (one frame per tick, sequence increment). Neither tests the independence
invariant: that operating one half-channel has no side-effects on the other.

Without this VP, an implementation that shares a clock source or tick counter
between half-channels could pass VP-016/VP-017 individually while violating
BC-2.01.003's independence guarantee. This VP closes that coverage gap.

Independence matters because:
1. Downstream ARQ retransmit storms must not disturb the upstream tick schedule
   (BC-2.01.003 EC-001).
2. Different tick intervals (e.g., 10ms upstream, 50ms downstream) must coexist
   without interference.
3. Path selection may differ per direction (BC-2.01.003 EC-004); scheduling
   independence enables this.

## Source Contract

- **BC:** BC-2.01.003 — Upstream and Downstream Half-Channels Operate with
  Independent Clocks and Sequence Spaces
- **Postcondition 4:** "Each half-channel's empty-tick clock fires on its own schedule."
- **Edge Case EC-001:** "Downstream ARQ retransmit storm while upstream is idle:
  Upstream continues emitting empty-tick frames at normal interval."
- **Edge Case EC-002:** "Upstream tick configured at 10ms; downstream at 50ms:
  Both clocks run independently."

## Proof Harness Skeleton

```go
package halfchannel_test

import (
	"testing"
	"time"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/halfchannel"
)

// TestPropHalfChannelIndependence verifies that producing frames from HalfChannel A
// does not affect HalfChannel B's tick schedule or frame count.
func TestPropHalfChannelIndependence(t *testing.T) {
	properties := gopter.NewProperties(nil)

	// Generate two distinct tick intervals in the 5–50ms range (ADR-008).
	genDistinctIntervals := gopter.CombineGens(
		gen.IntRange(5, 25),  // interval A: 5–25ms
		gen.IntRange(26, 50), // interval B: 26–50ms (always != A)
	).Map(func(vals []interface{}) [2]time.Duration {
		a := time.Duration(vals[0].(int)) * time.Millisecond
		b := time.Duration(vals[1].(int)) * time.Millisecond
		return [2]time.Duration{a, b}
	})

	genFrameCount := gen.IntRange(1, 100)

	properties.Property(
		"producing N frames from A does not change B's tick count or schedule",
		prop.ForAll(
			func(intervals [2]time.Duration, n int) bool {
				// Use a fake clock so the test is deterministic and fast.
				clock := halfchannel.NewFakeClock()

				hcA := halfchannel.New(halfchannel.Config{
					TickInterval: intervals[0],
					Direction:    halfchannel.Upstream,
				}, clock)
				hcB := halfchannel.New(halfchannel.Config{
					TickInterval: intervals[1],
					Direction:    halfchannel.Downstream,
				}, clock)

				// Record B's state before touching A.
				bTicksBefore := hcB.TickCount()
				bNextTickBefore := hcB.NextTickTime()

				// Produce N frames from A by advancing A's clock N ticks.
				for i := 0; i < n; i++ {
					clock.AdvanceA(intervals[0])
					hcA.Tick()
				}

				// B's clock is independent — it has not advanced.
				bTicksAfter := hcB.TickCount()
				bNextTickAfter := hcB.NextTickTime()

				return bTicksBefore == bTicksAfter &&
					bNextTickBefore == bNextTickAfter
			},
			genDistinctIntervals,
			genFrameCount,
		),
	)

	properties.TestingRun(t)
}
```

| Factor | Assessment | Notes |
|--------|-----------|-------|
| Input space size | bounded | tick intervals from 5–50ms range; N in 1–100 frames |
