# VP-061: Metrics Output Contains No Session Content or Keystroke Data

## Property Statement

∀ response produced by `internal/metrics` query handlers (`PathsListHandler`,
`RouterMetricsHandler`, and the `RouterStatusHandler` alias shim):

1. The serialized JSON output contains no field named or semantically equivalent to:
   `session_bytes`, `keystroke_count`, `payload_bytes`, `keystroke_data`, `content`,
   `upstream_bytes`, or any field carrying session-layer payload content.
2. The field set is bounded to routing/path telemetry only:
   `path_id`, `router_addr`, `rtt_ms`, `rtt_p99_ms`, `loss_pct`, `status`, `quality`
   (alias only), `frame_count`, `hmac_fail_count`, `drop_cache_hits`, `path_distribution`.
3. No struct type used in the metrics serialization path embeds or transitively
   contains fields from `internal/session`, `internal/tmux`, or `cmd/sbctl`
   session-payload types.

This property enforces BC-2.06.003 Invariant 2 and DI-001 (carrier-grade content
separation: metrics contain no session content).

**Verification phase:** Phase 6 (formal hardening). Not blocking Wave 5
implementation — implementation correctness is verified by VP-047 (integration).
This property provides the formal content-absence guarantee.

## Source Contract

- **BC:** BC-2.06.003 — Per-path RTT and loss metrics queryable via sbctl
- **Invariant 2:** "Metrics do not include session content, keystroke counts, or any user data."
- **DI-001:** Carrier-grade content separation — metrics contain no session-layer content.

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| code-audit | manual + grep/ast analysis | yes (static) | Enumerate all struct types reachable from metrics response serialization; verify none carry session-payload fields |

### Audit Protocol

The auditor executes the following steps in Phase 6:

1. `grep -r 'json:"' internal/metrics/` — enumerate all JSON-tagged fields in the
   metrics package.
2. Verify each tagged field name appears in the approved field set (Invariant 2 above).
3. `grep -r 'session\|keystroke\|payload_bytes\|content' internal/metrics/` — assert
   zero matches (excluding comments and import paths).
4. Walk the Go import graph from `internal/metrics` to confirm no direct or transitive
   import of session-payload carrying types from `internal/session` or `internal/tmux`.
5. Record findings in Phase 6 audit log.

## Proof Harness Skeleton

```go
// This property is verified by static analysis (grep + import graph walk).
// The test below provides a dynamic regression guard: it serializes a live
// metrics response and asserts the JSON contains none of the prohibited keys.

package metrics_test

import (
	"encoding/json"
	"strings"
	"testing"

	"github.com/arcavenae/switchboard/internal/metrics"
)

// prohibitedKeys is the exhaustive set of field names that must never
// appear in any metrics JSON response (DI-001 enforcement).
var prohibitedKeys = []string{
	"session_bytes",
	"keystroke_count",
	"keystroke_data",
	"payload_bytes",
	"upstream_bytes",
	"content",
}

// TestMetrics_NoSessionContentInOutput verifies VP-061: the serialized
// metrics response for all handler types contains no prohibited content fields.
func TestMetrics_NoSessionContentInOutput(t *testing.T) {
	t.Parallel()

	// Serialize a representative PathsListResponse (the broadest response type).
	resp := metrics.PathsListResponse{
		Paths: []metrics.PathEntry{
			{
				PathID:     "path-001",
				RouterAddr: "127.0.0.1:9000",
				RTTMs:      15.0,
				RTTp99Ms:   metrics.RTTValue{Valid: true, Value: 22.0},
				LossPct:    0.1,
				Status:     "active",
			},
		},
	}

	data, err := json.Marshal(resp)
	if err != nil {
		t.Fatalf("json.Marshal: %v", err)
	}

	lowered := strings.ToLower(string(data))

	for _, key := range prohibitedKeys {
		if strings.Contains(lowered, `"`+key+`"`) {
			t.Errorf("prohibited field %q found in metrics JSON output (DI-001 violation)", key)
		}
	}
}

// TestRouterMetrics_NoSessionContentInOutput verifies VP-061 for the
// RouterMetricsResponse type (per-SVTN forwarding metrics).
func TestRouterMetrics_NoSessionContentInOutput(t *testing.T) {
	t.Parallel()

	resp := metrics.RouterMetricsResponse{
		FrameCount:     1000,
		HMACFailCount:  2,
		DropCacheHits:  5,
		PathDistribution: map[string]int64{"path-001": 990, "path-002": 10},
	}

	data, err := json.Marshal(resp)
	if err != nil {
		t.Fatalf("json.Marshal: %v", err)
	}

	lowered := strings.ToLower(string(data))

	for _, key := range prohibitedKeys {
		if strings.Contains(lowered, `"`+key+`"`) {
			t.Errorf("prohibited field %q found in router metrics JSON output (DI-001 violation)", key)
		}
	}
}
```
