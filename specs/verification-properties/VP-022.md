# VP-022: Replay.OnUpstream Never Delivers Same Sequence Twice

## Property Statement

∀ sequence of OnUpstream calls on a Replay instance:
  ∀ chan_seq N:
    count(frames passed to deliver() with chan_seq == N) ≤ 1

A duplicate upstream frame (same chan_seq as a previously delivered frame)
must be silently discarded and must not be passed to the downstream deliver
callback a second time.

## Source Contract

- **BC:** BC-2.02.004 — Upstream idempotent replay window: each frame carries last N keystrokes
- **Postcondition/Invariant:** Invariant — "Replay deliver callback is called at most once per chan_seq."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| proptest | gopter v0.2.9+ | no | random sequences of OnUpstream calls including deliberate duplicates; tracks deliver count per chan_seq |

## Proof Harness Skeleton

```go
package replay_test

import (
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/frame"
	"github.com/arcavenae/switchboard/internal/replay"
)

func TestProp_Replay_NoDuplicateDelivery(t *testing.T) {
	properties := gopter.NewProperties(nil)

	properties.Property("OnUpstream delivers each chan_seq at most once", prop.ForAll(
		func(seqs []uint8) bool {
			deliverCount := make(map[uint64]int)
			r := replay.New(replay.Config{WindowSize: 64})
			r.OnDeliver(func(f frame.Frame) {
				deliverCount[f.ChanSeq]++
			})

			for _, s := range seqs {
				f := frame.Frame{}
				f.ChanSeq = uint64(s % 16) // small range to force duplicates
				r.OnUpstream(f)
			}

			for _, count := range deliverCount {
				if count > 1 {
					return false
				}
			}
			return true
		},
		gen.SliceOfN(32, gen.UInt8()),
	))

	properties.TestingRun(t)
}
```
