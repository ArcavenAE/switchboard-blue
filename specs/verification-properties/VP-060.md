# VP-060: Daemon Lifecycle — Connect-Failure Exits Non-Zero; SIGTERM/SIGINT Triggers Clean Shutdown Without Goroutine Leak

## Property Statement

∀ launch of the `cmd/switchboard` binary:

**Property A (connect-failure path):**
1. If `SessionConnector.Connect(ctx)` returns a non-nil error (both tmux
   control-mode and PTY fallback exhausted), the process logs the error to
   stderr in E-SYS-002 format and exits with a non-zero exit code (exit 1).
2. No relay goroutines are started before the process exits.
3. The process does not panic.

**Property B (clean-shutdown path):**
1. On receipt of SIGTERM or SIGINT while the relay loop is running, the root
   context is cancelled.
2. The relay goroutine (frames channel), the sweep ticker goroutine, and the
   frames-dropped ticker goroutine all exit — no goroutines are leaked.
3. `sc.Close()` is called exactly once (`sync.Once`-guarded).
4. The process exits with code 0.
5. No panic occurs.

These properties cover BC-2.04.007 PC-1 and PC-2 postconditions. They are
observable by the OS process supervisor and verified by integration tests that
spawn the binary as a subprocess.

## Source Contract

- **BC:** BC-2.04.007 — Access Node Daemon Startup Succeeds or Exits Non-Zero;
  SIGTERM/SIGINT Triggers Clean Shutdown
- **Postconditions (PC-1):** exit code 1, E-SYS-002 to stderr, no relay goroutines
  started, no panic.
- **Postconditions (PC-2):** root ctx cancelled, all goroutines drain, `sc.Close()`
  called once, exit code 0, no goroutine leak, no panic.
- **Invariants:** Fail-before-serve (I-1), Exactly-once close (I-2), Signal
  idempotency (I-3), DI-001 compliance (I-4).

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| integration (subprocess) | `os/exec` + `go test` | yes | Launch binary as subprocess; inject failures via mock session backend; send OS signals; inspect exit code and stderr |
| goroutine leak detection | `t.Cleanup` + `time.AfterFunc` timeout | yes | After signal sent, assert no leaked goroutines within 500ms deadline |

The verification strategy is integration testing via subprocess because the property
is observable only at the OS process boundary — exit code and signal handling are
effectful shell behaviours that lie outside the pure-core proptest boundary.

## Proof Harness Skeleton

```go
package main_test

import (
    "context"
    "os"
    "os/exec"
    "strings"
    "syscall"
    "testing"
    "time"
)

// TestDaemon_ConnectFailure_ExitsNonZero verifies VP-060 Property A:
// when the session backend is unavailable, the daemon exits 1 with E-SYS-002
// and does not enter the relay loop.
func TestDaemon_ConnectFailure_ExitsNonZero(t *testing.T) {
    t.Helper()
    // Build the binary under test first (or use a pre-built path from $TEST_BINARY).
    binary := binaryPath(t)

    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    t.Cleanup(cancel)

    // Launch with an unreachable socket / test flag that forces Connect() to fail.
    cmd := exec.CommandContext(ctx, binary, "--test-connect-fail")
    var stderr strings.Builder
    cmd.Stderr = &stderr

    err := cmd.Run()
    if err == nil {
        t.Fatal("expected non-zero exit, got exit 0")
    }

    exitErr, ok := err.(*exec.ExitError)
    if !ok {
        t.Fatalf("expected ExitError, got: %T %v", err, err)
    }
    if exitErr.ExitCode() == 0 {
        t.Fatalf("expected non-zero exit code, got 0")
    }

    if !strings.Contains(stderr.String(), "fatal: cannot connect to session backend") {
        t.Errorf("expected E-SYS-002 message on stderr, got: %q", stderr.String())
    }
}

// TestDaemon_SIGTERM_CleanShutdown verifies VP-060 Property B:
// SIGTERM cancels the root context, drains all goroutines, calls sc.Close() once,
// exits 0, no panic.
func TestDaemon_SIGTERM_CleanShutdown(t *testing.T) {
    t.Helper()
    binary := binaryPath(t)

    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    t.Cleanup(cancel)

    // Launch daemon in test mode that uses a mock backend (Connect succeeds).
    cmd := exec.CommandContext(ctx, binary, "--test-mock-backend")
    if err := cmd.Start(); err != nil {
        t.Fatalf("start daemon: %v", err)
    }

    // Allow relay loop to start.
    time.Sleep(50 * time.Millisecond)

    // Deliver SIGTERM.
    if err := cmd.Process.Signal(syscall.SIGTERM); err != nil {
        t.Fatalf("send SIGTERM: %v", err)
    }

    // Wait for clean exit within deadline.
    done := make(chan error, 1)
    go func() { done <- cmd.Wait() }()

    select {
    case err := <-done:
        if err != nil {
            t.Fatalf("expected exit 0, got: %v", err)
        }
    case <-time.After(500 * time.Millisecond):
        _ = cmd.Process.Kill()
        t.Fatal("daemon did not exit within 500ms of SIGTERM — goroutine leak suspected")
    }
}

// binaryPath returns the path to the compiled test binary.
// In CI, TEST_BINARY env var points to the pre-built binary.
// Locally, it builds via `go build`.
func binaryPath(t *testing.T) string {
    t.Helper()
    if p := os.Getenv("TEST_BINARY"); p != "" {
        return p
    }
    tmp := t.TempDir()
    out := tmp + "/switchboard"
    cmd := exec.Command("go", "build", "-o", out, "./cmd/switchboard")
    if out, err := cmd.CombinedOutput(); err != nil {
        t.Fatalf("build binary: %v\n%s", err, out)
    }
    return out
}
```
