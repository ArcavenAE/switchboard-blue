# VP-072: mgmt.Server Sets Write Deadline Before Every sendJSON Call (Slowloris-on-Write Defense)

## Property Statement

Before every `sendJSON` call in `handleConnection`, `mgmt.Server` MUST call
`conn.SetWriteDeadline(time.Now().Add(d))` where:
- `d = mgmt.HandshakeTimeout` (10s) for handshake-phase sends: CHALLENGE, AUTH_OK,
  AUTH_FAIL.
- `d = mgmt.RPCIdleTimeout` (30s) for RPC response sends.

After each `sendJSON` completes (success or error), the write deadline MUST be
cleared to `time.Time{}` (the zero value, meaning "no deadline") before the
goroutine proceeds to its next blocking operation.

This closes the symmetric slowloris-on-write slot-exhaustion vector (CWE-400):
a client that accepts the TCP connection but reads infinitely slowly cannot hold
a connection goroutine indefinitely just by not reading the server's output.

The write deadline is independent of the read deadline (PC-1 read deadline behavior
is covered by VP-064).

## Proof Harness Skeleton

```go
package mgmt_test

import (
    "context"
    "crypto/ed25519"
    "crypto/rand"
    "net"
    "testing"
    "time"

    "github.com/arcavenae/switchboard/internal/mgmt"
)

// TestWriteDeadline_SlowlorisDefense_VP072 verifies VP-072:
// Write deadline is set before every sendJSON; a slow reader causes connection timeout,
// not indefinite goroutine hang.
func TestWriteDeadline_SlowlorisDefense_VP072(t *testing.T) {
    t.Parallel()

    // Override HandshakeTimeout to a short value for test speed.
    origTimeout := mgmt.HandshakeTimeout
    mgmt.HandshakeTimeout = 100 * time.Millisecond
    t.Cleanup(func() { mgmt.HandshakeTimeout = origTimeout })

    ln, err := net.Listen("tcp", "127.0.0.1:0")
    if err != nil {
        t.Fatalf("listen: %v", err)
    }
    t.Cleanup(func() { ln.Close() })

    _, daemonKey, _ := ed25519.GenerateKey(rand.Reader)
    srv := mgmt.NewServer(ln, daemonKey, nil, nil, "dev")

    ctx, cancel := context.WithCancel(context.Background())
    t.Cleanup(cancel)
    go srv.Serve(ctx) //nolint:errcheck

    // Connect a client that never reads from the socket.
    slowConn, err := net.DialTimeout("tcp", ln.Addr().String(), time.Second)
    if err != nil {
        t.Fatalf("dial: %v", err)
    }
    t.Cleanup(func() { slowConn.Close() })
    // Do not read anything — simulate a slowloris write stall on the server side.

    // The server's CHALLENGE write should time out. After HandshakeTimeout the
    // connection goroutine must exit (no goroutine leak). We verify the server is
    // still healthy by connecting a second, cooperative client.
    time.Sleep(mgmt.HandshakeTimeout + 50*time.Millisecond)

    opPub, opPriv, _ := ed25519.GenerateKey(rand.Reader)
    keySet := mgmt.NewOperatorKeySet([]ed25519.PublicKey{opPub})
    // Rebuild server on same listener (or connect on same live server).
    // The point: server is not deadlocked by the slow-reader goroutine.
    goodConn, err := net.DialTimeout("tcp", ln.Addr().String(), time.Second)
    if err != nil {
        t.Fatalf("VP-072 violated: server deadlocked or stopped after slow-reader: %v", err)
    }
    t.Cleanup(func() { goodConn.Close() })
    _ = opPriv
    _ = keySet
    // Optionally complete handshake on goodConn to confirm server is still serving.
    // (Full handshake covered by VP-064; here we just check accept works.)
}
```
