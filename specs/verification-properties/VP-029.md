# VP-029: Config.Validate Rejects Missing Required Fields

## Property Statement

∀ required field F in {listen_addr, tick_interval} of config.Config
  (internal/config/config.go:118 — "Required fields: ListenAddr, TickInterval"):
  zeroing F while all other fields are valid causes
  (&Config{...}).Validate() to return a non-nil error
  ∧ the error carries code E-CFG-001 (internal/config/config.go:43)
  ∧ the error message names field F

Each required field must be independently validated. Omitting any single
required field must produce at least one error; the error must be non-nil
and carry E-CFG-001 when any required field is absent.

Note: `tick_interval` zero-value means absent (not just out-of-range);
the validation check at config.go:203 treats any value outside [5ms, 50ms]
— including 0 — as a required-field violation.

## Source Contract

- **BC:** BC-2.09.003 — Router startup fails cleanly on malformed config with actionable error message
- **Postcondition/Invariant (derived):** BC-2.09.003 Postcondition 1 — "The daemon exits with a non-zero exit code before accepting any connections" when given a malformed config. A missing required field is a malformed-config subcase: `(*Config).Validate()` must return a non-nil E-CFG-001 error naming the absent field. This VP is a DERIVED property: BC-2.09.003 Postcondition 1 establishes the top-level guarantee (exit non-zero on malformed config); this VP specifies the mechanism — `(*Config).Validate()` returns a named field error — that realizes Postcondition 1 for the missing-required-field case. BC-2.09.003 Postcondition 2 (stderr E-CFG-001 format) and Invariant 2 (error messages name the specific field) additionally anchor the requirement that the error is non-nil and field-specific.

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| table-driven | standard testing.T | yes | one test case per required field: (1) ListenAddr="" with valid TickInterval; (2) TickInterval=0 with valid ListenAddr; (3) TickInterval=1ms (below min) with valid ListenAddr |

**Justification for table-driven over proptest:** The set of required fields is finite and enumerable (exactly two: `listen_addr` and `tick_interval`, per config.go:117–118 doc comment). There is no random input dimension — each test case either has the field absent (zero value) or not. Proptest would iterate over the same two cases with no additional coverage. Table-driven is the correct tool for finite enumeration; it also matches the existing implementation at `TestVP029_ErrorMessageNamesFieldAndSuggestion` (config_test.go:647). The feasibility table already noted "finite / one test case per required field" in v1.1; this re-anchor makes the method declaration consistent.

## Proof Harness Skeleton

```go
package config_test

import (
	"errors"
	"strings"
	"testing"
	"time"

	"github.com/arcavenae/switchboard/internal/config"
)

// TestVP029_MissingRequiredFieldReturnsECFG001 verifies that zeroing each
// required field of config.Config independently causes Validate to return
// a non-nil E-CFG-001 error naming the absent field.
func TestVP029_MissingRequiredFieldReturnsECFG001(t *testing.T) {
	t.Parallel()

	cases := []struct {
		name      string
		cfg       *config.Config
		wantField string
	}{
		{
			name: "listen_addr_empty",
			cfg: &config.Config{
				ListenAddr:   "",
				TickInterval: 10 * time.Millisecond,
			},
			wantField: "listen_addr",
		},
		{
			name: "tick_interval_zero",
			cfg: &config.Config{
				ListenAddr:   "0.0.0.0:9090",
				TickInterval: 0,
			},
			wantField: "tick_interval",
		},
		{
			name: "tick_interval_below_min",
			cfg: &config.Config{
				ListenAddr:   "0.0.0.0:9090",
				TickInterval: 1 * time.Millisecond,
			},
			wantField: "tick_interval",
		},
	}

	for _, tc := range cases {
		tc := tc
		t.Run(tc.name, func(t *testing.T) {
			t.Parallel()

			err := tc.cfg.Validate()
			if err == nil {
				t.Fatalf("expected validation error for missing %s, got nil", tc.wantField)
			}

			var ce *config.ConfigError
			if !errors.As(err, &ce) || ce.Code != "E-CFG-001" {
				t.Fatalf("expected E-CFG-001, got %v", err)
			}
			if !strings.Contains(err.Error(), tc.wantField) {
				t.Fatalf("expected field name %q in error, got: %q", tc.wantField, err.Error())
			}
		})
	}
}
```
