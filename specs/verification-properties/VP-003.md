# VP-003: EncodeOuterHeader Produces Exactly 44 Bytes

## Property Statement

∀ valid OuterHeader h:
  len(EncodeOuterHeader(h)) == 44 ∧ err == nil

The wire format is fixed-width within major protocol version 0 (v0.x). No valid
input may cause the encoder to produce a shorter or longer buffer. This
property is an independent check from VP-001's round-trip: it verifies the
size invariant in isolation so a size regression is directly observable.

## Source Contract

- **BC:** BC-2.01.004 — Frame outer-header encoding and decoding at 44-byte fixed layout
- **Postcondition/Invariant:** Postcondition 1 — "The serialized outer header is exactly 44 bytes in big-endian byte order."

## Proof Harness Skeleton

```go
package frame_test

import (
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/frame"
)

func TestPropEncodeSize_OuterHeader(t *testing.T) {
	properties := gopter.NewProperties(nil)

	properties.Property("EncodeOuterHeader always produces exactly 44 bytes", prop.ForAll(
		func(h frame.OuterHeader) bool {
			encoded, err := frame.EncodeOuterHeader(h)
			if err != nil {
				return false
			}
			return len(encoded) == 44
		},
		genValidOuterHeader(), // reuse generator from VP-001
	))

	properties.TestingRun(t)
}
```
