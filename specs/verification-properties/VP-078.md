# VP-078: sbctl paths ping Reports rtt_ms as float64 With No Quality/Status Field

## Property Statement

∀ invocation of `sbctl paths ping --router=<addr>` that completes a
successful round trip (dial succeeds, Tier-1 authentication succeeds,
`paths.ping` RPC completes):

1. The JSON response contains `rtt_ms` typed as a float64 (never a string,
   never omitted, never a `"pending"` sentinel — `paths ping` performs no
   sample accumulation, so there is no pending state to represent).
2. The response contains no field named or semantically equivalent to
   `quality`, `status`, `green`/`yellow`/`red`, or any other health/quality
   classification. `paths.ping` is a one-shot stopwatch, not a classifier
   (BC-2.06.004 PC-4, Description).
3. Property 1 and Property 2 both hold identically for a fast round trip
   (single-digit-ms `rtt_ms`) and a measurably slow round trip
   (hundreds-of-ms `rtt_ms`, forced via an artificial daemon-side delay) —
   a slow response is NOT an error and does not change the response shape
   (BC-2.06.004 EC-003, Invariant 2).

This property enforces BC-2.06.004 Postcondition 1, Postcondition 4, and
Invariant 2 (no quality/status classification — that remains exclusively
`router.status`'s job per BC-2.06.003 PC-3).

## Source Contract

- **BC:** BC-2.06.004 — On-Demand Single-Target Reachability Probe via `sbctl paths ping`
- **PC-1:** "sbctl reports round-trip time in milliseconds measured client-side... `{"router": "<addr>", "rtt_ms": <float64>}`."
- **PC-4 / Invariant 2:** "`paths.ping` performs no per-path metrics computation and returns no quality classification (no green/yellow/red field)... Quality classification remains exclusively `router.status`'s job."
- **EC-003:** "Connection succeeds, authentication succeeds, but the round trip is slow... NOT an error. `rtt_ms` reports the measured (larger) value; exit 0... no green/yellow/red output."

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| integration | `go test` against a stub daemon simulating the ADR-012 challenge-response handshake | yes | Two sub-cases: fast round trip (immediate response) and slow round trip (artificial daemon-side delay before response) |

### Test Scenarios

1. **Fast round trip:** stub daemon completes the ADR-012 handshake and
   returns `{"pong": true}` immediately. Assert the decoded response JSON
   has `rtt_ms` present as a JSON number (not string), and has no
   `quality`/`status` key at any level.
2. **Slow round trip:** stub daemon sleeps a fixed, measurable delay
   (e.g. 200ms) after the handshake before writing the `paths.ping`
   response. Assert `rtt_ms` reflects at least the forced delay (loose
   lower-bound check, not an exact-timing assertion — avoids CI flakiness),
   exit code is 0 (not an error), and — identically to the fast case — no
   `quality`/`status` key is present.
3. Both sub-cases assert `rtt_ms`'s JSON-decoded Go type is `float64` via
   `json.Unmarshal` into `map[string]interface{}`, not merely that the raw
   bytes look numeric.

## Proof Harness Skeleton

```go
package main_test

// Illustrative shape only. cmd/sbctl's paths-ping integration tests already
// establish a stub-daemon + ADR-012-handshake fixture family for exercising
// `sbctl paths ping`; this skeleton assumes an equivalent fixture without
// re-deriving it.

import (
	"encoding/json"
	"testing"
)

// TestVP078_PathsPing_RTTIsFloat64_NoQualityField_FastAndSlow verifies
// VP-078: rtt_ms is always a float64, quality/status is always absent,
// for both fast and slow round trips.
func TestVP078_PathsPing_RTTIsFloat64_NoQualityField_FastAndSlow(t *testing.T) {
	cases := []struct {
		name  string
		delay bool // true: stub daemon sleeps before responding
	}{
		{"fast_round_trip", false},
		{"slow_round_trip", true},
	}

	for _, tc := range cases {
		tc := tc
		t.Run(tc.name, func(t *testing.T) {
			// ... dial stub daemon (delayed or not per tc.delay), run
			// `sbctl paths ping --router=<sock>`, decode the JSON envelope's
			// data payload into a generic map ...

			var data map[string]interface{}
			// data = decode(...)

			rtt, ok := data["rtt_ms"]
			if !ok {
				t.Fatal("rtt_ms missing from response")
			}
			if _, isFloat := rtt.(float64); !isFloat {
				t.Errorf("rtt_ms decoded as %T, want float64", rtt)
			}

			for _, prohibited := range []string{"quality", "status", "green", "yellow", "red"} {
				if _, present := data[prohibited]; present {
					t.Errorf("prohibited field %q present in paths ping response (Invariant 2 violation)", prohibited)
				}
			}
		})
	}
}
```
