# VP-013: SessionAuth Rejects Upstream from Read-Only Key

## Property Statement

∀ key ConsoleKey with role == RoleReadOnly registered via SessionAuth.RegisterKey,
  payload []byte where len(payload) > 0:
  sa.Allow(key, sessionName, payload) → errors.Is(err, session.ErrUpstreamReadOnly)

A console key granted the read-only role on a session must never be
permitted to send upstream (keystroke) frames with non-empty payload.
The access node enforces this via SessionAuth.Allow regardless of Tier 1
admission status. The role is an enum (RoleFull, RoleReadOnly); the
upstream check is deterministic on the role value and payload length.

## Source Contract

- **BC:** BC-2.04.005 — Read-only console receives downstream stream; upstream keystrokes are rejected by access node
- **Postcondition/Invariant:** Postcondition implied by BC-2.04.005 title and BC-2.05.003 Invariant DI-010 — the access node enforces per-session role restrictions; read-only consoles receive no upstream path.

## Proof Harness Skeleton

```go
package session_test

import (
	"errors"
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/session"
)

func TestPropSessionAuth_ReadOnlyKeyDeniedUpstream(t *testing.T) {
	properties := gopter.NewProperties(nil)

	properties.Property("read-only key always returns ErrUpstreamReadOnly for non-empty upstream payload", prop.ForAll(
		func(payload []byte) bool {
			if len(payload) == 0 {
				// Empty payload is the empty-tick exemption (BC-2.04.005 EC-004);
				// Allow returns nil — not a violation.
				return true
			}

			sa := session.NewSessionAuth()
			const sessionName = "test-session"
			key := session.ConsoleKey("test-console")
			sa.RegisterKey(sessionName, key, session.RoleReadOnly)

			err := sa.Allow(key, sessionName, payload)
			return errors.Is(err, session.ErrUpstreamReadOnly)
		},
		gen.SliceOf(gen.UInt8()),
	))

	properties.TestingRun(t)
}
```
