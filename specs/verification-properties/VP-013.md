# VP-013: SessionAuth Rejects Upstream from Read-Only Key

## Property Statement

∀ console_key PublicKey with role == readonly,
  frame_type FrameType where frame_type == Upstream:
  Authorize(console_key, auth_list, frame_type) → ErrReadOnlyUpstreamDenied

A console key granted the read-only role on a session must never be
permitted to send upstream (keystroke) frames. The access node enforces
this regardless of Tier 1 admission status. The role is an enum (readonly,
readwrite); the upstream check is deterministic on the role value.

## Source Contract

- **BC:** BC-2.04.005 — Read-Only Console Receives Downstream Stream; Upstream Keystrokes Are Rejected by Access Node
- **Postcondition/Invariant:** Postcondition implied by BC-2.04.005 title and BC-2.05.003 Invariant DI-010 — the access node enforces per-session role restrictions; read-only consoles receive no upstream path.

## Proof Harness Skeleton

```go
package session_test

import (
	"crypto/ed25519"
	"crypto/rand"
	"errors"
	"testing"

	"github.com/leanovate/gopter"
	"github.com/leanovate/gopter/gen"
	"github.com/leanovate/gopter/prop"

	"github.com/arcavenae/switchboard/internal/frame"
	"github.com/arcavenae/switchboard/internal/session"
)

func TestPropSessionAuth_ReadOnlyKeyDeniedUpstream(t *testing.T) {
	properties := gopter.NewProperties(nil)

	properties.Property("read-only key always returns ErrReadOnlyUpstreamDenied for upstream frames", prop.ForAll(
		func(_ int) bool {
			pub, _, err := ed25519.GenerateKey(rand.Reader)
			if err != nil {
				return true
			}

			authList := session.NewAuthList()
			authList.AddWithRole(pub, session.RoleReadOnly)

			// Upstream frame type.
			err2 := session.AuthorizeFrameType(pub, authList, frame.TypeUpstream)
			return errors.Is(err2, session.ErrReadOnlyUpstreamDenied)
		},
		gen.IntRange(0, 0), // dummy arg; all state is internal
	))

	properties.TestingRun(t)
}
```
